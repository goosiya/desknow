# DeskNow 프로젝트 규칙

## 용어 규칙 — 지역(IMPORTANT, 반드시 준수)

이 프로젝트의 동/읍/면·시군구 단위 지역 개념은 **무조건 "지역" 하나로만 부른다.**
**"행정동"·"법정동" 이라는 한글 표현을 새로 쓰는 것을 금지한다.** (KTH 확정 2026-06-19)

### 배경
프로젝트 초반에 지역 코드 기준을 **행정동 → 법정동(法定洞, 카카오 `b_code`)** 으로 바꿨다.
실제 구현·데이터는 법정동 b_code가 맞다. 그런데 기획/스토리/주석/UI에 "행정동"과 "법정동"이
구분 없이 섞여 들어가 혼란을 유발했다. 또한 일반 사용자에게 "행정동/법정동" 표현은 생소하다.
그래서 노출되는 모든 한글 용어를 **"지역"** 으로 통일한다.

### 규칙
- **소스 코드(웹/API의 `.ts`/`.tsx`/`.py`)** — UI 문자열·주석·docstring 모두 "지역"만 사용.
  "행정동"·"법정동" 표현 금지. (2026-06-19 전수 치환 완료.)
- **설계/진행 문서(`_bmad-output/**` 의 md)** — 과거 문서는 그대로 두되, **앞으로 작성·수정하는
  모든 문서(스토리·스프린트·아키텍처·PRD 등)에서 "행정동"을 절대 쓰지 않는다.** "지역"으로 표기.
- **데이터 기준(법정동 b_code)은 영문 식별자로만 표현한다** — `admin_dong_code`(법정동을 담은
  misnomer 컬럼, 리네임=마이그레이션이라 defer), `b_code`, `legal_dong.json`. 한글 프로즈로
  "법정동"이라 쓰지 말 것.
- **유일한 예외 2곳(수정 금지):**
  1. `apps/api/app/rooms/data/legal_dong.json` — 실제 지명 데이터(예: "충청남도 당진시 행정동"은
     실재하는 법정동 이름). 데이터 무수정.
  2. `apps/api/app/rooms/regions.py` 도큐먼트의 **데이터셋 공식 명칭 인용**("국토교통부 법정동코드
     공공데이터셋", data.go.kr 15123287). 고유명사 — 동일 데이터셋 재확보를 위해 보존.
