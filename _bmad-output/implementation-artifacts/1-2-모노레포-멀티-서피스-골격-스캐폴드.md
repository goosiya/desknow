# Story 1.2: 모노레포 & 멀티 서피스 골격 스캐폴드

Status: done

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a 개발자,
I want create-turbo 기반 모노레포에 사용자 웹·관리자 웹·RN 앱·FastAPI 백엔드와 공유 패키지의 골격을 세우길,
so that 세 클라이언트와 단일 백엔드가 한 저장소에서 빌드·실행되는 "걸음마 가능한" 골격이 마련된다.

> 🎯 **이 스토리의 본질은 "걸음마 골격(walking skeleton)"이다.** 구조·빌드·기동 파이프라인만 세운다. 디자인 토큰(1.6)·DB 연결(1.4)·core/time·core/errors(1.5)·인증(1.7/1.8)·OpenAPI→SDK 생성(1.9)은 **이 스토리 범위 밖**이다. 공유 패키지(`packages/*`)와 도메인 모듈은 **빈 골격(placeholder)** 으로만 만든다 — 후속 스토리가 채운다.

## Acceptance Criteria

1. **(AC1 — 구조 생성)** 아키텍처 §Starter 초기화 절차(create-turbo → create-next-app ×2 → create-expo-app → FastAPI/uv)를 수행하면, `apps/{web,admin,mobile,api}` + `packages/{api-client,ui,config}` 구조가 존재하고 각 도구가 init 시점 최신 버전(Turborepo 2.9.x / Next.js 16.x / Expo SDK 56)으로 고정된다.
2. **(AC2 — 기동)** `pnpm turbo dev`(web/admin/mobile)와 `uv run uvicorn`(api)을 실행하면 세 프론트가 각자 포트로 기동되고, 백엔드가 `GET /api/v1/health` → 200을 응답하며, RN 앱은 LAN IP API base로 백엔드에 연결된다.
3. **(AC3 — FastAPI 셸)** 백엔드가 기동되면 CORS(웹 origin 등록), `/api/v1` 라우터 프리픽스, 도메인 모듈 패키지 골격(`auth·rooms·reservations·reviews·notifications·chatbot·admin`)이 **빈 상태**로 존재한다.
4. **(AC4 — 느슨한 공존)** 파이썬(`apps/api`)은 pnpm 워크스페이스 **밖**에서 uv로 느슨하게 공존하며(pnpm이 관리하지 않음), JS↔Python 결합은 OpenAPI 스키마로만 이루어진다(이 스토리에서 SDK 생성은 안 하지만, 결합 경계가 코드 import가 아님을 구조로 보장).

> ⚠️ **회귀 방지 — 기존 자산 보존(최우선 불변식):** 이 리포에는 이미 `apps/api`(Story 1.1 최소 부트스트랩: `config.py`·`.env`·`.env.example`·`pyproject.toml`·`tests/`), 루트 `.gitignore`, `docs/`, `_bmad`/`_bmad-output`/`.claude`가 존재한다. **`create-turbo desknow`를 빈 디렉터리 가정으로 그대로 실행하면 안 된다**(충돌/중첩 폴더 생성). 아래 Task 1의 in-place 전략을 따른다. 특히 `apps/api/app/core/config.py`·`.env.example`·루트 `.gitignore`는 **절대 덮어쓰지 말고** 확장한다.

### 핵심 구현 디테일 (실수 방지)

- **포트:** web=3000, admin=3001, api=8000. 두 Next 앱 포트 충돌 방지가 `turbo dev` 동시 기동의 전제.
- **CORS:** `CORSMiddleware`로 `http://localhost:3000`·`http://localhost:3001` 등록. RN/모바일은 CORS 미적용(네이티브). `allow_credentials=True`(웹 쿠키 인증 1.8 대비), `allow_methods`/`allow_headers` 합리적 기본.
  [Source: architecture.md#API(L174), #Authentication(L160)]
- **uvicorn `--host 0.0.0.0`:** 모바일 LAN IP 접근의 전제. `127.0.0.1` 바인딩이면 RN이 못 닿는다.
- **헬스 엔드포인트는 DB 무의존:** 1.4 전이므로 DB 연결 시도 금지. `{"status":"ok"}` liveness만.
- **`main.py`의 fail-fast:** `get_settings()` 호출 시점에 1.1 `Settings`가 필수 키(`KAKAO_REST_API_KEY`/`KAKAO_JS_KEY`/`OPENAI_API_KEY`) 누락이면 `ValidationError`. `.env`는 소유자가 채워둠 → 정상 기동 예상. `DATABASE_URL`은 아직 Optional(1.4에서 필수화).
- **pnpm + Expo 상호작용:** 알려진 마찰점. 기본 시도 → 실패 시 `node-linker=hoisted`(루트 `.npmrc`) 또는 Metro monorepo 설정. 적용분을 완료 노트에 기록.
- **lint/type 게이트:** deferred-work.md의 1.1 리뷰 이월 항목 회수. ruff/mypy를 `apps/api`에 도입하되, 1.1 `config.py`가 ruff/mypy를 통과하도록 필요한 최소 정리만(대규모 리팩터 금지 — 기존 동작·테스트 보존).
