## Story

As a 개발자,
I want UTC 저장 / Asia/Seoul 판정의 단일 시간 유틸(`core/time`)과 표준 에러 스키마·에러코드 상수·전역 예외 핸들러(`core/errors`)를 갖추길,
so that 5개 시간경계 FR(FR-5·12·16·18·20)과 전 도메인 에러 처리가 **단일 출처로 결정적으로** 동작한다(NFR-1). 이후 스토리(1.7 가입·1.8 인증·E3 탐색·E4 예약·E5 알림/후기)가 시간 판정과 에러 응답을 손으로 짓지 않는다.

## Acceptance Criteria

1. **(AC1 — 시간 규약: UTC 저장 / KST 판정)** `core/time`이 갖춰진 상태에서 "오늘 / 현재시각 / N시간 전 / 24시간 이내"를 판정하면, **모든 판정이 룸 소재지 타임존(MVP=Asia/Seoul) 기준**으로 수행되고, 저장·전송은 **UTC(ISO-8601 `...Z`)** 로 일관된다. 날짜 경계 판정은 **절대 프로세스/클라이언트 로컬 타임존에 의존하지 않는다**(architecture.md L296 금지 안티패턴).
2. **(AC2 — 표준 에러 스키마 & 에러코드 상수)** `core/errors`와 **전역 예외 핸들러**가 등록된 상태에서 도메인 오류가 발생하면, 응답이 **`{ "detail": { "code": "...", "message": "..." } }`** 형태로 표준화되고, 코드는 **상수(`SLOT_CONFLICT`·`EMAIL_TAKEN`·`CANCEL_WINDOW_PASSED`·`FORBIDDEN_ROLE` 등)로만** 사용된다(**문자열 하드코딩 금지**, architecture.md L288).
3. **(AC3 — HTTP 상태 매핑)** 동시성 충돌은 **409**, 권한은 **403**, 미인증은 **401**, 검증 실패는 **422**로 매핑된다. 네 분류가 모두 표준 스키마(`detail.code`/`detail.message`)로 응답됨이 테스트로 실증된다.

> **이 스토리의 범위 결정(명시):**
> - **1.5는 횡단 *유틸*과 *인프라*만 제공한다.** 시간경계 FR(FR-5 핀색·FR-12 슬롯·FR-16 6h 취소·FR-18 24h 리마인드·FR-20 이용완료)과 도메인 에러를 *발생시키는* 라우터/서비스는 **각 소비 스토리(E3·E4·E5)가 구현**한다. 1.5는 그들이 호출할 단일 출처 함수·상수·핸들러·응답 스키마를 고정하고, 그 계약을 **DB·라우터 없이** 단위/경량 통합 테스트로 실증한다.
> - **검증 실패(422)의 단일화 결정:** FastAPI 기본 `RequestValidationError`는 `{ "detail": [ {loc, msg, type}, … ] }`(배열) 형태라 본 스토리의 표준 스키마와 충돌한다. 와이어 계약을 단일화하기 위해 **`RequestValidationError`를 표준 스키마(`{ detail: { code: "VALIDATION_ERROR", message } }`)로 변환하는 핸들러를 1.5에서 등록**한다(아직 검증 대상 엔드포인트가 health뿐이라 회귀 표면 최소 = 지금이 적기). 라우팅 레벨 404/405(Starlette `HTTPException`)는 **표준화 대상이 아니다**(프레임워크 기본 유지 — 범위 밖, 아래 안티패턴 참조).

  - [x] **도메인 예외 `DomainError(Exception)`**: `code: ErrorCode`, `message: str`, `status_code: int`을 보유. **기본 상태는 코드→상태 레지스트리에서 도출**하되 호출처가 override 가능. 편의 서브클래스 또는 팩토리(택1, 단순하게):
    ```python
    DEFAULT_STATUS: dict[ErrorCode, int] = {
        ErrorCode.SLOT_CONFLICT: 409,
        ErrorCode.EMAIL_TAKEN: 409,
        ErrorCode.CANCEL_WINDOW_PASSED: 409,
        ErrorCode.FORBIDDEN_ROLE: 403,
        ErrorCode.UNAUTHENTICATED: 401,
        ErrorCode.VALIDATION_ERROR: 422,
    }
    class DomainError(Exception):
        def __init__(self, code: ErrorCode, message: str, status_code: int | None = None):
            self.code = code
            self.message = message
            self.status_code = status_code if status_code is not None else DEFAULT_STATUS[code]
            super().__init__(message)
    ```

### 소비 계약 — 누가 이 유틸을 쓰는가 (구현은 각 스토리, 1.5는 계약만 고정)

NFR-1(epics.md L90)이 명시한 **5개 시간경계 FR** = 본 `core/time`의 소비처:
- **FR-5**(E3 탐색·핀색): "오늘 남은 빈 슬롯 ≥1" 판정 → `today_in_tz()`. (epics.md L564)
- **FR-12**(E4 슬롯 가용성): "현재시각 이후" 슬롯 도출 → `now_utc()`/`to_tz()`.
- **FR-16**(4.7 취소): "예약 시작 6시간 전(룸 타임존 기준) 이내면 취소 차단" → `is_within_hours(start, 6)`. (epics.md L809-818)
- **FR-18**(5.2 도래 리마인드): "시작 24시간 이내 확정 예약" → `is_within_hours(start, 24)`. (epics.md L56, L870)
- **FR-20**(5.5 후기): "이용 완료(마지막 슬롯 종료 경과)" → `hours_until(end) < 0` 류. (epics.md L63)

에러코드 소비처: `EMAIL_TAKEN`→1.7(가입), `UNAUTHENTICATED`(401)/`FORBIDDEN_ROLE`(403)→1.8(인증·RBAC), `SLOT_CONFLICT`→E4(동시성), `CANCEL_WINDOW_PASSED`→4.7. **본 스토리는 이 코드/함수가 존재하고 표준 스키마로 응답됨만 실증**한다(라우터 없음).

### 정확한 시간 규약 (AC1 — 그대로 적용)

- **표준 라이브러리만**: `from datetime import UTC, datetime, date` + `from zoneinfo import ZoneInfo`. 서드파티 `pytz`·`pendulum` 도입 금지(stdlib로 충분, 의존성 최소).
- **aware-only 불변식**: 시스템 경계를 넘는 모든 datetime은 **tz-aware**. naive는 `ValueError`로 즉시 거부 → "로컬 타임존으로 날짜 경계 판정"(architecture.md L296 금지) 원천 차단.
- **`...Z` 직렬화**: Pydantic v2 기본 직렬화는 aware datetime을 `+00:00`로 낸다. 와이어 규약은 `Z`(architecture.md L263). → `isoformat_utc()`가 단일 출처. (후속 스토리의 Pydantic 응답 모델은 datetime 필드에 이 규약을 쓰도록 — 재사용 `Annotated[datetime, PlainSerializer(lambda d: isoformat_utc(d), return_type=str)]` 타입 alias 제공은 **nice-to-have**, 본 스토리는 `isoformat_utc` 함수 제공이 필수.)
- **now 주입**: 모든 판정 함수는 `now`를 받아 결정적. 도메인 코드가 `datetime.now()`를 직접 호출하지 않도록 `now_utc()`를 유일 진입점으로.

### 정확한 에러 규약 (AC2/AC3 — 그대로 적용)

- **스키마**: `{ "detail": { "code": "<ErrorCode>", "message": "<사람이 읽는 안내>" } }`(architecture.md L260-262). 응답 래퍼 없음 원칙과 일관(L258) — 단 에러는 `detail` 객체로 표준화(FastAPI 관례 확장).
- **코드 상수화**: `ErrorCode(StrEnum)`. 라우터/서비스는 `ErrorCode.X`만. 프론트는 `detail.code`로 분기(메시지는 표시·로깅, 분기는 코드).
- **상태 매핑(AC3)**: 409=동시성/상태충돌(SLOT_CONFLICT·EMAIL_TAKEN·CANCEL_WINDOW_PASSED), 403=FORBIDDEN_ROLE, 401=UNAUTHENTICATED, 422=VALIDATION_ERROR. 레지스트리(`DEFAULT_STATUS`)가 코드→상태 단일 출처. 호출처 override 가능하나 기본을 신뢰.
- **검증 422 단일화**: `RequestValidationError` 핸들러가 FastAPI 기본 배열 형태를 표준 스키마로 변환. (OpenAPI 문서의 422 스키마 자동표기는 기본값이 남을 수 있음 — 후속에 `ErrorResponse` 명시 가능, 본 스토리 범위 밖의 nice-to-have.)

### 흔한 실수 방지 (anti-patterns)

- ❌ `datetime.now()`(naive)·`date.today()`로 "오늘/현재" 판정 → 프로세스 로컬 tz 의존(architecture.md L296 금지). `now_utc()`+`today_in_tz()`만.
- ❌ `ZoneInfo("Asia/Seoul")`를 쓰면서 `tzdata` 미설치 → Windows에서 `ZoneInfoNotFoundError`. Task 1 필수.
- ❌ ISO 문자열을 `+00:00`로 전송 → 와이어 규약 `Z` 위반. `isoformat_utc()`.
- ❌ 에러 코드 문자열 하드코딩(`"SLOT_CONFLICT"`) → `ErrorCode` enum만(AC2).
- ❌ 도메인 에러를 raw `HTTPException(409, detail="문자열")`로 → 표준 스키마(`detail.code/message`) 깨짐. `DomainError`만.
- ❌ Starlette `HTTPException`/404를 전역 오버라이드 → 프레임워크 충돌·`test_health_requires_v1_prefix` 회귀. 도메인+검증만 표준화.
- ❌ 테스트 위해 실 `app`에 임시 라우트 추가 → 미니 FastAPI 앱을 테스트 내부에서 생성(앱 오염 금지).
- ❌ `with TestClient(app)`로 test_main 변경 → lifespan DB 검증 트리거(1.4 회귀 불변식). 모듈 레벨 패턴 유지.
- ❌ async/`pytz`/`pendulum` 등 불필요 의존 도입 → stdlib(zoneinfo+datetime)로 충분.

## Latest Tech Information (2026-06 확인)

- **`zoneinfo` + `tzdata`(Windows 함정)**: stdlib `zoneinfo`(Python 3.9+)는 **OS 제공 IANA tz DB**를 읽는다. **Windows는 시스템 tz DB가 없어** `tzdata`(PyPI fallback) 미설치 시 `ZoneInfoNotFoundError("No time zone found with key Asia/Seoul")`. 일부 슬림 리눅스 컨테이너도 동일. → **`tzdata`를 명시적 런타임 의존성으로 고정**(이식성·결정성). [Python docs: zoneinfo / tzdata data-only package]
- **`datetime.UTC`·`StrEnum`**: `datetime.UTC`(별칭, 3.11+)·`enum.StrEnum`(3.11+) 모두 사용 가능(프로젝트 `requires-python>=3.12`). `StrEnum` 멤버는 `str` 인스턴스라 JSON 직렬화 시 값이 그대로(에러코드 상수에 이상적).
- **Pydantic v2 datetime 직렬화**: aware datetime → 기본 `2026-06-14T05:00:00+00:00`(`Z` 아님). 와이어 규약 `Z`를 위해 명시 변환(`isoformat_utc`) 또는 `PlainSerializer` 사용. [Pydantic v2 직렬화 문서]
- **FastAPI 예외 핸들러**: `app.add_exception_handler(ExcType, handler)` 또는 `@app.exception_handler(ExcType)`. `RequestValidationError`(`from fastapi.exceptions import RequestValidationError`)를 오버라이드하면 기본 422 본문을 커스텀 가능. 핸들러는 `JSONResponse`(`from fastapi.responses import JSONResponse`) 반환. [FastAPI: Handling Errors / custom exception handlers]
