## Story

As a 사용자,
I want 웹·앱·관리자가 동일한 "에너제틱 만다린" 디자인 토큰과 접근성·모션 하한선을 공유하길,
so that 화면 간 이질감이 없고(NFR-4) 접근성 기준을 충족한 일관된 셸 위에서 모든 기능이 구현된다(NFR-5, UX-DR1·11·12·13).

## Acceptance Criteria

1. **(AC1 — 토큰 3표면 동일 적용)** `packages/ui`에 토큰 단일 출처가 갖춰진 상태에서, 웹·관리자(shadcn/Tailwind v4)와 RN 앱이 토큰을 소비하면, **DESIGN.md의 색**(primary `#FF8A1E`, 의미색 success/pin-available `#157F45`·pin-full `#7E7466`·destructive `#CC3328` 등 전체 팔레트), **Pretendard 타이포 램프**(display~caption, 본문 행간 `1.6`), **spacing**(4px 기반), **radius**(0.5rem 패밀리), **elevation** 토큰이 **세 표면에서 동일하게 적용된다.** 동일성은 **토큰 단일 출처(`packages/ui/src/tokens.ts`)** 로 보장되며, web 소비(CSS preset)와 source 간 일치는 **parity 테스트**로 실증된다(RN은 같은 source를 직접 import → 구성상 동일).

2. **(AC2 — 라이트 전용 · 시맨틱 구조 · 다크 확장 여지)** 토큰을 정의하면, **시맨틱 구조**(shadcn CSS 변수 규약: `--background`/`--foreground`/`--primary`/`--ring`/… + 의미색)로 설계되어 다크 확장 여지를 남기되, **다크 값은 작성하지 않는다.** web/admin의 기존 `@media (prefers-color-scheme: dark)` 블록은 제거하고, RN은 라이트 스킴으로 고정한다(어디에도 다크 토큰 값 없음).

3. **(AC3 — 접근성 · 모션 셸 기본값)** 컴포넌트가 렌더되면, 다음이 **기본값으로** 보장된다: ⓐ **키보드 포커스 링** = `ring` 색 `#D86E0A`(크림 배경 대비 ≥3:1), ⓑ **터치 타겟 ≥44×44px(iOS) / ≥48×48dp(Android)** — 모든 탭 대상(탭바·FAB·배너 닫기·인터랙티브 컨트롤), ⓒ **`prefers-reduced-motion` 존중** — 비필수(장식) 모션은 축소/제거하되 **≤100ms 기능 피드백은 유지.**

4. **(AC4 — 반응형 셸 + 전역 레이어)** **반응형 셸**이 자리잡는다 — **웹**: `lg`(2열, 지도+사이드 목록 자리) / `md`(지도 우선 + 목록 시트 자리) / `sm`(단일 컬럼 + 하단 내비), **앱**: 단일 컬럼 + **하단 탭 1급 진입 3개**(스터디룸 찾기 · 예약현황 · 즐겨찾기), **관리자**: 데이터테이블·폼 중심(사이드바 내비). 그리고 **전역 레이어**(플로팅 챗봇 FAB **자리** · 인앱 배너 **자리**)가 세 표면 모두에 Z-순서로 자리잡는다.

> **🚧 이 스토리의 범위 결정 (스코프 크리프 방지 — 매우 중요):**
> - **1.6은 "토큰 + 셸(골격)"만 제공한다 — 기능 화면이 아니다.** 지도/목록/상세/예약/즐겨찾기/챗봇/배너의 **실제 기능 UI와 로직은 각 소비 에픽(E3·E4·E5·E7)이 구현**한다. 본 스토리는 그들이 올라설 토큰·셸·전역 레이어 **자리(슬롯·placeholder)** 만 고정한다.
> - **전역 레이어는 "자리"만이다:** 챗봇 FAB는 **프라이머리 원형 버튼 placeholder**(클릭 시 동작 없음/콘솔 로그, 챗봇 로직은 E7). 인앱 배너는 **빈 컨테이너 슬롯**(배너 데이터·dismiss 영속은 E5). 절대 챗봇/배너 기능을 구현하지 않는다.
> - **shadcn은 토큰 기반 + 1개 프루프 프리미티브(Button)까지만.** 전체 컴포넌트 도입은 각 소비 스토리가 `shadcn add`로 그때그때 추가한다(미리 컴포넌트 동물원 만들지 않음).
> - **RN 기기/Expo Go 검증은 본 스토리 밖이다.** Expo SDK56↔Expo Go 비호환으로 **기기 실측은 E7(EAS dev build)로 이관**(메모리·deferred-work.md 정렬). 1.6은 RN 토큰·셸 **코드 작성 + 타입/린트 그린**까지 책임지며, Pretendard config plugin은 dev build에서만 시각 확인된다(아래 Dev Notes "RN 검증 경계").
> - **범위 밖 명시:** 로그인/가입 화면(1.7·1.8), `packages/api-client` SDK(1.9), 다크 테마 값, 카카오맵, 지도/슬롯 컴포넌트의 실제 구현. 본 셸은 이들의 **자리만** 잡는다.

### 아키텍처 핵심 결정 (이 스토리에 직접 적용)

- **토큰 단일 출처 = `packages/ui/src/tokens.ts`(순수 데이터·무프레임워크).** architecture는 "web/admin/mobile은 도메인 로직 공유 안 하되 **타입·SDK·UI 토큰만 공유**"(L356), "디자인 토큰 packages/ui"(epics L204)로 명시. **packages/ui에 React/RN 컴포넌트를 넣지 않는다** — web이 RN 컴포넌트를(또는 그 반대) import하면 깨지므로, 공유물은 **플랫폼 무관 토큰**으로 한정. 셸·컴포넌트는 각 앱 소유.
  - **드리프트 차단:** web은 CSS(preset)로, RN은 TS로 소비 → 두 표현이 갈라질 수 있음. **tokens.ts를 정본**으로 두고 preset CSS는 그 값을 미러, **parity 테스트로 일치 강제**(AC1 "동일하게 적용"의 실증 수단). RN은 같은 tokens.ts를 직접 import하므로 구성상 동일.
- **Tailwind v4 = CSS-first(`@theme`), `tailwind.config.ts` 없음**(현재 web/admin 상태와 일치). 토큰은 CSS 변수로 표현. **hex 그대로 사용 가능**(v4는 bare `var()` 매핑 — v3의 `hsl(var(...))` 래퍼 쓰지 말 것).
- **shadcn = 시맨틱 CSS 변수 규약 상속 + 델타 오버라이드**(DESIGN "shadcn 기본 정본 상속, 브랜드 델타만"). 토큰 키를 shadcn 규약(`--primary`·`--background`·`--ring`…)으로 두면 후속 `shadcn add`가 자동으로 만다린 토큰을 입는다. `baseColor:"stone"`(웜 뉴트럴, 크림/브라운 팔레트에 최근접).
- **RN은 NativeWind 없음 → StyleSheet + 토큰 객체.** Tailwind 클래스는 RN에서 동작 안 함. RN은 `theme.ts`(토큰 매핑) + `StyleSheet`로 소비. (NativeWind 도입은 **범위 밖**·큰 추가 — 하지 않는다.)
- **금지 안티패턴(architecture L292-296, DESIGN Do/Don't):** ① **색만으로 상태 표현 금지**(색+아이콘/텍스트 병행) — 본 스토리는 토큰만 정의하므로 소비 컴포넌트(핀·배지·슬롯·배너)가 지킬 규약을 **토큰 주석/문서로 전달**(예: `pinAvailable`은 반드시 체크 아이콘 동반). ② 프라이머리 만다린을 상태 의미색으로 쓰지 말 것(의미색은 `success`/`pinFull` 별도). ③ 위계 장식용 그림자 금지(기본 면=flat, 그림자는 sheet/dialog/toast/fab만).

### 함정 & 디스아스터 방지 (반드시 읽을 것)

1. **packages/ui에 React/RN을 넣지 말 것.** 토큰은 순수 데이터. 컴포넌트를 넣으면 web↔RN import가 깨진다.
2. **첫 cross-package import 인프라 누락 = 빌드 실패.** web/admin `transpilePackages:['@desknow/ui']` + mobile `metro.config.js`. 없으면 "Module not found"/트랜스파일 에러.
3. **Tailwind v4 hex는 bare-var.** `--color-primary: var(--primary)` (O) / `hsl(var(--primary))`(v3, X).
4. **다크 값 작성 금지(AC2).** web/admin globals.css의 기존 `@media (prefers-color-scheme: dark)` **제거**. RN `DarkTheme` 분기 **제거**. parity 테스트가 다크 부재를 단언.
5. **RN 가변폰트 불가.** Pretendard는 **정적 weight otf**로. config plugin은 **dev build 전용**(Expo Go에선 안 보임 → 기기검증 E7).
6. **reduced-motion 무차별 차단 금지.** `* { transition: none !important }`는 ≤100ms 기능 피드백까지 죽인다(AC3 위반). 장식 애니메이션만 게이트.
7. **shadcn `tailwindcss-animate`(v3) 금지** → `tw-animate-css`(v4). `baseColor` web/admin 동일(`stone`).
8. **전역 레이어는 자리만.** 챗봇/배너 기능 구현 = 스코프 크리프(E7·E5). FAB onPress=no-op.
9. **mobile demo 잔재(explore 탭·generic theme) 정리.** 3탭(찾기/예약현황/즐겨찾기)로 정렬, 안 쓰는 데모 컴포넌트 제거(린트 미사용 경고 방지).
