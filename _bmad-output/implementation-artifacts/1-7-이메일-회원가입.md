## Story

As a 예약자 또는 제공자,
I want 이메일·비밀번호와 역할(booker/provider)을 지정해 가입하길,
so that 내 역할로 즉시 로그인 가능한(=1.8 로그인이 수락하는 상태의) 계정을 갖는다 (FR-1, FR-3, NFR-6, UJ-1·2·3).

## Acceptance Criteria

1. **(AC1 — 가입 성공 & Argon2 해싱)** 유효한 이메일·비밀번호·역할(`booker`/`provider`)로 가입하면, `users` 테이블에 계정이 생성되고(**비밀번호는 Argon2 해싱 저장, 평문 저장 금지** — NFR-6), 해당 역할로 **즉시 로그인 가능한 상태**(활성 계정 + 검증 가능한 해시)가 된다. 응답은 **201 Created**이며 본문은 사용자 리소스(`id`·`email`·`role`·`is_active`·`created_at`)이고 **`password_hash`를 절대 포함하지 않는다**.
2. **(AC2 — 이메일 형식 검증)** 이메일 형식이 유효하지 않으면 가입이 거부되고(**422**) 표준 에러 스키마로 안내된다. 검증은 백엔드(신뢰 경계, Pydantic)가 최종 수행한다(architecture.md L277).
3. **(AC3 — 이메일 중복 차단)** 이미 가입된 이메일로(**역할과 무관하게**) 가입을 시도하면 **409 `EMAIL_TAKEN`** 으로 거부된다. 중복은 **이메일 단위**로 차단되며, `users.email`의 **DB UNIQUE 제약(`uq_users_email`)이 진실의 원천**이다(서비스 선검사 + 경합 시 IntegrityError 변환 = defense-in-depth).
4. **(AC4 — 비밀번호 정책, FR-3)** 비밀번호가 **최소 8자 + 대문자1 + 숫자1 + 특수문자1**을 만족하지 않으면 **백엔드(신뢰 경계)에서 422**로 거부된다(프론트 검증은 UX 보조일 뿐 최종 강제 아님).

> **이 스토리의 범위 결정(명시 — 매우 중요):**
> - **1.7은 백엔드 *가입 엔드포인트*(`POST /api/v1/auth/register`)와 `users` 테이블만 구현한다.** AC1~4는 전부 백엔드 동작(테이블 생성·Argon2 해싱·409/422 응답)이며, AC4가 "프론트 검증은 UX 보조"로 명시한다.
> - **프론트 가입 폼(web/admin/mobile)은 본 스토리 범위가 아니다.** 아키텍처는 "프론트는 `packages/api-client` 생성 SDK로만 백엔드 호출, 직접 fetch 금지"(L290)를 강제하는데 그 **SDK는 Story 1.9에서 생성**된다. 또한 UX 문서(EXPERIENCE.md L53)는 가입을 "전역 헤더/메뉴" 진입으로만 다루고 상세 화면 사양이 없다. → 프론트 가입/로그인 UI는 **SDK(1.9) 이후**의 후속 작업으로 둔다. (말미 "질문/확인" 참조.)
> - **JWT 발급·`refresh_tokens`·로그인·RBAC는 1.7 범위가 아니다(=1.8).** AC1의 "즉시 로그인 가능"은 *토큰 발급*이 아니라 *1.8 로그인이 수락할 수 있는 계정 상태*(활성 + 검증 가능한 해시)를 의미한다.

- [x] **Task 5 — `app/auth/service.py` 작성: `register_user` (AC: 1, 3)**
  - [x] **신규.** 도메인 로직(이메일 정규화·중복 검사·해싱·삽입)을 라우터에서 분리한다(아키텍처 §Boundaries L349 — service 계층).
    ```python
    from sqlalchemy.exc import IntegrityError
    from sqlmodel import Session, select
    from app.auth.models import User
    from app.auth.schemas import RegisterRequest
    from app.core.errors import DomainError, ErrorCode
    from app.core.security import hash_password

    def register_user(session: Session, data: RegisterRequest) -> User:
        email = data.email.strip().lower()  # 대소문자 무관 중복 차단 → 소문자 정규화 저장
        existing = session.exec(select(User).where(User.email == email)).first()
        if existing is not None:
            raise DomainError(ErrorCode.EMAIL_TAKEN, "이미 가입된 이메일입니다.")  # 409
        user = User(email=email, password_hash=hash_password(data.password), role=data.role)
        session.add(user)
        try:
            session.commit()
        except IntegrityError as exc:  # 경합: 선검사 통과 후 동시 삽입 → UNIQUE 위반
            session.rollback()
            raise DomainError(ErrorCode.EMAIL_TAKEN, "이미 가입된 이메일입니다.") from exc
        session.refresh(user)
        return user
    ```

  - [x] **이메일 소문자 정규화** 저장으로 대소문자만 다른 중복가입을 차단한다. (선검사 + UNIQUE 제약 이중 방어 — 선검사는 친절한 메시지, UNIQUE는 경합 안전망.)

### 정확한 인증/보안 규약 (AC1 — 그대로 적용)

- **해싱 = pwdlib + Argon2**(architecture.md L157). `PasswordHash.recommended()`가 Argon2(기본 파라미터)를 쓴다(`pwdlib[argon2]` 설치 확인). **bcrypt/passlib/직접 hashlib 금지.**
- **pwdlib 시그니처 함정:** `hash(password)` / **`verify(password, hash)`**. pwdlib `__init__` docstring 예제가 `verify(hash, "password")`로 적혀 있으나 **실제 시그니처는 `verify(self, password, hash)`**(소스 `_hash.py:59` 확인) — 순서를 뒤집으면 항상 False가 되어 1.8 로그인이 조용히 깨진다.
- **Argon2는 평문 길이 제한 없음:** bcrypt의 72바이트 절단 함정이 없으므로 비밀번호 최대 길이 캡/사전 해시가 불필요하다.
- **평문 저장 절대 금지(NFR-6):** 컬럼명은 `password_hash`. 응답 모델(`UserPublic`)에 `password_hash` 필드를 두지 않아 직렬화로도 새지 않게 한다.

### 정확한 데이터/네이밍 규약 (AC1/AC3 — 1.4 상속)

- **테이블 `users`**(복수 snake_case). PK `pk_users`(uuid). **UNIQUE `uq_users_email`**(1.4 네이밍 규약 `uq_%(table_name)s_%(column_0_N_name)s`가 자동 생성 — 손으로 제약명 짓지 않는다).
- **`created_at`**: `*_at` = **UTC `timestamptz`**(architecture.md L234). `Column(DateTime(timezone=True))` + `default_factory=now_utc`(1.5 — `datetime.now()` 직접 호출 금지).
- **`id`**: `uuid.UUID`(SQLModel→`sa.Uuid()`→PG 네이티브 UUID). `default_factory=uuid.uuid4`(Python측 생성 → 삽입 전 id 확정, 테스트 refresh 불필요에 유리).
- **와이어 snake_case 전 구간 유지**(architecture.md L286): 요청/응답 JSON 필드는 `password_hash`/`is_active`/`created_at` 등 snake_case 그대로(camelCase 변환 레이어 금지).

### 정확한 에러/검증 규약 (AC2/AC3/AC4 — 1.5 상속)

- **409 중복:** `DomainError(ErrorCode.EMAIL_TAKEN, "...")` → 1.5 `domain_error_handler`가 `{detail:{code:"EMAIL_TAKEN", message}}`로 응답(409는 `DEFAULT_STATUS` 레지스트리 기본). `ErrorCode.EMAIL_TAKEN`은 **이미 시드됨** — 추가 정의 불필요.
- **422 검증:** 이메일 형식·비번 정책·역할 위반은 **Pydantic 검증** → `RequestValidationError` → 1.5 `validation_exception_handler`가 `{detail:{code:"VALIDATION_ERROR", message}}`로 **자동 단일화**. → **422용 별도 핸들러/코드 작성 금지**(이미 횡단 인프라가 처리).
- **상태코드:** 생성=**201**(architecture.md L241 "201(생성)"). 중복=409, 검증=422. (AC1이 코드를 명시하진 않지만 아키텍처가 201을 규정.)
- **검증 시점:** 백엔드가 신뢰 경계(architecture.md L277 "비번 복잡도·역할·소유권 백엔드 최종"). 프론트 검증은 UX 보조일 뿐.

### 흔한 실수 방지 (anti-patterns)

- ❌ 평문 비밀번호 저장 / 컬럼명 `password` → `password_hash` + Argon2(NFR-6).
- ❌ bcrypt/passlib/직접 hashlib → pwdlib `PasswordHash.recommended()`(아키텍처 지정).
- ❌ `verify(hash, password)` 순서(docstring 예제 오기) → **`verify(password, hash)`**.
- ❌ 응답에 `password_hash` 노출 → `response_model=UserPublic`(해시 필드 없음).
- ❌ `EMAIL_TAKEN`/`"EMAIL_TAKEN"` 문자열 하드코딩·raw `HTTPException(409)` → `DomainError(ErrorCode.EMAIL_TAKEN)`(1.5).
- ❌ 비번 정책/이메일 검증을 직접 422 핸들러로 작성 → Pydantic 검증이 1.5 핸들러로 자동 422 변환.
- ❌ `role="admin"` 가입 허용 → `Literal["booker","provider"]`(admin 시드 전용).
- ❌ 대소문자만 다른 이메일 중복 허용 → `.strip().lower()` 정규화 저장.
- ❌ 선검사만 의존(경합 시 중복 삽입) → UNIQUE 제약 + `IntegrityError`→`EMAIL_TAKEN` 변환(이중 방어).
- ❌ `SQLModel.metadata.create_all()` / 다른 도메인 테이블 동시 생성 → Alembic `users` 단일 마이그레이션(1.4 규약).
- ❌ `datetime.now()`(naive·로컬 tz)로 `created_at` → `now_utc()`(1.5 단일 출처).
- ❌ 응답 `created_at`을 `+00:00`로 → `isoformat_utc()`(`...Z`, 1.5).
- ❌ auth 라우터를 `app`에 직접 등록(`/api/v1` 우회) → `api_router.include_router(auth_router)`.
- ❌ `tests/test_main.py`를 `with TestClient(app)`로 변경 → lifespan DB 검증 트리거(1.4 회귀). 모듈 레벨 유지.
- ❌ `dependency_overrides` 정리 누락 → `finally: app.dependency_overrides.clear()`(테스트 오염).
- ❌ EmailStr 사용 전 `email-validator` 미설치 → Task 1로 `pydantic[email]` 추가(ImportError 차단).
- ❌ env.py 허브에 `app.users` import(주석 예시 오기) → 실제는 `app.auth`.
