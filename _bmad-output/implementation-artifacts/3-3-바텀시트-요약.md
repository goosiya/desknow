# Story 3.3: 바텀시트 요약

Status: done

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a 예약자,
I want 핀(또는 목록 항목)을 탭하면 떠오르는 바텀시트에서 가격·영업시간·예약 가능 여부를 스크롤 없이 한눈에 보고, 끌어올려 부대시설까지 확인하길,
so that 상세로 들어가기 전에 후보를 빠르게 비교한다 (FR-6, UJ-1·2, UX-DR3, NFR-5).

> **이 스토리는 3.2가 만든 "최소 바텀시트 셸"(이름 + 상태 배지 + "상세 정보는 곧" 자리)을 실제 콘텐츠로 채운다.** 3.2는 핀 탭 → 시트 열림 동작만 만들었고, 1차/2차 정보 위계·신선 단일 조회·즐겨찾기 어포던스·그래버/드래그·포커스 트랩·Esc는 **본 스토리가 채울 자리**라고 `RoomSheet.tsx`에 명시해 두었다.

## 범위 결정 (KTH 확정 2026-06-15 — create-story)

3.2가 `RoomMapItem`을 **좌표·이름 4필드만**(`{room_id, name, lat, lng}`)으로 의도적으로 한정했고(과조회·`provider_id` 노출 방지), 바텀시트가 요구하는 **가격·영업시간·부대시설·신선 예약가능**은 어떤 엔드포인트도 공급하지 않는다(`GET /{room_id}`는 아직 없음 — `PATCH /{room_id}`만 존재). 이 데이터 갭과 인접 스토리 경계(즐겨찾기=3.7·상세 페이지=4.2)를 다음과 같이 확정한다:

1. **데이터 소스 = 신규 공개 `GET /api/v1/rooms/{room_id}`(본 스토리 소유, 단일 룸 신선 조회).** 3.2가 좌표 공급용 `GET /rooms`를 수직 슬라이스로 추가한 선례를 따라, 본 스토리가 단일 룸 요약 엔드포인트를 추가한다. 응답 = 요약 필드(가격·수용·룸 형태·부대시설·영업시간) + **`derive_slots`(2.1) 재사용한 오늘 신선 `remaining_slots`**. 이로써 시트의 "예약 가능 여부"가 **탭 시점 동결 스냅샷이 아니라 신선값**이 된다 → **3.2 code-review deferred(RoomSheet stale 배지) 회수**(deferred-work.md L6). 4.2 "상세 정보 화면"은 이 엔드포인트를 **확장**(후기/평점·예약 UI 추가)하지 본 스토리와 중복하지 않는다. **공개·무인증**(탐색 첫 화면, PRD §FR-2 — `/rooms`·`/availability`와 동일 근거)·**`provider_id` 미노출**(공개 엔드포인트).

2. **즐겨찾기(하트) = 어포던스만, 배선은 3.7.** 즐겨찾기 인프라(`favorites` 테이블·POST/DELETE 엔드포인트·옵티미스틱 토글·기기/계정 영속·전 표면 동기화)는 **Story 3.7이 명시 소유**("바텀시트·상세·목록 어디서든 하트를 토글하면…")하며 3.3보다 **나중**이다. 따라서 본 스토리는 **하트 버튼 UI(아이콘·`aria-label`·터치타겟 ≥44px)와 자리만** 두고, 실제 토글·영속은 3.7이 배선한다. 1.7~2.3 프론트 폼을 전용 후속 스토리로 이관한 스코프 규율과 일관.

3. **상세 이동 = Link(`/rooms/[room_id]`) + 최소 스텁 라우트.** 실제 상세 페이지는 **Story 4.2 소유**이고 `/rooms/[room_id]` 라우트는 아직 없다. 본 스토리는 시트에 상세 이동 Link를 두고, **404(막다른 화면 NFR-5 위반) 방지용 최소 스텁 페이지**(`app/rooms/[room_id]/page.tsx` — "상세 화면은 곧 제공돼요" placeholder)를 만든다. 4.2가 이 스텁을 실제 상세(후기·예약 UI)로 채운다. 3.2가 `/reservations`·`/favorites` placeholder 라우트를 만든 선례와 일관.

4. **바텀시트 구현 = `vaul` 드로어.** 그래버/드래그 펼침·접힘(AC2)과 포커스 트랩·Esc·포커스 복귀·스크롤 잠금·reduced-motion(NFR-5·UX-DR12)을 **검증된 구현으로 한 번에** 충족한다. `vaul`은 Radix Dialog 위에 만든 React 바텀시트 표준이자 shadcn `Drawer`의 기반이라 a11y는 Radix에서 상속하고 드래그/스냅은 vaul이 제공한다. **`snapPoints=[0.4, 1]`이 "접힘(1차만)/펼침(2차까지)"에 직대응**한다. 신규 런타임 의존성 1개(`vaul`). 토큰(`elevation.sheet`·`rounded.xl`·`card`)만 입혀 스타일링한다.

## Acceptance Criteria

**AC1 — 핀/목록 항목 탭 시 시트가 열리고, 1차 정보가 스크롤 없이·2차가 그 아래 위계로 보인다**
**Given** 핀(3.2) 또는 목록 항목(3.4가 같은 시트 재사용) 탭
**When** 바텀시트가 열리면
**Then** **1차 정보(가격 · 오늘 영업시간 · 예약 가능 여부 배지)가 접힘 스냅(스크롤 없이 한 화면)에 표시**되고, **2차 정보(부대시설 · 수용 인원 · 룸 형태)가 그 아래 하위 위계**로 배치된다. 1차는 시각적으로 우선(큰 가격·상태 배지), 2차는 라벨+태그 형태다. *(주소·거리는 데이터 미보유 → Dev Notes "데이터 현실 갭" 참조: 본 스토리 비포함.)*

**AC2 — 상세 이동 · 즐겨찾기 어포던스 · 그래버/드래그 펼침·접힘이 동작한다**
**Given** 열린 바텀시트
**When** 사용자가 조작하면
**Then** ① **상세 보기 Link**가 `/rooms/{room_id}`로 이동하고(스텁 라우트 — 4.2가 실제 상세로 채움), ② **즐겨찾기 하트 어포던스**(아이콘 + `aria-label="즐겨찾기"` + 터치타겟 ≥44px)가 존재하며(실제 토글·영속은 Story 3.7이 배선 — 본 스토리는 자리만), ③ **그래버(handle)/드래그로 펼침·접힘**(`vaul` 스냅 `[0.4, 1]`)이 동작하고 드래그로 내려 닫을 수 있다. 데스크톱 마우스에서도 그래버 드래그·탭이 동작한다.

**AC3 — 시트는 떠오르는 유일 면이며 포커스 진입·Esc·포커스 트랩이 적용된다 (NFR-5)**
**Given** 바텀시트
**When** 열고 닫으면
**Then** 시트는 **지도 위로 떠오르는 유일 면**(`elevation.sheet` 그림자 · **상단만** `rounded.xl` · `bg-card`)이고, **열릴 때 포커스가 시트 안으로 진입**하며, **Esc로 닫히고**(또는 드래그-다운/닫기 버튼/오버레이 클릭), **포커스 트랩**이 적용돼 Tab이 시트 밖으로 새지 않는다. 닫으면 포커스가 합리적으로 복귀한다. 위계 장식용 그림자 금지(시트/오버레이만 그림자).

**AC4 — 시트의 "예약 가능 여부"는 신선 단일 조회값이다 (3.2 stale 스냅샷 회수)**
**Given** 신규 `GET /api/v1/rooms/{room_id}`(공개)
**When** 시트가 열리면
**Then** 가격·영업시간·부대시설·수용·룸형태와 **오늘 신선 `remaining_slots`**를 단일 룸 조회로 가져오고, **예약 가능 배지(✓ 예약 가능 / ✕ 마감)는 이 신선값**(`remaining_slots >= 1`)으로 결정된다 — 핀 탭 시점의 동결 스냅샷이 아니다. **색 단독 금지**(배지는 색 + 아이콘 + 텍스트 3중 신호). 엔드포인트는 **`provider_id`·`is_active` 등 내부 필드를 노출하지 않는다**(공개·과조회 방지). 미존재/비활성 룸은 404 `ROOM_NOT_FOUND`.

**AC5 — 시트 콘텐츠 로딩/에러가 막다른 화면 없이 처리되고 reduced-motion을 존중한다**
**Given** 시트가 열린 상태
**When** ① 요약 조회 로딩 중 / ② 조회 실패(비-2xx·네트워크) 상황이면
**Then** ① 시트 헤더(이름 — 핀/항목에서 즉시)는 바로 보이고 1차/2차 자리는 **스켈레톤**, ② 실패 시 **"정보를 못 불러왔어요" + 다시 시도**(막다른 화면 금지, 3.2 AC5 정신 계승)가 표시된다. `prefers-reduced-motion` 시 떠오름/스냅 모션이 축소된다(`vaul` 자동 + 토큰). 모든 마이크로카피는 친근한 해요체.

### ★데이터 현실 갭 — 주소·거리는 모델에 없다 (만들지 말 것)

### 슬롯·시간 anti-pattern — 서버가 1회 계산, 클라는 날짜 포맷만 (architecture L293·L362)

- **`remaining_slots`는 서버가 `derive_slots`로 1회 계산**(`get_room_summary`). 클라는 `>= 1` 자명 분기(`summaryStatus`)만 — 슬롯 재계산 0.
- `todayBusinessHours`는 **KST 오늘 요일을 고르는 날짜 포맷**일 뿐(슬롯 도출 아님). `business_hours`를 슬롯으로 환산하는 코드를 **절대 작성하지 않는다**.
- `>= current` 직접 인스턴트 비교(`aggregate_availability` 동일) — `is_within_hours`/`hours_until`(1.5 float 경계 defer) 회피. [Source: service.py aggregate_availability L207-210 / architecture L293·L362]
