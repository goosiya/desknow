# Story 3.4: 행정동 콤보 목록 조회

Status: done

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a 예약자,
I want 시/군/구·동/읍/면 콤보로 지역을 골라 그 지역의 스터디룸을 가격·시설·예약 가능 여부와 함께 목록으로 보고, 마음에 드는 곳을 탭해 상세로 들어가길,
so that 특정 동네 전체를 훑어 모임 장소를 비교한다 (FR-8, UJ-1, UX-DR8, NFR-5).

> **이 스토리는 "스터디룸 찾기"에 지도 외의 두 번째 탐색 도구(목록)를 추가한다.** 3.2가 만든 지도(`MapView`)와 나란히 **지도/목록 토글**을 두고, 목록 모드에서 **행정동 콤보**(시/군/구 → 동/읍/면)로 지역을 좁혀 룸을 나열한다. 목록 항목 탭은 3.3이 **목록 재사용을 의도해 지도-비결합으로 설계한** `RoomSheet`를 그대로 연다(신규 시트 없음).

## 범위 결정 (KTH 확정 2026-06-15 — create-story)

> **※ 용어 주의 — 법정동 vs 행정동 (KTH 확정 2026-06-15):** 기획 문서(PRD·epics·UX·FR-8·본 스토리 제목)는 "**행정동** 콤보"로 표기했으나, **실제 구현 기준은 법정동(法定洞)이다.** 2.2가 카카오 `address.b_code`(=법정동 코드)를 `admin_dong_code`에 저장했고, 법정동이 지번 주소 기준으로 더 안정적·표준이라 그대로 채택한다(행정동 전환=2.2/모델/마이그레이션/백필 필요 → 비채택). 따라서 "행정동"은 "동 단위 지역 탐색"의 **느슨한 라벨**일 뿐이며, **데이터·코드·번들 참조 데이터셋 기준은 모두 법정동**이다(번들 = **법정동코드 전체자료**, 행정동코드 아님 — 코드↔데이터셋 정합 필수). 사용자 노출 라벨은 "시/군/구·동/읍/면"으로 중립이라 사용자는 법정/행정 구분을 보지 않는다. 필드명 `admin_dong_code`는 법정동을 담은 misnomer이나 리네임=마이그레이션이라 defer.

핵심 데이터 갭: `Room.admin_dong_code`에는 **법정동 b_code(10자리 숫자코드)만** 저장되고(2.2 `GeocodeResult.admin_dong_code` = 카카오 `b_code`), **사람이 읽는 지역명(시/군/구·동/읍/면)은 어디에도 저장되지 않는다**(geocode가 준 `address_name`은 2.2에서 `_NON_ROOM_COLUMN_FIELDS`로 silent-drop). 지역명 참조 데이터·시드도 전무하다. 이 갭(3.3의 "주소·거리 데이터 갭"과 동형)과 인접 스토리 경계를 다음과 같이 확정한다:

1. **지역명 출처 = 백엔드 번들 법정동 코드→이름 정적 참조 (KTH 확정 — 옵션 A).** rooms 모델·2.2 등록·마이그레이션을 **건드리지 않고**, 백엔드에 **법정동 코드→이름 정적 참조**(시도·시군구·읍면동 3레벨)를 번들로 둔다. 신규 콤보/목록 엔드포인트가 **보유 룸의 b_code를 이 참조에 조인**해 콤보 라벨·필터 목록을 구성한다. 모델에 지역명 컬럼을 추가하고 백필하는 옵션 B(마이그레이션·2.2·라이브 DB 백필 필요)는 채택하지 않는다. **마이그레이션 0·모델 무변경·2.2 무변경·3.4 자기완결**(3.3 "보유 데이터로만 구성" 규율과 일관).

2. **목록 행 데이터 = 신규 지역필터 목록 엔드포인트 (KTH 확정 — 옵션 A).** UJ-1이 목록에서 **가격·시설 비교**를 요구하나 기존 엔드포인트는 행 단위 가격을 주지 않는다(`list_active_rooms`=좌표·이름 4필드, `/availability`=집계값만). 따라서 **이름·가격·룸 형태·부대시설·신선 `remaining_slots`**를 담은 리스트 항목 스키마(`RoomListItem`)를 **지역 필터로 반환하는 신규 엔드포인트**를 추가한다. 행 탭 → 기존 `RoomSheet` 재사용(시트는 자체 신선 단일 조회). 이로써 **deferred-work.md L5(공개 룸 스캔 — "행정동 필터=3.4")를 부분 회수**한다(bbox/반경=3.5·레이트리밋=배포는 잔존).

3. **즐겨찾기(하트) = 어포던스만, 배선은 3.7.** 목록 행/시트의 하트는 아이콘·`aria-label`·터치타겟만(3.3과 동일). 실제 토글·영속·전 표면 동기화는 Story 3.7.

4. **상세 이동 = 시트의 상세 보기 Link(`/rooms/[room_id]` 스텁, 3.3이 생성·4.2가 채움).** 목록 행 탭 → 시트 → 상세 Link. 본 스토리는 신규 라우트를 만들지 않는다(스텁 이미 존재).

5. **지도/목록 토글 = 단일 뷰 전환 + 선택 지역 보존.** "같은 탐색 도구 안에서 지도↔목록 전환, 선택 상태 보존"(EXPERIENCE L123)을 토글(한 번에 한 뷰)로 충족한다. **`lg` 2열(지도+사이드 목록) 반응형**(EXPERIENCE L181·deferred-work L76)은 본 스토리 핵심 AC가 아니므로 **선택적 폴리시/후속**으로 둔다(토글 단일 뷰가 기능 충족). **반경↔행정동 전환=3.5·권한거부→목록 우회=3.6**가 이 컨테이너를 확장한다.

## Acceptance Criteria

**AC1 — 시/군/구 선택 시 하위 동/읍/면 콤보가 갱신되고, 선택 지역의 룸이 목록으로 나열된다**
**Given** 목록 모드의 행정동 콤보(시/군/구 + 동/읍/면)
**When** 시/군/구를 선택하면
**Then** ① **하위 동/읍/면 콤보가 그 시/군/구에 속한(룸이 있는) 동/읍/면으로 갱신**되고, ② 선택 지역(시/군/구만 또는 동/읍/면까지)의 룸이 **목록(룸 카드/리스트 로우)으로 나열**된다. 각 행은 **이름 · 가격(`8,000원/시간`) · 예약 가능 배지 · 부대시설 태그 · 룸 형태**를 보여(UJ-1 비교), 가용성 배지는 **신선 `remaining_slots`** 기준(색+아이콘+텍스트 3중 신호 — 색 단독 금지). 콤보는 **룸이 있는 지역만** 제시한다(빈 지역 미노출).

**AC2 — 목록 항목 조작 시 상세 이동·즐겨찾기 토글이 가능하다**
**Given** 목록 행
**When** 행(또는 행 안의 컨트롤)을 조작하면
**Then** ① 행 탭 → **3.3 `RoomSheet`가 열린다**(지도-비결합 재사용 — `roomId`+`name`+`fallbackStatus`로 호출). 시트가 상세 보기 Link(`/rooms/{room_id}` 스텁)와 즐겨찾기 어포던스를 제공한다. ② 즐겨찾기 하트 어포던스(아이콘 + `aria-label="즐겨찾기"` + 터치타겟 ≥44px)가 **행에도** 존재한다(실제 토글·영속은 Story 3.7 — 본 스토리는 자리만, dead 토글 금지). 아이콘 단독 컨트롤(하트·지도/목록 토글)은 모두 접근 이름을 가진다(NFR-5).

**AC3 — 선택 지역에 룸이 없으면 막다른 화면 없이 빈 상태가 제안과 함께 표시된다**
**Given** 선택 지역(또는 룸이 전혀 없는 초기 상태)
**When** 조회 결과가 0건이면
**Then** **"이 지역엔 등록된 곳이 없어요."** 빈 상태가 **인접 동/반경 확대 제안**(예: "다른 동네를 골라보거나, 지도에서 주변을 넓혀볼 수 있어요.")과 함께 표시된다 — 막다른 화면 금지(NFR-5). 카피는 친근한 해요체.

**AC4 — 목록은 신선 가용성으로 표시되고, 공개 엔드포인트는 내부 필드를 노출하지 않는다**
**Given** 신규 지역필터 목록 엔드포인트(공개)
**When** 목록을 조회하면
**Then** 각 행의 `remaining_slots`는 **서버가 `derive_slots`(2.1) 재사용으로 1회 집계한 오늘(ROOM_TZ) 신선값**이고(클라 슬롯 재계산 0 — architecture L362), 예약 가능 배지는 `remaining_slots >= 1`로 도출된다. 엔드포인트는 **`provider_id`·`is_active`·`lat`/`lng`·`admin_dong_code`를 노출하지 않는다**(공개 표면·과조회 방지, `RoomMapItem`/`RoomSummary` 선례). 비활성 룸은 목록·콤보 모두에서 제외된다.

**AC5 — 지도/목록 토글로 전환되며 선택 지역이 보존되고, 로딩/에러가 막다른 화면 없이 처리된다**
**Given** 스터디룸 찾기 화면
**When** 지도↔목록을 토글하면
**Then** ① **같은 화면 안에서 뷰가 전환**되고(`지도`|`목록` 토글, `aria-label`·`aria-pressed`/탭 시맨틱), **선택한 지역 상태가 보존**된다(목록→지도→목록 시 콤보 선택 유지). ② 콤보/목록 로딩 중 **shadcn Skeleton 4–6행**, ③ 조회 실패 시 **"목록을 못 불러왔어요." + 다시 시도**(막다른 화면 금지). `prefers-reduced-motion` 존중(전역 토큰 상속).
