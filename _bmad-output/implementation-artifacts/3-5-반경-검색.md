# Story 3.5: 반경 검색

Status: done

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a 예약자,
I want 현재 위치를 기준으로 반경(기본 3km, 조정 가능) 내 스터디룸만 가까운 순으로 목록에서 보고, 행정동 검색과 같은 도구 안에서 반경↔행정동을 오가며(선택 상태 보존),
so that 지금 갈 수 있는 거리의 빈 방을 빠르게 좁힌다 (FR-9, UJ-2, NFR-5).

> **이 스토리는 3.4가 만든 "스터디룸 찾기" 목록 도구에 두 번째 검색 방식(반경)을 추가한다.** 3.4의 `ExploreView`(지도/목록 토글)·`RoomList`·`GET /rooms/search`·신선 슬롯 집계·`RoomSheet` 재사용 패턴을 **그대로 확장**한다 — 신규 시트/상세 라우트·신규 모델·마이그레이션은 없다. 핵심 신규는 ① 백엔드 Haversine 거리 필터(`/rooms/search`에 `lat·lng·radius_km` 추가) ② 프론트 목록 모드 안의 `행정동|반경` 검색방식 서브토글 + 현위치 획득 + 반경 조정 컨트롤이다.

## 범위 결정 (KTH 확정 2026-06-16 — create-story)

1. **공간 필터 = Haversine(Python).** 좌표(`lat`/`lng`)는 일반 float 컬럼이고 PostGIS는 미설치다. 아키텍처 L151-152가 "반경 = PostGIS ST_DWithin 또는 Haversine — **MVP 규모는 후자도 충분**"을 명시했고, 3.4 지역 필터도 Python 수행이라 일관된다. **순수 함수 `haversine_km`로 거리 계산 후 Python에서 필터·정렬**한다. **마이그레이션 0·PostGIS 도입 0·신규 백엔드 의존성 0**(거리 계산은 stdlib `math`). SQL 공간쿼리/PostGIS/공간 인덱스 최적화는 3.4 region Python 필터와 동일 계열의 **deferred**(데이터 증가 시점 — bbox 계열).

2. **엔드포인트 = 기존 `GET /rooms/search` 확장.** 3.4가 추가한 `/rooms/search`에 **선택 쿼리 파라미터 `lat`·`lng`·`radius_km`를 추가**한다(신규 엔드포인트 아님). `region_code`와 같은 공개·읽기전용 표면이며, SDK는 파라미터만 증가(최소 churn). 신규 `/rooms/nearby` 엔드포인트는 비채택(라우팅 순서 가드·operation·SDK 함수 추가 부담).

3. **UI = 목록 모드 안 `행정동|반경` 검색방식 서브토글.** EXPERIENCE L49("목록 도구로 행정동 콤보·반경"), L123("같은 탐색 도구 안에서 …, **반경↔행정동 전환. 상태(선택 지역·반경) 보존**")에 따라, `ExploreView`의 **목록 모드** 안에 검색방식 토글(`행정동` | `반경`)을 둔다. 두 방식 모두 `RoomList`로 결과를 렌더한다. 최상위 뷰 토글은 기존대로 `지도 | 목록` 그대로다(반경은 목록 도구의 검색방식이지 세 번째 최상위 뷰가 아님).

4. **거리 = 가까운 순 정렬만, 거리값 미노출.** 필터를 위해 어차피 거리를 계산하므로 **반경 결과를 거리 오름차순 정렬**한다("지금 갈 수 있는 거리" 취지). 단 `RoomListItem` **스키마는 무변경**(거리 필드 미추가 — SDK 타입 동일). 거리 숫자/도보시간 표시는 3.3에서 defer된 거리 데이터 갭의 일부로 **후속(4.2 상세 계열) deferred**.

5. **위치 권한 처리(AC3) = 본 스토리는 "반경 비활성 + 이유 안내 + 행정동 유도"까지.** 위치 거부/불가 시 반경 검색방식을 비활성하고 이유를 안내하며 행정동 검색으로 유도한다(FR-7 교차). **권한 거부 시 탐색 진입을 자동으로 목록(행정동)으로 우회시키는 전면 처리는 Story 3.6 소유** — 본 스토리는 그 우회가 재사용할 "반경 비활성 + 안내" 조각을 만든다(중복/침범 금지).

6. **반경 조정 컨트롤 = 프리셋 세그먼트(신규 의존성 0).** 반경값은 프리셋(예: 1·2·3·5·10 km, 기본 3km)을 고르는 **접근성 있는 radiogroup 세그먼트 버튼**으로 조정한다 — shadcn Slider(=`radix-ui` Slider 추가) 도입 없이 토큰·기본 컴포넌트로 구현(신규 의존성 0). 연속 슬라이더는 선택적 후속.

## Acceptance Criteria

**AC1 — 위치 좌표가 있으면 반경 내 룸만 결과에 포함되고, 기본 3km이며 사용자가 조정할 수 있다**
**Given** 현위치 좌표(위도·경도)와 목록 모드의 반경 검색방식
**When** 반경 검색을 하면
**Then** ① **반경 내(중심↔룸 Haversine 거리 ≤ 반경) 활성 룸만** 결과에 포함되고(반경 밖 제외), ② **기본 반경은 3km**이며 사용자가 **프리셋(1·2·3·5·10 km 등)으로 조정**할 수 있고(조정 시 결과 갱신), ③ 결과는 **가까운 순(거리 오름차순) 정렬**된다. 각 행은 3.4와 동일하게 **이름 · 가격(`8,000원/시간`) · 예약 가능 배지(신선 `remaining_slots`, 색+아이콘+텍스트 3중 신호) · 부대시설 · 룸 형태**를 보여준다(거리 숫자는 미표시 — 범위 결정 #4).

**AC2 — 반경↔행정동 검색을 같은 도구 안에서 전환하며 선택 상태가 보존된다**
**Given** 목록 모드의 검색방식 토글(`행정동` | `반경`)
**When** 반경↔행정동을 전환하면
**Then** ① **같은 화면(목록 도구) 안에서 검색방식만 전환**되고, ② **양쪽 선택 상태가 보존**된다 — 행정동(시군구/동 콤보 선택)과 반경(반경값) 상태가 전환 왕복(행정동→반경→행정동) 후에도 유지된다. 검색방식 토글은 접근 이름(`aria-label`·`aria-pressed` 또는 radiogroup 시맨틱)을 가진다(NFR-5). 최상위 `지도|목록` 토글의 기존 선택 보존(3.4 AC5)도 회귀 없이 유지된다.

**AC3 — 위치 권한이 없으면 반경 검색은 비활성되고 이유가 안내되며 행정동 검색으로 유도된다 (FR-7 교차)**
**Given** 위치 권한 거부·미허용·미지원(`navigator.geolocation` 부재) 상태
**When** 반경 검색방식에 접근하면
**Then** ① **반경 검색방식이 비활성**되고(또는 선택 시 결과 대신 안내), ② **그 이유가 안내**되며("위치를 못 받았어요. 동네를 골라서 찾아볼게요." 등 친근한 해요체), ③ **행정동 검색으로 유도**된다(전환 버튼/기본 행정동 방식 노출). 어떤 경우에도 **막다른 빈 화면이 없다**(NFR-5). 권한 거부 시 탐색 진입 자체를 목록으로 자동 우회하는 전면 처리는 **Story 3.6 소유**(본 스토리는 반경 비활성+안내 조각까지).

**AC4 — 공개 엔드포인트가 반경 파라미터를 검증·신선 슬롯으로 응답하고 내부 필드를 노출하지 않는다**
**Given** `GET /rooms/search`에 `lat`·`lng`·`radius_km`(선택) 추가
**When** 반경 파라미터로 조회하면
**Then** ① **`lat`/`lng`가 둘 다 있을 때만 반경 필터가 적용**되고(부분 좌표=반경 미적용 — graceful), `radius_km` 미지정 시 **기본 3.0**, ② 범위 밖 파라미터(`lat∉[-90,90]`·`lng∉[-180,180]`·`radius_km≤0` 또는 상한 초과)는 **422**(FastAPI Query 검증→1.5 핸들러, 신규 ErrorCode 0), ③ 각 행 `remaining_slots`는 3.4와 동일하게 **서버가 `derive_slots`(2.1) 재사용으로 1회 집계한 오늘(ROOM_TZ) 신선값**(클라 슬롯 재계산 0 — architecture L362), ④ 응답은 **`provider_id`·`is_active`·`lat`/`lng`·`admin_dong_code`를 노출하지 않고**(3.4 `RoomListItem` 그대로), 비활성 룸은 제외된다. `region_code`와 반경 파라미터가 함께 오면 **둘 다 적용**(교집합)된다.

**AC5 — 반경 결과의 로딩/빈/에러가 막다른 화면 없이 처리된다**
**Given** 반경 검색방식
**When** 로딩/결과 없음/조회 실패 상황이면
**Then** ① 로딩=**shadcn Skeleton 4–6행**(3.4 재사용), ② 빈 결과=**"이 근처엔 아직 없어요."** + **반경 확대/행정동 전환 제안**(EXPERIENCE L133 — "동네를 넓혀볼까요?"), ③ 조회 실패=**"목록을 못 불러왔어요." + 다시 시도**(막다른 화면 금지). `prefers-reduced-motion` 존중(전역 토큰 상속). 카피는 친근한 해요체.
