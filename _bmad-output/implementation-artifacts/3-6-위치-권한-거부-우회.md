# Story 3.6: 위치 권한 거부 우회

Status: done

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a 위치 권한을 거부(또는 미허용·미지원)한 예약자,
I want 탐색에 진입하면 지도 대신 **행정동 콤보 목록**으로 즉시 우회되고 그 이유를 안내받길,
so that 막다른 빈 화면 없이 동네를 골라 탐색을 계속한다 (FR-7, UJ-2, NFR-5).

> **이 스토리는 Epic 3 탐색의 "막다른 화면 금지" 마지막 조각을 닫는다.** 3.2(지도)·3.4(행정동 목록)·3.5(반경)가 만든 `ExploreView`(지도/목록·행정동/반경 토글)·`useGeolocation`(거부/미지원 단일 출처)·`MapView`·`RoomList`/`RegionCombos`를 **재사용·통합**한다. **신규 엔드포인트·모델·시트·상세 라우트·마이그레이션·신규 의존성·SDK 재생성은 없다**(백엔드 무변경 — 행정동/반경 조회는 3.4/3.5에서 완성). 핵심 신규는 **순수 프론트** ① 진입 시 위치 권한을 감지해 **거부/미지원이면 지도 대신 행정동 목록으로 자동 우회**(+안내) ② 위치 신호를 `ExploreView` 단일 소유로 **통합**(`MapView` 인라인 위치 로직 제거·좌표 prop 주입 → 위치 요청 1회)이다.

## 범위 결정 (KTH 확정 2026-06-16 — create-story)

1. **우회 트리거 = 진입 시 위치 1회 요청 → 거부/미지원 콜백에서 자동 전환.** "거부" 판정은 위치를 한 번 조회해야 알 수 있으므로, `ExploreView` 마운트 시 `useGeolocation`을 **즉시 enable**한다. 허용(`granted`)이면 지도 유지(현위치 중심), **거부/미지원(`denied`/`unsupported`)이면 `viewMode`를 `list`·`searchMode`를 `region`으로 자동 스냅** + 안내. Permissions API(`navigator.permissions.query`) 사전 조회는 **비채택**(브라우저 geolocation 지원 실측·첫 방문자 `prompt` 폴백 경로로 복잡도↑ — 단발 요청이 단순하고 3.5 패턴 계승).

2. **위치 로직 = `ExploreView` 단일 소유 + `MapView` 좌표 주입(통합).** 현재 위치 요청이 **2곳**(`MapView` 인라인 `getCurrentPosition` + `useGeolocation`)에 중복된다(3.5가 "MapView 공유화는 선택적 후속"으로 남김). 본 스토리가 자동 우회를 위해 권한 상태를 `ExploreView`가 알아야 하므로, 이 통합을 **지금 수행**한다: `useGeolocation`을 `ExploreView`가 단일 소유 → 허용 시 좌표를 **지도(중심)·반경(center) 양쪽에 공급**, `MapView`의 인라인 위치 로직은 **제거**하고 `coords`/`locationDenied`를 **prop으로 주입**받는다. **위치 요청 1회.** (3.2 AC1 현위치 중심·AC5④ 서울 폴백+배너는 prop 기반으로 **회귀 보존**.)

3. **수동 지도 오버라이드 = 허용, 서울 폴백 배너 회귀 보존.** 자동 우회는 **초기 진입 기본값일 뿐** 지도 토글을 막지 않는다(막다른 화면 금지·사용자 자율). 우회 후 사용자가 `지도`를 누르면 지도로 진입하고, 그 경우 `MapView`는 **서울 시청 폴백 중심 + "현재 위치를 못 받았어요. 동네를 골라서 찾아볼게요." 배너**(3.2 AC5④)를 그대로 표시한다. 자동 전환은 **세션당 1회**(사용자가 이후 수동 토글하면 덮어쓰지 않는다).

4. **안내 카피 = 단일 출처 재사용.** "현재 위치를 못 받았어요. 동네를 골라서 찾아볼게요."(EXPERIENCE L101·L141·L245). 3.5가 이미 반경 모드 거부 시 같은 카피를 카드로 노출하고, `MapView`(3.2)도 동일 배너를 가진다. 본 스토리는 **자동 우회된 행정동 모드 상단에 동일 카피의 안내**를 둔다(중복 문자열은 허용 — 위치별 표시면이 다름; 상수화는 선택적).

5. **백엔드·SDK = 무변경.** 행정동 콤보(`GET /rooms/regions`·`/rooms/search?region_code=`)는 3.4, 반경은 3.5에서 완성. 본 스토리는 **프론트 전용**(`apps/web`) — 백엔드 코드·OpenAPI·SDK 재생성 **없음**. 따라서 1.9 드리프트 게이트(Layer A/B)는 본 스토리 대상 아님(무변경이라 자동 통과).

## Acceptance Criteria

**AC1 — 위치 권한 거부/미허용/미지원 시 탐색 진입이 지도 대신 행정동 목록으로 즉시 우회된다 + 안내** (FR-7, UJ-2)
**Given** 위치 권한 **거부**(`denied`) 또는 **미지원**(`navigator.geolocation` 부재)
**When** 탐색(`/`)에 진입하면
**Then** ① 지도 대신 **행정동 콤보 목록 조회로 즉시 우회**된다(`viewMode="list"`·`searchMode="region"`로 자동 전환 — `RegionCombos` + region `RoomList` 노출), ② **"현재 위치를 못 받았어요. 동네를 골라서 찾아볼게요."** 안내가 행정동 모드 상단에 표시된다(친근한 해요체·`role="status"`), ③ **위치 허용(`granted`)이면 우회하지 않고 지도가 유지**되며 현위치 중심으로 표시된다(3.2 보존). 위치 측정 중(`prompting`)에는 우회를 확정하지 않는다(거부 콜백 도착 시 전환 — 깜빡임 방지).

**AC2 — 위치가 없는 상태에서 반경 검색은 비활성되고 이유가 안내된다 (3.5 AC3 회귀 보존)** (FR-7 교차)
**Given** 위치 권한 거부·미허용·미지원 상태
**When** 목록 모드에서 **반경** 검색방식에 접근하면
**Then** 반경 검색은 **비활성/결과 대신 안내**되고("위치를 못 받았어요…" + "동네(행정동)로 찾기" 유도), 막다른 화면이 없다. **3.5에서 구현된 동작을 회귀 없이 유지**한다(본 스토리의 위치 통합 리팩터가 이 경로를 깨지 않는다).

**AC3 — 어떤 경우에도 막다른 빈 화면 없이 다음 행동이 안내된다** (NFR-5)
**Given** 위치 거부/미지원 · 행정동 미선택 · 빈 결과 · 조회 실패 중 어느 상태든
**When** 탐색 화면에 머무르면
**Then** 막다른 빈 화면이 **없고**, 항상 다음 행동이 안내된다 — 위치 거부=동네 고르기 유도, 행정동 미선택="동네를 골라 주변 스터디룸을 찾아보세요."(3.4), 빈 결과=인접 동/반경 제안(3.4/3.5), 조회 실패=다시 시도(3.4/3.5). 카피는 친근한 해요체(UX-DR10).

**AC4 — 위치 신호가 ExploreView 단일 출처로 통합되어 위치 요청이 1회만 발생한다** (KTH 결정 #2)
**Given** `ExploreView`가 `useGeolocation`을 단일 소유(마운트 시 enable)
**When** 탐색에 진입하면
**Then** ① 위치는 **1회만 요청**되고(`MapView` 인라인 `getCurrentPosition` **제거**), ② 허용 시 좌표가 **지도(중심 이동)·반경(center)** 양쪽에 공급되며, ③ `MapView`는 `coords`(허용 좌표)·`locationDenied`(거부 신호)를 **prop으로 받아** 현위치 중심 또는 **서울 시청 폴백 + 배너**(3.2 AC5④)를 표시한다 — `MapView`는 더 이상 `navigator.geolocation`을 직접 호출하지 않는다. 지도 로드·핀 렌더·재시도·핀→시트(3.2)는 **회귀 없이 유지**된다.

**AC5 — 자동 우회는 세션당 1회 기본값이며 수동 지도 진입을 막지 않는다** (KTH 결정 #3)
**Given** 위치 거부로 행정동 목록에 자동 우회된 상태
**When** 사용자가 최상위 **지도** 토글을 누르면
**Then** ① 지도로 **진입할 수 있고**(자동 우회가 토글을 잠그지 않음), ② 그 경우 `MapView`는 **서울 폴백 중심 + "위치를 못 받았어요…" 배너**를 표시하며(3.2 AC5④ 회귀), ③ 이후 위치 상태 변화로 **다시 자동 우회되지 않는다**(자동 전환은 초기 1회 — 사용자 수동 선택을 덮어쓰지 않는다). 최상위 `지도|목록`·검색방식 `행정동|반경` 토글의 선택 보존(3.4 AC5·3.5 AC2)도 회귀 없이 유지된다.

### 진입 자동 우회 = 세션당 1회, 수동 토글 비파괴 (KTH 결정 #1·#3 / AC1·AC5)

- `geo.status`가 `denied`/`unsupported`로 **확정될 때 초기 1회** `viewMode="list"`+`searchMode="region"`로 스냅. `granted`=지도 유지(현위치 중심), `prompting`/`idle`=대기(우회 미확정 — 깜빡임 방지).
- **`useRef` 가드로 비파괴 보장:** ① 한 번 자동 우회했으면 다시 안 함 ② 사용자가 토글을 직접 눌렀으면(수동 의사) 자동 전환 차단(AC5③). → 거부 후 사용자가 지도를 보겠다고 누른 선택을 위치 상태 변화가 덮어쓰지 않는다.
- **막다른 화면 금지(NFR-5·AC3):** 우회 후 행정동 모드는 미선택 시 "동네를 골라 주변 스터디룸을 찾아보세요."(3.4 RoomList inactive)로 항상 다음 행동 제시. [Source: EXPERIENCE.md L129·L141·L245 / RoomList.tsx L39-47]

### ★set-state-in-effect 함정 — 3.5에서 실제 발생 (필수 회피)

- 3.5 dev에서 `useGeolocation` effect 동기 setState가 `react-hooks/set-state-in-effect` lint에 차단됐고, `queueMicrotask`/콜백으로 옮겨 해결했다([3-5 Debug Log]). 본 스토리의 **`ExploreView` 자동 우회 effect**도 같은 함정 위험 — `geo.status` 변화 반응 effect에서 `setViewMode`/`setSearchMode` 호출이 cascading render로 잡힐 수 있다.
- **대응:** `geo.status`는 이미 비동기 콜백에서 확정된 값이므로 `useEffect(() => {...}, [geo.status])`에서 **조건부 1회** setState는 "상태 변화에 대한 반응"(허용 패턴)이다. 그러나 lint 규칙이 보수적이므로 **빌드/lint로 반드시 실측**하고, 걸리면 ① ref 가드로 1회성 보장 ② 필요 시 이벤트성 처리로 회피. `MapView`의 `setCenter`는 카카오 API 호출이라 setState 아님(안전). [Source: 3-5 Debug Log "lint 함정(react-hooks/set-state-in-effect)" / useGeolocation.ts L36-46]
