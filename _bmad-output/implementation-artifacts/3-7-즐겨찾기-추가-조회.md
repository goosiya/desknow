# Story 3.7: 즐겨찾기 추가 / 조회

Status: done

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a 예약자,
I want 마음에 든 룸을 즐겨찾기에 추가·제거하고 모아보길,
so that 다음에 빠르게 다시 찾는다 (FR-10, UJ-1, UX-DR7).

## Acceptance Criteria

**AC1 — 어디서든 하트 토글(옵티미스틱·영속)**
**Given** `favorites` 테이블과 로그인 사용자
**When** 바텀시트(3.3)·목록(3.4) 어디서든 하트를 토글하면
**Then** 즐겨찾기 상태가 옵티미스틱하게 **≤100ms**로 반영되고(활성=`destructive` 채운 하트 / 비활성=`muted-foreground` 외곽선 하트), 서버 실패 시 이전 상태로 롤백되며, **로그인 사용자별로 영속**된다.

**AC2 — 즐겨찾기 페이지 모아보기**
**Given** `/favorites` 페이지
**When** 진입하면
**Then** 저장한 룸을 모아보고 각 행에서 **상세(`/rooms/[room_id]`)로 이동**할 수 있으며, 빈 상태는 **"마음에 든 곳을 즐겨찾기해두면 여기 모여요."**로 안내된다. 로딩=Skeleton, 조회 실패=재시도(막다른 화면 금지).

**AC3 — 비활성 룸 라벨 + 진입 차단**
**Given** 즐겨찾기한 룸이 비활성(`is_active=false`)인 경우
**When** 즐겨찾기 목록에 표시되면
**Then** **'비활성' 라벨**이 표시되고 **신규 예약 진입(상세 링크)이 차단**된다(막다른 화면 금지 — 라벨로 상태 안내). 본 스토리는 `room.is_active` 필드만 **읽는다**(관리자 비활성 동작=FR-31/E8 — 순방향 의존 아님).

**AC4 — 미로그인 게이팅(로그인 유도)**  *(KTH 확정 2026-06-16)*
**Given** 미로그인(비인증) 상태
**When** 하트를 클릭하면
**Then** 하트는 **표시되되**(외곽선) 토글되지 않고 **"로그인하면 저장돼요"** 안내 + 로그인 링크를 노출한다(막다른 화면 금지). 즐겨찾기 페이지도 미로그인 시 동일하게 로그인 유도 안내를 보인다.

**AC5 — 인증 배선(쿠키 전송·세션 판별)**  *(KTH 확정 2026-06-16 — 본 스토리가 웹 최초 인증 기능)*
**Given** 백엔드 httpOnly 쿠키 인증(1.8 완비) + CORS `allow_credentials=True`
**When** 웹이 인증 필요 SDK를 호출하면
**Then** SDK fetch 클라이언트가 **`credentials: "include"`로 크로스오리진 쿠키를 전송**하고, `GET /api/v1/auth/me`(authMe)로 **로그인 상태를 판별**한다. **전용 로그인 화면/폼은 본 스토리 비포함**(별도 스토리) — 즐겨찾기 배선·게이팅·페이지만 본 스토리 범위.

> **스코프 경계(KTH 확정):** ① **전용 로그인 화면/폼 = 별도 스토리**(본 스토리는 인증 배선 + authMe 세션 판별까지만; 개발/테스트는 기존 토큰으로 쿠키 주입). ② **모바일(`apps/mobile`) 비포함** — 3.2~3.6 일관되게 web 전용, 모바일 즐겨찾기는 E7. ③ **상세(`/rooms/[room_id]`) 하트 = 4.2** — 상세는 현재 3.3 스텁이라 하트 어포던스가 없음. 본 스토리는 `FavoriteButton`을 **공유 컴포넌트**로 추출해 바텀시트·목록 두 진입점을 배선하고, 4.2 상세가 동일 컴포넌트를 재사용하도록 한다("어디서든"의 web 충족 = 시트+목록, 상세는 4.2 graft).

### 인증 배선 배경 (AC4·AC5 — 왜 필요한가)

- 3.2~3.6은 전부 **공개·읽기전용**. 3.7이 **web 최초 인증 필요 기능**이라 인증 배선의 빈틈이 처음 드러난다.

- **빈틈**: 웹 SDK가 `baseUrl`만 설정 → `fetch`가 크로스오리진(localhost:3000→8000) httpOnly 쿠키를 **기본 미전송**. `credentials:"include"` 필요(AC5 B1).
- **빈틈**: 웹에 **로그인 UI 부재**(AppShell "로그인"=placeholder 링크). **KTH 확정: 전용 로그인 화면/폼은 별도 스토리.** 본 스토리는 배선+세션 판별까지만. 개발/E2E 검증은 기존 토큰으로 쿠키를 주입해 수행(또는 후속 로그인 스토리 후 통합).
- **미로그인 게이팅(AC4)**: authMe 401 → 하트는 보이되 클릭 시 "로그인하면 저장돼요"(막다른 화면 금지). 즐겨찾기 페이지도 동일.

### 라이브러리 실측 포인트 (반복함정 프리플라이트)
