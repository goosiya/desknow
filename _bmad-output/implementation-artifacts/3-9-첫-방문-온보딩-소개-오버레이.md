# Story 3.9: 첫 방문 온보딩 소개 오버레이

Status: done

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a 처음 방문한 사용자,
I want 첫 진입에서 서비스를 간략히 소개받고 "지금 시작"을 누르면 다시는 그 안내가 뜨지 않길,
so that 처음엔 무엇을 하는 서비스인지 알고, 재방문부터는 마찰 없이 바로 탐색한다 (PRD "마찰 없는 흐름" 보강 — FR 외 신규, KTH 2026-06-15).

> **이 스토리의 본질 — 프론트 전용·기기 로컬 1회 노출.** 백엔드 API/FR/마이그레이션/SDK 재생성 **0**.
> 진입 화면(탐색=지도) **위에 뜨는 모달 오버레이** 하나 + **"본 적 있음" 클라이언트 로컬 플래그**가 전부다.
> 서버는 "본 적 있음"을 추적하지 않는다(로그인 무관·쿠키 미사용·서버 미전송). 캐시/스토리지 삭제·새 기기·
> 시크릿 모드면 플래그가 없어 다시 1회 노출된다.
>
> **이 스토리는 Epic 3 최초의 모바일(apps/mobile) 실 기능 작업이다(KTH 2026-06-16 확정 — 웹+모바일 동시).**
> 3.2~3.8은 전부 웹 전용이었고 모바일은 E7로 deferred 되어 왔으나, 온보딩은 지도/목록에 의존하지 않는
> **자기완결 오버레이**라 모바일 placeholder 화면 위에서도 성립한다. 두 표면을 **각각 독립 구현**한다
> (저장소·UI 프리미티브가 플랫폼마다 다름 — 코드 공유 불가, 설계 패턴만 공유).

## Acceptance Criteria

epics.md(L698-712) 원문 BDD:

1. **(AC1 — 첫 방문 노출)** **Given** 처음 방문(온보딩 플래그 없음) **When** 탐색 진입 화면을 열면 **Then**
   간략한 서비스 소개와 하단 행동 버튼("지금 시작")이 **진입 화면 위 오버레이**로 표시된다.

2. **(AC2 — 시작 → 영속)** **Given** 온보딩 오버레이 **When** 사용자가 "지금 시작"을 누르면 **Then** 오버레이가
   사라지고 바로 탐색(지도)으로 진입하며, **"본 적 있음" 플래그가 클라이언트 로컬에 영속**된다
   (웹=`localStorage`, 앱=AsyncStorage — 쿠키 대신 서버 미전송 경량 플래그).

3. **(AC3 — 재방문 무노출)** **Given** 플래그가 있는 재방문(같은 기기·캐시 유지) **When** 진입하면 **Then**
   오버레이는 **표시되지 않고** 곧장 탐색으로 진입한다. 캐시/스토리지 삭제·새 기기·시크릿 모드면 플래그가
   없어 다시 1회 노출(로그인 무관·기기 로컬 — 서버 "본 적 있음" 추적 없음).

4. **(AC4 — 접근성·막다른 화면 금지)** **And** 오버레이는 접근성(**포커스 진입·Esc/버튼으로 닫힘·포커스 트랩**)과
   **reduced-motion**을 준수하고(NFR-5·UX-DR12), 막다른 화면을 만들지 않는다(닫으면 항상 탐색으로).

### KTH 확정 사항 (2026-06-16 — create-story 시점)

> AC 자체에 없거나 해석이 갈리는 지점을 사전 확정한다. 이 절은 dev agent에게 **AC와 동등한 구속력**을 갖는다.

- **확정① — 범위 = 웹 + 모바일 동시(KTH 명시 선택).** apps/web 과 apps/mobile **둘 다** 본 스토리에서 구현한다.
  관리자 웹(apps/admin)은 운영자용이라 제외(에픽 명시). 두 표면은 저장소(localStorage vs AsyncStorage)와 UI
  프리미티브(Radix Dialog vs RN Modal)가 달라 **코드 공유 없이 각각 구현**하되, "플래그 읽기 → 미노출이면 표시
  → 닫으면 영속" 동작 패턴은 동일하게 맞춘다.

- **확정② — 오버레이 구현 = 기존 의존성 재사용(웹), AsyncStorage 1건만 추가(모바일).**
  - **웹: 신규 의존성 0.** `radix-ui ^1.5.0`(이미 설치 — 3.4 select 도입분)의 **`Dialog`** 를 쓴다.
    포커스 트랩·Esc·`aria-modal`·포커스 복귀·스크롤 잠금을 **상속**한다(직접 구현 0 — RoomSheet가 vaul로 쓰는
    것과 같은 Radix 계열). 토스트/모달 라이브러리 신규 도입 금지(Epic 3 "신규 의존성 0" 기조).
  - **모바일: AsyncStorage 1건 추가 — 에픽이 명시("AsyncStorage 등가물")한 정당한 추가.**
    `@react-native-async-storage/async-storage`(expo 호환). RN 빌트인 동기 스토리지가 없어 불가피하다
    (expo-secure-store 는 비밀 저장용·과설계). 모바일 오버레이 자체는 RN 빌트인 `Modal`(신규 의존성 0).

- **확정③ — "닫힘"의 정의 = 어떤 방식으로 닫아도 플래그를 영속한다.** AC2는 "지금 시작" 버튼만 명시하나,
  Esc·오버레이 바깥 클릭(웹)·Android 하드웨어 백(모바일)으로 닫아도 **소개를 본 것**이므로 동일하게 플래그를
  심는다(재방문 무노출). 막다른 화면 금지(AC4)와 일관 — 어떤 닫힘도 탐색으로 복귀시키고 다시 안 띄운다.

- **확정④ — 모바일 마운트 위치 = `app/index.tsx`(찾기 탭 = 진입 화면).** 전역 `_layout.tsx`가 아니라
  진입 탭 화면에 얹어 "진입 화면 위"라는 AC1을 만족시킨다(모바일 진입 화면은 현재 placeholder 이나, 온보딩은
  그 위에 떠도 무방 — 지도 미구현은 E7, 온보딩은 자기완결). 웹은 `app/page.tsx`(=`/` 진입)에 얹는다.

- **확정⑤ — 카피(해요체·EXPERIENCE 보이스 준수, dev 가 그대로 사용).** 카피 단계 확정값:
  - 제목: **"DeskNow에 오신 걸 환영해요"**
  - 본문: **"내 주변에서 지금 비어 있는 스터디룸을 지도와 목록으로 찾아 바로 예약할 수 있어요."**
  - 버튼: **"지금 시작"**
  - (KTH가 추후 카피만 조정할 수 있으나, dev 는 이 값으로 구현한다 — 빈 문구/placeholder 금지.)

### 설계 핵심 — 첫 방문 1회 노출 (공통 패턴, 두 표면 동일)

```
플래그 = 스토리지에서 읽기 ("desknow:onboarding:seen")
초기 렌더(서버/측정 전) → 미표시 (깜빡임·하이드레이션 미스매치 방지)
플래그 없음 → shouldShow = true → 오버레이 표시
닫힘(버튼/Esc/백/바깥탭 — 확정③) → 스토리지에 "1" 기록 + 즉시 미표시
재진입 → 플래그 있음 → 영구 미표시 (서버 추적 0 · 기기 로컬)
```

- **웹 SSR 깜빡임 = 최대 함정.** `localStorage`는 서버에 없으므로 **초기 상태를 "미표시"로 두고 effect 에서만
  켠다.** 동기로 읽으려 하면 하이드레이션 미스매치. 반대로 SSR에서 "표시"로 가정하면 본 적 있는 재방문자에게도
  한 틱 깜빡인다 → 둘 다 금지. mounted-gate 패턴이 정답(`useOnlineStatus.ts`의 `getServerSnapshot=true` 정신과
  같은 SSR 안전 사상).
- **모바일 AsyncStorage 는 비동기** → 읽기 완료 전엔 미표시, 완료 후 표시. 언마운트 가드 필수.
- **막다른 화면 금지(AC4):** 오버레이는 *이미 렌더된 진입 화면 위에* 뜬다. 닫으면 그 화면이 그대로 드러난다
  (별도 네비게이션 불필요). 닫는 수단이 항상 존재(버튼 + Esc/백)해야 한다.
- **로그인·네트워크·위치와 무관한 축:** 온보딩 플래그는 기기 로컬 1개 불리언일 뿐. 세션/네트워크 단절(3.8)/
  위치 권한(3.6)과 엮지 말 것. 네트워크 호출 0(신규 쿼리·SDK 0).
