# Story 4.2: 상세 정보 화면

Status: done

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a 예약자,
I want 룸의 가격·부대시설·영업시간·위치·후기를 상세에서 위계 있게 보고 같은 페이지에서 예약을 전개하길,
so that 화면 이동 없이 정보 확인부터 예약까지 이어간다 (FR-11, UJ-1·2).

> **이 스토리는 Story 3.3이 만든 "상세 스텁 라우트"(`/rooms/[room_id]/page.tsx` — "상세 화면은 곧 제공돼요")를 실제 상세 화면으로 채운다.** 3.3은 ① 단일 룸 신선 요약 엔드포인트 `GET /api/v1/rooms/{room_id}`(`RoomSummary`)와 ② 재사용 가능한 순수 로직(`roomSummary.ts`)·`useRoomSummary` 훅·③ 막다른 화면 방지용 스텁 라우트를 남겨두었다. 4.2는 그 자산 위에 **상세 화면 + 같은 페이지 내 예약 전개 메커니즘**을 얹고, `RoomSummary`에 **좌표(lat/lng)를 추가 노출**해 상세에 **위치 미니 지도**를 표시한다.

## 범위 결정 (KTH 확정 2026-06-16 — create-story)

이 화면이 닿는 세 경계(예약 UI=4.3~4.5 · 후기=Epic 5 · 주소 데이터 갭)를 다음과 같이 확정한다:

1. **예약 전개 = 메커니즘 + placeholder만.** "같은 페이지 내 예약 전개"(AC2)의 **섹션 전개·여백 우선 레이아웃·페이지 이동 0** 메커니즘만 4.2가 구현하고, 그 안의 **달력 + 슬롯 피커는 "예약 준비 중" placeholder**로 둔다. 실제 캘린더/날짜 선택은 **Story 4.3**, 연속 슬롯 선택은 **4.4**, 즉시 예약 확정은 **4.5**가 이 자리를 채운다. 에픽 경계 준수 — 4.2는 예약 흐름의 **컨테이너**를 만들고, 내용물은 후속이 채운다.

2. **후기/평점 = 후기 섹션 자리(placeholder).** 후기 도메인(모델·작성 API·평점 집계)은 **Epic 5 소유**(5-5 후기 작성·5-6 후기 답글)이며 **아직 존재하지 않는다.** 3차 위계에 후기 섹션 **자리**를 두되 **"아직 등록된 후기가 없어요"(곧 제공)** placeholder로 표시하고, 실데이터·작성·평점 집계는 Epic 5가 이 자리에 배선한다. **본 스토리에서 reviews 모델/API를 신설하지 않는다.**

3. **위치/주소 = 좌표 노출 + 미니 지도.** ★child 권장(데이터 갭 유지)과 다른 KTH 결정 — **백엔드 확장을 택함.** `RoomSummary`에 **`lat`/`lng`를 추가 노출**(`schemas.py` 변경 · OpenAPI/SDK 재생성)하고, 상세 2차 위계에 **위치 미니 지도(단일 핀)**를 표시한다(3.2 `loadKakaoMaps` 재사용). **주소 텍스트는 여전히 `Room` 모델에 미저장**(3.3 확정 데이터 갭)이라 표시하지 않는다 — **좌표 기반 위치(미니 지도)만**. 주소 텍스트 저장·표시는 별도 후속으로 defer. **따라서 4.2는 프론트 전용이 아니라 백엔드 스코프(RoomSummary 좌표 노출 + SDK 재생성)를 포함한다.**

> **공개 노출 안전성:** `lat`/`lng`는 **공개 스터디룸의 위치**로, 지도 표시가 곧 제품 목적이다(3.2 `RoomMapItem`도 좌표를 공개 노출 중). `provider_id`·`is_active`·`created_at`·`admin_dong_code`(법정동 코드 — 사람이 읽는 값 아님)는 **여전히 미노출**(내부/소유 필드).

## Acceptance Criteria

**AC1 — 상세 진입 시 3단 정보 위계가 표시된다 (FR-11)**
**Given** 상세 화면(`/rooms/{room_id}`)
**When** 진입하면
**Then** 정보가 **3단 위계**로 표시된다:
- **1차(가장 시각적으로 우선):** 가격(`formatPrice(price_per_hour)+"원 /시간"` 큰 글씨) · 오늘 영업시간(`todayBusinessHours`→`formatHours`; `is_closed_today` 또는 오늘 영업행 없음 → "오늘 휴무") · **예약 가능 배지**(`summaryStatus(remaining_slots)`: 가능=`✓ 예약 가능` / 마감=`✕ 마감`). **신선 `remaining_slots` 기준**(핀/탭 스냅샷 아님). **색 단독 금지**(색 + 아이콘 + 텍스트 3중 신호).
- **2차:** 부대시설 태그(`amenities`→`AMENITY_LABELS` 칩) · 수용 인원(`최대 N인`) · 룸 형태(`ROOM_TYPE_LABELS[room_type]`) · **위치 미니 지도**(AC3).
- **3차:** 후기 섹션 자리 — **"아직 등록된 후기가 없어요"(곧 제공)** placeholder(Epic 5 배선 seam · reviews 모델/API 신설 안 함).

**AC2 — 같은 페이지 내 예약 전개 메커니즘이 동작한다 (FR-11, UX-DR4)**
**Given** 상세 화면
**When** "예약하기"를 누르면
**Then** **별도 페이지 이동 없이 같은 페이지 내에서** 예약 영역(섹션)이 전개된다(스크롤/확장 · **여백 우선 레이아웃** `spacing.6`~`spacing.8`). 전개된 영역 안의 **달력 + 슬롯 피커는 "예약 준비 중" placeholder**로 표시된다(실제 캘린더/슬롯은 Story 4.3·4.4·4.5가 이 자리를 채움 — seam 주석 명시). URL/라우트는 동일 페이지를 유지한다(상세 → 예약 전개가 한 화면).
**And** 즐겨찾기 하트는 **Story 3.7의 공유 `FavoriteButton`을 재사용**해 실제 토글·영속이 동작한다(상세도 3.7이 명시한 "바텀시트·상세·목록 어디서든" 토글 표면 — 비로그인 시 "로그인하면 저장돼요" 유도는 3.7 동작 그대로).

**AC3 — 위치 미니 지도가 표시되고 좌표가 신규 노출된다 (범위 결정 #3)**
**Given** `RoomSummary`에 `lat`/`lng`가 포함됨(본 스토리 신규)
**When** 상세 2차 위계가 렌더되면
**Then** 룸 좌표(`lat`/`lng`)를 중심으로 한 **위치 미니 지도 + 단일 핀**이 표시된다(`loadKakaoMaps` 재사용 — 3.2 동적 로더, REST 키 없이 저장 좌표만 사용). 주소 텍스트는 모델 미저장이라 표시하지 않는다(좌표 기반 위치만).
**And** 지도 SDK 로드 실패 시 **지도 영역만** graceful degrade("지도를 못 불러왔어요" 작은 자리 표시)되고 **상세 화면 전체는 막히지 않는다**(나머지 정보·예약 전개는 정상).

**AC4 — 백엔드: `RoomSummary`가 좌표를 노출하고 SDK가 동기화된다**
**Given** 공개 `GET /api/v1/rooms/{room_id}`
**When** 단일 룸 요약을 조회하면
**Then** 응답에 **`lat`/`lng`가 포함**되고, `provider_id`·`is_active`·`created_at`·`admin_dong_code`는 **여전히 미노출**이다. 미존재/비활성 룸은 **404 `ROOM_NOT_FOUND`**(신규 ErrorCode 0 — 2.3 기존). **OpenAPI/SDK가 재생성**되어 `RoomSummary` 타입에 `lat`/`lng`가 반영되고 **드리프트 게이트(Layer A·B·check-types)가 PASS**한다. **마이그레이션 0**(스키마 컬럼 추가 아님 — `rooms.lat/lng`는 2.1부터 저장 중, 응답 노출만 추가).

**AC5 — 상태 분기가 막다른 화면 없이 처리된다 (NFR-5, 3.8 패턴 계승)**
**Given** 상세 화면
**When** 각 상태가 되면
**Then**
- **로딩:** 상세 스켈레톤(정보 자리).
- **정보 로드 실패(비-2xx):** "정보를 못 불러왔어요" + **다시 시도** + **찾기로 뒤로**(막다른 화면 금지).
- **네트워크 단절:** **3.8 `useOnlineStatus`/`NetworkNotice` 재사용** — "네트워크 연결이 끊겼어요. 연결되면 다시 보여드릴게요." 배너 + **읽기는 캐시 유지**(TanStack 메모리), 예약은 연결 후. **카피는 "네트워크 단절"**([[terminology-network-disconnect-not-offline]] — UX 원안 "오프라인이에요"를 덮는다).
- **404(미존재/비활성 룸):** "그 방은 더 이상 없어요" + 찾기로 돌아가기(막다른 화면 금지).
- **빈 상태:** **N/A**(룸이 있어야 진입 — 누락이 아닌 명시 N/A, EXPERIENCE 상태 매트릭스).
- `prefers-reduced-motion` 시 예약 전개/모션이 축소된다. 모든 마이크로카피는 친근한 해요체.

### 백엔드 — 좌표 노출은 최소 변경 (마이그레이션 0)

- `rooms.lat`/`rooms.lng`는 **2.1부터 DB에 저장**(models.py L109-110·CHECK 범위). 4.2는 **응답 스키마 노출만** 추가 → **마이그레이션 0**. `get_room_summary`는 `room`을 이미 로드하므로 `lat=room.lat, lng=room.lng` 매핑 한 줄.
- **공개 노출 안전:** 좌표는 지도 제품 목적상 공개(3.2 `RoomMapItem` 선례). `admin_dong_code`(법정동 코드)·`provider_id`·`is_active`·`created_at`은 **계속 미노출**. 주소 텍스트는 모델 미저장(만들지 말 것).
- **드리프트 게이트 필수:** 스키마 필드 추가는 OpenAPI/SDK 재생성 누락 시 Layer A/B가 차단(3.2/3.3 실증). snake_case(`lat`/`lng`) 보존.

### 슬롯·시간 anti-pattern — 서버가 1회 계산, 클라는 표시만 (architecture L293·L362)

- 예약 가능 배지 = **신선 `remaining_slots`**(`summaryStatus`) — 클라가 `business_hours`를 슬롯으로 환산하는 코드 **절대 금지**. `todayBusinessHours`는 "오늘 요일 행 선택" 날짜 포맷일 뿐(3.3 동일 계약).
