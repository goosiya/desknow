# Story 4.3: 날짜·슬롯 가용성 표시

Status: done

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a 예약자,
I want 달력에서 날짜를 고르고 그날의 빈 슬롯만 선택 가능한 형태로 보길,
so that 예약 가능한 시간만 명확히 구분해 고른다 (FR-12, UJ-1·2, NFR-1).

> **이 스토리는 Story 4.2가 RoomDetail 의 예약 전개 영역에 남긴 "예약 준비 중" placeholder(`#reservation-section`)를 실제 달력 + 슬롯 피커로 채운다.** 4.2는 ① 같은 페이지 예약 전개 메커니즘(`reservationOpen` 토글·`aria-expanded`/`aria-controls`)과 ② placeholder 자리(seam 주석 "달력=4.3·연속 슬롯=4.4·즉시 예약=4.5")를 남겨두었다. 4.3은 그 자리에 **날짜 선택(달력) + 그날의 1시간 슬롯 가용/비활성 표시 + 빈 날 안내·다음 빈 날짜 제안**까지를 얹는다. **슬롯 탭/드래그/키보드/연속 범위 선택은 전부 Story 4.4** — 4.3은 "어느 슬롯이 선택 가능한지 보이는" 표시 책임까지다(아래 스코프 경계).

## 범위 결정 (KTH 확정 2026-06-16 — create-story)

이 스토리가 닿는 네 경계(백엔드 슬롯 조회 · 예약 가능 기간 · 달력 구현 · 4.4 선택 상호작용)를 다음과 같이 확정한다:

1. **슬롯 API = 신규 엔드포인트 `GET /api/v1/rooms/{room_id}/slots?date=YYYY-MM-DD`.** 3.1(룸별 집계 카운트)·3.3(단일 룸 요약 `remaining_slots`)은 **날짜별 슬롯 목록을 주지 않는다.** 4.3은 백엔드에 **그날의 1시간 슬롯을 슬롯별 상태(`available`/`past`/`reserved`)로 반환하고 `next_available_date`(다음 빈 날짜)를 함께 계산하는** 신규 공개 조회 엔드포인트를 추가한다. `derive_slots`(2.1)를 재사용하고 **`reserved_starts=frozenset()`**(예약 차감 연결 = Story 4.9 seam — 3.1·3.3 동일 패턴). 아키텍처 TanStack 키 `['room', roomId, 'slots', date]`(architecture L275)와 정합.

2. **예약 가능 기간 상한 = 30일(오늘 포함 ~ 오늘+29, ROOM_TZ 기준).** PRD/epics 미정의 — 달력 선택 가능 범위와 `next_available_date` forward 검색 상한을 **30일**로 고정한다. 과거 날짜·30일 초과 날짜는 달력에서 선택 불가(프론트), 백엔드 forward 검색도 30일 내로 제한.

3. **달력 = 경량 커스텀 7열 월 그리드(신규 의존성 0).** `react-day-picker`/`date-fns` 등 라이브러리 **도입하지 않는다.** UX 목업(`key-detail-slotpicker.html`)의 7열 월 그리드 구조를 직접 구현하고 **키보드 내비게이션·aria를 직접** 작성한다(프로젝트 의존성 보수 기조·짧은 기간 30일 부합).

4. **선택 경계 = 4.3은 렌더·날짜선택·빈날제안까지, 슬롯 선택 상호작용은 전부 4.4.** 4.3이 소유: 달력 날짜 선택(키보드 포함)·그날 슬롯의 **가용/비활성 시각 렌더**·빈 날 안내 + 다음 빈 날짜 제안. **범위 밖(4.4)**: 슬롯 탭/드래그 선택·연속 범위(`primary` 강조)·슬롯 키보드 선택·"14시부터 17시까지 선택됨" 피드백·하단 확정 바. → **4.3의 슬롯은 클릭 핸들러 없는 표시 전용**(available 은 "선택 가능한 형태"로 보이되 누름 동작은 4.4가 배선).

## Acceptance Criteria

**AC1 — 백엔드: 날짜별 슬롯 + 다음 빈 날짜 조회 엔드포인트 (FR-12, 범위 결정 #1)**
**Given** 공개 `GET /api/v1/rooms/{room_id}/slots?date=YYYY-MM-DD`
**When** 활성 룸·유효 날짜로 조회하면
**Then** 200으로 그날의 **1시간 슬롯 리스트**(각 `{slot_start(UTC), status}`)와 **`next_available_date`**가 반환된다.
- `status` = `available` | `past`(룸 타임존 기준 현재시각 이전) | `reserved`(이미 예약된 `slot_start` — **현재 `reserved_starts=frozenset()`이라 미발생, Story 4.9 연결 seam**).
- 슬롯은 `derive_slots`(2.1) 재사용으로 도출(휴무·미영업 요일 → 빈 리스트). **슬롯 재계산 로직 신설 금지**(단일 출처).
- `next_available_date` = 요청 날짜 **다음날부터** 30일 상한 내 첫 "가용 슬롯 ≥ 1개" 날(ROOM_TZ 기준 `date`), 없으면 `null`.
- 미존재/비활성 룸 → **404 `ROOM_NOT_FOUND`**(2.3 기존·신규 ErrorCode 0). 잘못된 `date` 형식 → **422 `VALIDATION_ERROR`**(Pydantic→1.5 변환). **인증 없음(공개)** — 탐색·상세는 비로그인(PRD §FR-2, 3.1·3.3 동일). **마이그레이션 0·신규 의존성 0.**

**AC2 — 달력에서 날짜를 고르면 그날 슬롯이 상태별로 표시된다 (FR-12)**
**Given** 상세의 예약 전개 영역(4.2 `#reservation-section`) 안 달력
**When** 날짜를 선택하면
**Then** 그날의 1시간 슬롯이 표시되고, **과거 날짜·당일 현재시각 이전 슬롯(룸 타임존 기준)·이미 예약된 슬롯은 비활성**으로 표시된다.
- 슬롯 시각 = `available`(기본 면 + 테두리·"선택 가능한 형태") / `past`·`reserved`(`muted` 배경·`muted-foreground`·비활성) — **색 단독 금지**(아이콘/텍스트 라벨 병행, DESIGN L193·L273). 범례(legend)로 "마감·지난 시간" 표기.
- 달력: 과거 날짜 비활성·30일 초과 비활성·오늘 이후 선택 가능. 초기 선택 = 오늘(ROOM_TZ). 선택일 강조(`primary`).
- **슬롯은 표시 전용**(탭/드래그/키보드 선택은 4.4 — 범위 결정 #4). seam 주석으로 4.4 자리 명시.

**AC3 — 그날 빈 슬롯이 0개면 안내 + 다음 빈 날짜를 제안한다 (FR-12)**
**Given** 선택한 날의 `available` 슬롯이 0개(휴무·전부 지난 시간·전부 예약 등)
**When** 날짜를 선택하면
**Then** **"이 날은 다 찼어요. 다른 날을 골라보세요."** + **다음 빈 날짜 제안**(`next_available_date`)이 표시된다.
- `next_available_date`가 있으면 "예: 6월 18일은 자리가 있어요" 형태로 제안하고 **그 날로 이동하는 어포던스**(누르면 달력 선택 이동)를 둔다.
- `next_available_date`가 `null`(30일 내 빈 날 없음)이면 제안 없이 안내 문구만(막다른 화면 금지 — 달력은 계속 조작 가능).

**AC4 — 신선도·상태 분기가 막다른 화면 없이 처리된다 (NFR-1, 3.8 패턴, [[availability-freshness-policy]])**
**Given** 슬롯 영역
**When** 각 상태가 되면
**Then**
- **신선도:** 슬롯 조회는 **진입/날짜 변경 시 신선**(상세·슬롯 = 진입 시 최신 — [[availability-freshness-policy]]). 키 `['room', roomId, 'slots', date]`. query-client 기본 `refetchOnMount:'always'` 의존(useRoomSummary 선례).
- **로딩:** 슬롯 그리드 스켈레톤(달력은 즉시 표시 — 클라 계산).
- **실패(비-2xx):** 슬롯 영역만 "시간표를 못 불러왔어요" + 다시 시도(달력·상세 나머지는 정상 — 부분 degrade).
- **네트워크 단절:** 4.2의 `useOnlineStatus`/`NetworkNotice` 패턴 계승 — 단절을 슬롯 에러로 오인 표시하지 않음(`isOnline && isError` 게이팅). 읽기 캐시 유지.
- **404(예약 전개 중 룸 소멸):** 상세 본문이 이미 404 분기(4.2)를 가지므로 슬롯 영역은 상세 가드 하위에서만 렌더(중복 404 UI 만들지 않음).
- `prefers-reduced-motion` 존중. 모든 마이크로카피 해요체.

### 백엔드 — `/slots`는 `get_room_summary`의 "임의 날짜·슬롯별 상태" 확장 (마이그레이션 0)

- `get_room_summary`(3.3)는 **오늘** 슬롯의 **카운트**(`remaining_slots`)만 준다. `/slots`는 **임의 `target_date`**의 **슬롯 리스트 + 상태**를 준다 — 같은 `derive_slots`·같은 404 가드·같은 `>= current` 경계를 쓰되 카운트 대신 슬롯별 `status`를 매핑한다. **슬롯 도출 로직 재구현 0**(단일 출처).
- **상태 매핑 우선순위:** `reserved`(예약됨 — 현재 빈 집합이라 미발생) → `past`(`< current`) → `available`. `reserved`를 먼저 판정해야 4.9 연결 후 "예약+과거" 슬롯이 일관 표기된다.
- **`next_available_date` 검색 = 같은 룸 데이터 in-memory 반복**(DB 재조회 0). `business_hours`/`holiday_dates`를 1회 조회해 두고 후보 날짜마다 `derive_slots`만 재호출. 30일 상한(범위 결정 #2).
- **마이그레이션 0:** `rooms`/`business_hours`/`holiday_exceptions` 컬럼만 읽음 — 신규 리비전 없음(3.1·3.3·4.2와 동일). 라이브 Supabase `alembic upgrade head` 대상 변화 없음([[dev-workflow-policy-live-db-migration]] — 본 스토리는 스키마 무변경이라 마이그레이션 적용 없음).

### 신선도 계약 — 상세·슬롯 = 진입 시 최신 ([[availability-freshness-policy]], 3.1 표)

- 슬롯 영역은 **진입/날짜 변경 시 신선** 조회(핀 스냅샷 재사용 아님). 키 `['room', roomId, 'slots', date]`(architecture L275) — 날짜별로 캐시 분리. query-client 기본 `refetchOnMount:'always'`(useRoomSummary 의존)로 전개/날짜 변경 시 신선.
- **예약 차감 실값은 4.9.** 현재 `reserved_starts=frozenset()`이라 "이미 예약된 슬롯=비활성" 분기는 **구조적으로 배선되되 실제 reserved 슬롯은 4.9 차감 연결 후** 나타난다(3.1·3.3과 동일 seam). 4.9가 `reserved_starts`를 채우면 `/slots`·집계·요약이 **자동 반영**(전부 `derive_slots` 단일 출처).
- **뮤테이션 무효화는 4.5/4.9 소유:** 예약 확정/취소 성공 시 `['room', roomId, 'slots', date]`·핀 집계 invalidate(architecture L276) — 본 스토리는 **읽기만** 구현(무효화 트리거는 예약 뮤테이션이 생기는 4.5·4.9). 키를 정확히 맞춰 두는 것이 4.3의 책임(후속이 이 키를 invalidate).

### 슬롯·시간 anti-pattern — 서버가 도출, 클라는 표시만 (architecture L362·L293)

- **클라가 `business_hours`를 슬롯으로 환산하는 코드 절대 금지.** 슬롯 리스트·상태는 **전부 서버**(`/slots`)가 준다. 프론트 `slots.ts`는 **표시 포맷**(UTC→KST 벽시계 "14:00")·**달력 그리드 계산**(날짜 산술)·**선택 가능 날짜 범위 판정**만 — 슬롯 가용성 계산은 하지 않는다.
- `slot_start`는 **UTC 인스턴트**(서버), 표시는 **Asia/Seoul 벽시계**(클라 Intl 포맷). roomSummary.ts의 `Intl.DateTimeFormat("Asia/Seoul")` 패턴 재사용(신규 의존성 0).

### 달력 — 경량 커스텀, 라이브러리 미도입 (범위 결정 #3 · 회고 ④ 라이브러리 실측)

- **`react-day-picker`/`date-fns` 도입 금지.** UX 목업 7열 월 그리드를 직접 구현. 30일 짧은 범위·단일 월 중심이라 커스텀이 충분. **접근성(키보드 방향키·Enter 선택·`role=grid`/`gridcell`·`aria-selected`/`aria-disabled`·한국어 `aria-label`)은 직접 작성** — 이 부담을 수용하는 것이 본 결정의 트레이드오프.
- 날짜 산술은 네이티브 `Date` + Intl로(타임존 함정 주의: "오늘"은 ROOM_TZ 기준 — `kstToday()` 단일 출처). 달력 셀 선택 가능 범위 = `[today, today+29]`.
- **set-state-in-effect 함정(3.5/3.6 실발생):** 달력 초기 선택일(`kstToday()`)은 `useState` 초기값으로(effect에서 setState 금지). lint 실측 통과 형태로 작성.

### 선택 경계 — 4.3=표시, 4.4=선택 (범위 결정 #4 · 에픽 경계)

- 4.3 슬롯은 **클릭/선택/드래그 핸들러 없음.** `available` 슬롯은 "선택 가능한 형태"(기본 면+테두리)로 **보이기만** 한다 — 실제 누름·연속 범위(`primary` 강조)·키보드 선택·"14시~17시 선택됨" 피드백·하단 확정 바는 **전부 4.4**가 `SlotGrid`에 배선한다.
- **달력 날짜 선택은 4.3 소유**(키보드 포함) — 날짜를 골라야 그날 슬롯을 보여줄 수 있으므로. 헷갈리지 말 것: **날짜 선택=4.3, 슬롯 선택=4.4.**
- `SlotGrid`를 4.4가 확장하기 쉽게 props/구조를 설계(슬롯 배열·상태를 받아 표시 — 4.4가 onSelect·selectedRange를 추가). dead UI 만들지 않되 확장 seam은 주석으로 남김.

### 막다른 화면 금지 · 네트워크 단절 용어 (NFR-5, 3.8/4.2 패턴)

- 슬롯 로딩=스켈레톤(달력은 즉시) / 실패="시간표를 못 불러왔어요"+다시 시도(부분 degrade — 상세·달력 정상) / 빈 날="다 찼어요"+다음 빈 날짜 제안(AC3) / 단절=`NetworkNotice`(4.2 패턴, `isOnline && isError` 게이팅).
- **용어:** "네트워크 단절"·카피 "네트워크 연결이 끊겼어요…"([[terminology-network-disconnect-not-offline]]). 모든 마이크로카피 해요체.

### 라우팅 — `/{room_id}/slots`는 깊은 경로라 순서 무해 (2.2/2.3 가드 이해)

- 정적/동적 순서 가드는 **같은 깊이**에서 발생한다(`/availability` vs `/{room_id}`). `/{room_id}/slots`는 **2-세그먼트**라 1-세그먼트 라우트들과 매칭 충돌 없음 — 동적 `/{room_id}` 그룹 근처에 자유롭게 둔다. 모듈 docstring에 한 줄 추가(혼동 방지).
- `date`는 **필수 쿼리 파라미터**(`Query(...)`) — 누락/오형식은 Pydantic 422(1.5 핸들러 변환). 경로에 안 넣는 이유: 날짜는 필터지 리소스 식별자가 아님(REST 관례·`/search` 쿼리 선례).

### 표준 에러 (1.5) — 신규 ErrorCode 0

- 404=`ROOM_NOT_FOUND`(2.3 기존, `get_room_summary` 선례) · 422=`VALIDATION_ERROR`(Pydantic 자동). 슬롯 조회는 읽기·외부 호출 없음 → 신규 코드 불필요. 빈 슬롯·빈 다음날은 **에러가 아니라 정상 200**(빈 리스트·`null`).
