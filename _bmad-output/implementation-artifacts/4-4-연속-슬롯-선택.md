# Story 4.4: 연속 슬롯 선택

Status: done

## Story

As a 예약자,
I want 연속된 여러 슬롯을 한 번에 선택하길,
so that 실제 사용 길이에 맞춰 예약한다 (FR-13, UJ-1, UX-DR4, NFR-5).

> **이 스토리는 Story 4.3이 `SlotGrid.tsx`에 남긴 "4.4 SEAM"(표시 전용 → 선택 상호작용 배선)을 채운다.** 4.3은 그날 슬롯을 상태별(`available`/`past`/`reserved`)로 **표시만** 했다(클릭 핸들러 0). 4.4는 그 `available` 슬롯에 **탭·드래그·키보드로 연속 구간을 선택**하는 상호작용을 얹고, 선택 범위를 `primary`로 강조하며, **비연속·점유 구간 선택을 막고**(D1), 스크린리더에 "14시부터 17시까지 선택됨"을 피드백하고, 하단에 **선택 요약(범위·날짜·시간·금액)**을 표시한다. **실제 예약 확정(`예약하기` CTA·POST 제출·성공 피드백·캐시 무효화)은 전부 Story 4.5** — 4.4는 "어떤 연속 구간을 고를지"까지이고 그 선택 상태를 `ReservationPanel`이 보유/노출하는 데서 끝난다(아래 스코프 경계).

## 범위 결정 (KTH 확정 2026-06-16 — create-story)

이 스토리가 닿는 두 경계(4.4/4.5 분할 · 금액 표시)를 다음과 같이 확정한다:

1. **4.4/4.5 경계 = 요약 readout까지.** 4.4가 소유: 슬롯 **선택 상호작용**(탭/드래그/키보드)·**연속 범위 `primary` 강조**·**비연속/점유 차단**(D1)·SR "14시부터 17시까지 선택됨"·**하단 선택 요약 표시**(범위·날짜·시간·금액). `selectedRange` 상태는 **`ReservationPanel`이 보유/노출**한다. **범위 밖(4.5)**: `예약하기` CTA 버튼·실제 POST 제출·"예약이 완료됐어요!" 성공 피드백·낙관적 업데이트·`['room', roomId, 'slots', date]`/핀 집계 invalidate·동시성 충돌 안내("앗, 방금 다른 분이…"). → **4.4는 dead/비활성 버튼을 만들지 않는다**(4.2/4.3 placeholder 철학 계승) — `예약하기` 버튼 자체를 4.5가 추가한다. 4.4의 하단 바는 **선택 요약 readout만**.

2. **선택 요약에 금액 표시.** 하단 요약 = "**14:00–17:00 · 6월 16일 · 3시간 · 24,000원**" 형태(목업 `.confirm-row` 선례). 금액 = `선택 슬롯 수(시간) × price_per_hour`. **`price_per_hour`는 `RoomSummary`(스키마 L321)에 이미 있고 `RoomDetail`이 `useRoomSummary`로 이미 보유**(추가 조회 0) → `ReservationPanel`에 `pricePerHour` prop으로 전달한다. 총액은 표시만(예약 확정/결제 의미는 4.5 — 본 서비스는 결제 없음·FR-14).

## Acceptance Criteria

**AC1 — 탭/드래그로 연속 구간 선택 + primary 강조 + 비연속/점유 차단 (FR-13, D1, UX-DR4)**
**Given** 그날의 슬롯 피커(4.3 `SlotGrid`)
**When** 연속 구간(예: 14~17시)을 **탭 연속 또는 드래그**로 선택하면
**Then** 선택된 슬롯이 `primary`로 **연속 강조**되고, **비연속 선택은 시각적으로 불가능하게 막힌다**(D1).
- **선택 모델 = 앵커 + 연속-가용 확장.** 앵커 슬롯에서 시작해 인접 `available` 슬롯으로만 구간이 확장된다. **구간 안에 `past`/`reserved` 슬롯이 하나라도 끼면 그 지점에서 막혀** 비연속·점유 선택이 구조적으로 불가능하다(D1 + "구간 중 하나라도 빈 슬롯 아니면 막힘"을 단일 규칙으로 동시 충족).
- **탭:** `available` 슬롯 탭 → 그 슬롯 1칸 선택(앵커). 다른 `available` 슬롯 탭 → 앵커~대상 **연속-가용 구간**으로 확장(구간이 점유를 가로지르면 무시·확장 안 함). 현재 단일 선택 슬롯 재탭 → 선택 해제.
- **드래그:** `available` 슬롯에서 누른 채 끌면 연속-가용 범위로 실시간 확장(첫 점유 슬롯에서 멈춤). 떼면 확정. 터치/마우스 공통(Pointer Events). ≥44×44px 타깃(EXPERIENCE L166).
- **시각:** 선택 = `bg-primary`/`text-primary-foreground`/`border-primary`(목업 `.slot.sel`). 비선택 `available` = 기존 `bg-card border-border`. `past`/`reserved` = 기존 muted+취소선(4.3 보존). **색 hex 직접 금지 — Tailwind 토큰만**(3.3/4.2/4.3 선례).

**AC2 — 키보드 대안(방향키 + Enter 시작/끝) + 스크린리더 구간 피드백 (NFR-5, UX EXPERIENCE L163·L169)**
**Given** 키보드 사용자
**When** 슬롯을 선택하면
**Then** **방향키로 슬롯 포커스 이동 + Enter로 시작/끝을 지정**하는 키보드 대안이 제공되고, 스크린리더에 **"14시부터 17시까지 선택됨"**이 피드백된다(드래그가 유일 경로가 아님).
- **로빙 탭인덱스**(4.3 `Calendar` 패턴 미러): 슬롯 그리드 안에서 한 칸만 `tabIndex=0`, 나머지 `-1`. 방향키 ←/→ = ±1 슬롯, ↑/↓ = ±3(3열 그리드). 비활성(`past`/`reserved`) 슬롯은 포커스 스킵.
- **Enter/Space:** 첫 Enter = 시작(앵커) 지정. 둘째 Enter = 끝 지정 → 연속-가용 구간 확정(구간이 점유를 가로지르면 무시·앵커 유지). 앵커만 있는 상태에서 같은 칸 Enter → 1칸 선택 유지/해제.
- **SR 피드백:** `aria-live="polite"` 영역에 선택 구간을 **한국어 시각 구간**으로 알림 — "**14시부터 17시까지 선택됨**"(끝 = 마지막 선택 슬롯 시작 + 1시간, ROOM_TZ 벽시계). 단일 슬롯이면 "14시부터 15시까지 선택됨". 선택 슬롯 `aria-pressed`(또는 `aria-selected`)로 선택 상태 노출. **색 단독 금지** 일관(선택도 텍스트/aria로 병행 신호).

**AC3 — 하단 선택 요약(범위·날짜·시간·금액) + selectedRange를 패널이 보유 (범위 결정 #1·#2)**
**Given** 빈 연속 슬롯이 선택된 상태
**When** 선택이 바뀌면
**Then** 하단에 **"14:00–17:00 · 6월 16일 · 3시간 · 24,000원"** 형태 선택 요약이 표시된다.
- 범위 = `formatSlotLabel(첫 슬롯)`–`(마지막 슬롯 + 1h KST)`. 시간 = 선택 슬롯 수. 금액 = `슬롯 수 × pricePerHour`(천단위 구분 — `RoomDetail`의 `formatPrice` 패턴 재사용/공유).
- **선택 없음** = 요약 미표시(또는 "시간을 선택해 주세요" 안내) — 막다른 화면 금지·dead UI 금지.
- **`selectedRange`는 `ReservationPanel`이 보유/노출**(4.5가 이 상태를 읽어 `예약하기` 제출에 사용 — 4.5 seam 주석). **`예약하기` CTA·POST는 본 스토리에서 만들지 않는다**(범위 결정 #1).
- **날짜 변경 시 선택 초기화:** `selectedDate`가 바뀌면(달력 날짜 선택·다음 빈 날짜 제안 클릭) `selectedRange`를 리셋한다 — **렌더 중 상태 조정**(4.3 `Calendar` `prevValue` 패턴) 또는 `SlotGrid`를 `key={selectedDate}`로 재마운트해 **effect setState 금지**(3.5/3.6 함정).

**AC4 — 4.3 상태/막다른 화면 분기 무회귀 + 선택은 가용 슬롯에만 (NFR-1·NFR-5, 4.3 AC4 보존)**
**Given** 슬롯 영역
**When** 선택 상호작용을 얹어도
**Then** 4.3의 로딩(스켈레톤)·실패("시간표를 못 불러왔어요"+다시시도)·빈날("다 찼어요"+다음 빈 날짜 제안)·네트워크 단절(`isOnline && isError` 게이팅·`NetworkNotice`) 분기가 **그대로 동작**한다(선택은 정상 데이터일 때만 가능).
- `past`/`reserved` 슬롯은 **선택/포커스 불가**(표시 전용 유지) — 4.3 비활성 렌더(muted+취소선+sr-only) 보존.
- 빈 날(가용 0)·실패·로딩·단절 상태에선 선택 요약을 띄우지 않는다(선택할 게 없음).
- `prefers-reduced-motion` 존중(선택 강조 전환 모션 최소·≤100ms 기능 피드백만). 모든 마이크로카피 해요체.

### 선택 모델 — 앵커 + 연속-가용 확장이 D1을 구조적으로 강제 (FR-13·D1)

- **단일 규칙:** 앵커에서 시작해 **인접 `available` 슬롯으로만** 구간이 자란다. `clampContiguousAvailable`는 진행 방향 첫 비-`available`(`past`/`reserved`)에서 멈춘다. → **떨어진 두 슬롯이 한 구간이 될 수 없고**(비연속 불허·D1), **구간에 점유가 끼면 자동 차단**("구간 중 하나라도 빈 슬롯 아니면 막힘")이 **하나의 함수로 동시 충족**된다. 별도 "비연속 감지" 로직 불필요.
- **slot_start는 서버 UTC 인스턴트**(4.3). 선택은 **렌더된 `slots` 배열의 인덱스 구간**으로 다룬다(시각 산술로 연속성 판정 금지 — 배열은 이미 시간순). 끝 라벨만 "마지막 슬롯 +1h"를 KST 벽시계로 표시(예: 14·15·16 선택 → "14:00–17:00").
- **클라가 슬롯을 재계산하지 않는다**(architecture L362) — `slots.ts` 추가분도 인덱스 산술·표시 포맷·금액 곱셈만. 가용성 판정은 서버가 준 `slot.status`를 읽을 뿐.

### 상호작용 3경로 — 드래그·탭·키보드 (UX EXPERIENCE L151·L163·L169, NFR-5)

- **드래그 = 1급 제스처**(L151) — Pointer Events로 마우스+터치 통합(`onPointerDown/Move/Up`·`setPointerCapture`·`touch-action:none`). **키보드 대안 필수**(L163 — 드래그가 유일 경로가 되면 접근성 위반): 방향키 이동 + Enter 시작/끝. **탭 연속**도 허용(L151).
- **SR 피드백(L169):** "14시부터 17시까지 선택됨" — `aria-live="polite"`. 색 단독 금지(L165) 일관: 선택을 `aria-pressed`+요약 텍스트로도 신호.
- **터치 타깃 ≥44×44px**(L166) — 기존 `h-12`(48px) 유지/`tap-target`.
- **reduced-motion**(DESIGN L287-289): 선택 강조 전환은 ≤100ms 기능 피드백만, 비필수 모션 제거.

### 4.4/4.5 경계 — 선택까지가 4.4, 확정은 4.5 (범위 결정 #1 · 에픽 경계)

- **4.4 끝점:** `selection`(연속 구간) 상태가 `ReservationPanel`에 보유되고 하단 요약(범위·날짜·시간·금액)이 표시되는 것까지. **여기서 멈춘다.**
- **4.5 소유:** `예약하기` CTA 버튼·POST 제출(선택 슬롯 `slot_start[]`를 all-or-nothing 예약)·"예약이 완료됐어요!" 성공 피드백·낙관적 업데이트·`['room', roomId, 'slots', date]`/핀 집계 invalidate(architecture L276)·3스텝 완료(NFR-3). **동시성 충돌 안내**("앗, 방금 다른 분이 먼저 잡았어요"+인접 빈 슬롯 재표시·EXPERIENCE L136·L205) = **4.6**.
- **dead 버튼 금지:** 4.4는 `예약하기` 버튼을 비활성/seam으로도 **만들지 않는다**(4.2/4.3 placeholder 철학) — 4.5가 버튼+제출을 함께 추가한다. 4.4는 selection을 `ReservationPanel`이 노출하게만 두고(4.5가 읽을 수 있게), seam 주석으로 표시.

### 날짜 변경 시 선택 리셋 — set-state-in-effect 회피 (3.5/3.6 함정 · AC3)

- `selectedDate`가 바뀌면 이전 날의 `selection`은 무효 → 리셋. **권장: `<SlotGrid key={selectedDate} … />`** 로 날짜 변경 시 자연 재마운트(선택 상태가 `SlotGrid` 내부면 자동 클리어 / `ReservationPanel` 보유면 `onSelect(null)`을 트리거하기보다 `key`로 패널 하위 선택을 끊는 게 단순). **또는** `Calendar`의 `prevValue` 렌더-중-조정 패턴으로 `selectedDate` 변화 감지 시 `setSelection(null)`. **effect에서 setState 금지**(lint `react-hooks` 실측 — 3.5/3.6 실발생 함정).
- 단, `selection`을 `ReservationPanel`이 보유해 4.5가 읽어야 하므로, `key` 재마운트는 `SlotGrid` 내부가 아니라 **패널의 `selection`도 함께 리셋**되도록 설계(가장 단순: 패널이 `selectedDate`를 prevValue로 비교해 렌더 중 `setSelection(null)`; `SlotGrid`는 controlled `selection` prop을 받음).
