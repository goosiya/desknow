# Story 4.7: 예약 취소

## Story

As a 예약자,
I want 예약 시작 6시간 전까지 예약을 취소하길,
so that 계획이 바뀌어도 슬롯이 다른 사람에게 다시 열린다 (FR-16, UJ-2, NFR-7).

> **이 스토리는 "취소 엔드포인트 + 6h 게이트" BE 역량 스토리다 — 새 FE 표면을 만들지 않는다.** 취소의 **데이터 계층 프리미티브(`cancel_reservation`: 멱등·동일 트랜잭션 슬롯 재활성·종료상태 no-op)는 이미 4.1에 구현 완료**다(`reservations/service.py:173-184`·`_transition_to_terminal:153-170`). 4.1 docstring이 명시한다: *"6h 취소 윈도우(`is_within_hours`)는 **Story 4.7**이 이 함수 호출 *전*에 게이팅한다. 본 함수는 시간 경계를 판정하지 않는다."* 따라서 4.7은 **(A) 취소 엔드포인트**(`POST /rooms/{room_id}/reservations/{reservation_id}/cancel`)를 신설해 소유권·6h 윈도우·멱등을 강제하고, **(B) SDK 재생성**으로 후속 FE(4.8)가 소비할 계약을 노출하며, **(C) 6h 경계·멱등·동시취소 불변식을 자동화 테스트**로 증명한다. **취소 버튼·AC1 비활성 UX는 4.8(예약현황 list = 취소 버튼의 자연 거처) 소유**다(KTH 확정 — 아래 범위 결정 #1).

## Acceptance Criteria

**AC1 — 6h 윈도우 경과 시 취소 차단 (서버 권위·FR-16, NFR-1)**
**Given** 확정(`confirmed`) 예약
**When** 예약 시작(점유 슬롯 중 **가장 이른** `slot_start`, 룸 타임존 무관 순수 duration 기준)까지 **6시간 미만** 남았거나 이미 지난 시점에 취소를 요청하면
**Then** 엔드포인트가 **409 `CANCEL_WINDOW_PASSED`**("이제 6시간이 안 남아서 취소가 어려워요.")로 거부하고 **상태 전이·슬롯 변경 0**(예약은 `confirmed` 유지).
- **6h 경계 정책(deferred L91 의무 회수):** 취소 **가능** = `earliest_slot_start - now ≥ 6시간`(정확히 6h 남은 시점 **포함** = 취소 가능). 취소 **차단** = `earliest_slot_start - now < 6시간`(6h 미만·경계 미만·과거). 부동소수 경계 흔들림 방지를 위해 **`timedelta` 직접 비교**(`(earliest - now) >= timedelta(hours=6)`)로 판정한다(float `hours_until` 미사용 — 1µs 경계 모호성 제거). 경계 테스트 3종(정확히 6h·6h−1초·6h+1초) 의무.
- **에픽 AC1의 "취소 버튼 비활성 + 안내"는 4.8 UI**가 렌더한다(범위 결정 #1). 4.7은 그 버튼이 호출할 엔드포인트의 **서버측 최종 강제**를 책임진다 — FE가 우회하거나 시계 스큐로 활성 상태에서 눌러도 6h 경과면 409.

**AC2 — 6h 전 취소 시 슬롯 즉시 재활성 + 히스토리 취소 상태 (FR-16, NFR-7)**
**Given** 시작 6시간 전 이전(`earliest_slot_start - now ≥ 6h`)의 확정 예약
**When** 취소하면
**Then** 점유했던 **모든 슬롯이 동일 트랜잭션 내에서 즉시 DELETE(재활성)** 되고(같은 `(room_id, slot_start)`를 다른 예약이 재점유 가능), 예약은 `status='cancelled'`로 **히스토리에 잔존**하며 200 + 갱신된 `ReservationPublic`(`status='cancelled'`·`slot_starts=[]`)을 반환한다.
- **재구현 금지:** 4.1 `cancel_reservation`(→ `_transition_to_terminal`: status flip + `_release_slots` DELETE를 단일 트랜잭션 commit)을 **그대로 호출**한다. 6h 게이트는 이 호출 **전에** 라우터/서비스가 강제한다(4.1 함수는 시간 판정 안 함).

**AC3 — 종료 상태 멱등 (FR-16, SM-3·범위 결정 #2)**
**Given** 이미 종료 상태(`cancelled`·`rejected`)인 예약
**When** 추가 취소를 시도하면
**Then** **멱등하게 무시**되고(DB 쓰기 0) 200 + 현재 상태를 반환한다(에러 아님 — 6h 게이트도 적용 안 함: 종료 상태는 이미 슬롯이 비어 있어 시간 판정 무의미).
- **순서(중요):** 로드 → **종료 상태면 즉시 멱등 반환**(6h 검사 전) → 아니면(confirmed) 6h 게이트 → `cancel_reservation`. 종료 상태는 점유 슬롯이 0개라 `earliest_slot_start` 계산 자체가 불가하므로 **반드시 6h 검사보다 먼저** 멱등 분기한다.
- 4.1 `_transition_to_terminal:162-163`(`if reservation.status in _TERMINAL_STATUSES: return reservation`)이 멱등을 보장하나, 라우터가 **6h 게이트를 종료 상태에 적용하지 않도록** 분기 순서를 명시한다.

**AC4 — 소유권·인증 강제 (RBAC, FR-16·architecture.md §Boundaries)**
**Given** 취소 요청
**When** 요청자가 미인증이거나, 대상 예약이 존재하지 않거나, **요청자 본인의 예약이 아니면**
**Then** 미인증=401 `UNAUTHENTICATED`, 미존재·비소유=**404 `RESERVATION_NOT_FOUND`**(존재 여부 누설 금지 — `ROOM_NOT_FOUND` 2.3 패턴 미러: 미존재와 타인 예약을 동일 404로 합침), 역할 위반(provider/admin)=403 `FORBIDDEN_ROLE`.
- **인증:** `Depends(require_role("booker"))` — 4.5 라우터(`reservations/router.py:51·69`)의 `_require_booker` 모듈레벨 싱글톤 선례 재사용(ruff B008 회피).
- **소유권:** `session.get(Reservation, reservation_id)` → `None` 또는 `booker_id != principal.user_id` → `DomainError(RESERVATION_NOT_FOUND)`(404). 타인 예약을 403이 아니라 404로 막아 **타인 예약 존재 여부를 누설하지 않는다**(2.3 소유권=백엔드 최종 강제 선례).

**AC5 — 동시 취소 무해성 + 동시성 불변식 테스트 (NFR-7·deferred L12 회수)**
**Given** 같은 확정 예약에 동시(또는 순차) 취소 요청 2건
**When** 경합하면
**Then** 데이터 손상 0(슬롯 점유 행 중복/잔존 없음)·둘 다 최종적으로 `cancelled` 상태를 보고 멱등하게 수렴한다(둘째 전이는 이미 비워진 슬롯에 무해한 no-op).
- **deferred L12 회수(read-then-flip race):** 현 설계(status flip + 슬롯 DELETE)는 둘째 commit이 **무해한 no-op**(DELETE가 이미 0행, status는 이미 cancelled)이라 `SELECT FOR UPDATE`/버전 컬럼 없이도 데이터 정합이 깨지지 않는다 — 이 **무해성을 자동화 테스트로 증명**하고 Dev Notes에 명시한다(낙관적 락 미도입이 sound임을 문서화). 4.6 동시성 테스트 골격(`tests/reservations/test_concurrency.py`·`tests/integration/test_reservations_concurrency.py` skipif) 재사용 가능.

### 6h 경계 정책 — deferred-work L91 의무 회수

- **deferred L91**(`core/time.py:69-89` `hours_until` 부동소수 경계 정밀도)은 명시적으로 **"4.7 취소" 소비 스토리**를 트리거로 지정했다([[dev-workflow-policy-deferred-and-repeat-mistakes]] — 트리거 시 의무 회수). 4.7이 6h 경계 정책을 확정하고 경계 테스트를 추가해 **회수**한다.
- **정책 확정:** 취소 가능 = `earliest_slot_start - now ≥ 6시간`(정확히 6h 남은 시점 **포함**). "6h 전까지 취소"의 *까지*는 6h 마크 포함(에픽 L835·L841 "6시간 전까지"/"6시간 전 이내가 되면" → 6h 미만일 때 차단 = `< 6h`).
- **부동소수 회피:** `hours_until`(float, `total_seconds()/3600`)은 정확히 6.0h에서 1µs 차로 흔들릴 수 있다(L91). 4.7은 **`timedelta` 직접 비교**(`(earliest - now) >= timedelta(hours=6)`)를 써 float·epsilon 없이 결정적으로 판정한다 — `hours_until`/`is_within_hours`를 6h 게이트에 쓰지 않는다(그 함수들은 표시/다른 판정용으로 잔존). 이로써 L91의 "epsilon 허용오차 또는 초 단위 절삭" 위임을 **timedelta 정수 비교로 더 단순하게** 해소한다.
- **경계 테스트(의무):** 정확히 6h(=가능)·6h−1초(=409 차단)·6h+1초(=가능)·과거 시작(=차단). `now` 주입으로 결정적.

### 동시 취소 read-then-flip race — deferred-work L12 회수

- **deferred L12**(`service.py:144-161` 동시 cancel/reject 낙관적 락 부재)는 원래 **Story 4.6** 소유로 배정됐으나 4.6은 KTH 확정으로 **BE 프로덕션 코드 변경 0**(검증 전담)이라 미처리 — **cancel 도메인인 4.7이 자연 귀속**해 회수한다. (재-defer 아님: 트리거 발화·소유 스토리 도달.)
- **현 설계가 sound한 이유:** 같은 confirmed 예약에 두 취소가 동시 진입하면 둘 다 종료상태 검사를 통과해 각각 `status='cancelled'` flip + 슬롯 DELETE + commit한다. 그러나 ① status flip은 둘 다 같은 값(`cancelled`)이라 idempotent ② 슬롯 DELETE는 둘째가 **이미 0행**이라 무해 no-op ③ 점유 행 잔존·중복 0. **데이터 손상 없음** → `SELECT FOR UPDATE`/버전 컬럼 **미도입이 정당**.
- **4.7 회수 방식:** 낙관적 락을 *추가하지 않고*, 이 **무해성을 자동화 테스트로 증명** + Dev Notes에 "낙관적 락 미도입이 sound(둘째 전이=무해 no-op)"를 명시한다(AC5). 진짜 race는 4.6 통합 테스트 골격(skipif Postgres) 재사용 가능(선택 — 결정적 직렬로도 무해성 단언 충분).
[Source: deferred-work.md L11-12(L12) / reservations/service.py:139-170 / 4-6 (BE 무변경 — L12 미처리 확인)]

### 엔드포인트 형태 — 왜 POST …/cancel (DELETE 아님) (범위 결정 #2)

- **취소 = 상태 flip, 행 삭제 아님.** 4.1 모델 설계상 예약 행은 `status='cancelled'`로 **히스토리에 잔존**(models.py:11-12·128-132 — 4.7 "취소 상태로 잔존" + 4.8 현황). `DELETE`는 "리소스 제거" 의미라 오도하고 204는 결과 상태(cancelled)를 못 싣는다.
- **명시적 액션 동사** `POST /rooms/{room_id}/reservations/{reservation_id}/cancel` → 갱신된 `ReservationPublic`(`status='cancelled'`·`slot_starts=[]`) 반환. 멱등(반복 호출 = 현재 cancelled 반환).
- **중첩 경로 일관성:** 기존 4.5 라우터가 `/rooms/{room_id}/reservations`(중첩 — architecture.md L243)라 그 아래 하위 경로로 둔다. `room_id` 경로 파라미터는 소유권 가드에서 `reservation.room_id`와 일관성 검증(불일치=404).

### 소유권·인증 (AC4)

- **인증:** `require_role("booker")` — 4.5 라우터 `_require_booker` 싱글톤 재사용(router.py:51). provider/admin=403, 미인증=401.
- **소유권 = 백엔드 최종 강제(architecture.md §Boundaries L351 / 2.3 선례):** `session.get(Reservation, id)` → `None`/`booker_id != principal.user_id` → **404 `RESERVATION_NOT_FOUND`**(타인 예약 존재 누설 금지 — `ROOM_NOT_FOUND`가 미존재+비소유를 404로 합친 2.3 패턴 미러). 403이 아니라 404(타인 예약 id 추측 차단).
- **신규 ErrorCode `RESERVATION_NOT_FOUND`(404):** `ROOM_NOT_FOUND` 항목을 `ErrorCode`·`DEFAULT_STATUS`에 미러 추가. 이 한 건이 4.7 유일 신규 코드(`CANCEL_WINDOW_PASSED`는 기존 예약분).

### 4.7/4.8/4.9 경계 (혼동 방지)

- **4.7 소유(본 스토리):** 취소 **엔드포인트** + 6h 게이트 + 멱등 + 소유권 + SDK 재생성 + BE 테스트. **FE 0.**
- **4.8 소유:** 예약현황/히스토리 list **FE** — 다가오는/지난 구분, 룸 상세 이동, **취소 버튼 + AC1 비활성(6h 미만)·"이제 6시간이 안 남아서…" 안내 카피**, cancellable/earliest_slot_start **필드 노출(list 스키마)**. 4.7 엔드포인트를 `reservationsCancelReservation`로 소비.
