# Story 5.3: 예약 상태변경 통지 배너

Status: done

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a 예약자,
I want 내 예약이 거절·임의취소되면 다음 접속 시 **어느 예약이(룸·원래 일시) 어떻게(거절/취소) 바뀌었는지**
정확히 통지받고 '확인'으로 닫길,
so that 상태 변화를 놓치지 않고, 여러 예약 중 무엇이 바뀌었는지 헷갈리지 않는다 (FR-18a, UJ-3 거절 결말).

이 스토리는 **Epic 5의 통지 인프라(5.1)·도래 리마인드(5.2)를 잇는 "상태변경 통지 표시 정밀화"
스토리**다. 5.1이 generic `status_change` 배너·dismiss를, 5.2가 도래 리마인드 도출·머지·독립
트리거를 세웠다. **거절(6.2)·임의취소(8.3) 시점의 통지 *생성* 배선은 후속 에픽(E6/E8)** 이라 본
스토리에 생산 트리거는 없다. 5.3은 이미 동작하는 `status_change` 배너를 **정밀 카피(룸+원래 일시+
사유)로 끌어올리고**, 표시·'확인'·독립 트리거를 테스트로 고정한다.

**KTH 확정 (2026-06-17, 본 스토리):**
> **① slot_start 표면화 + 정밀화.** `status_change` 항목에도 **원래 점유 슬롯 시각**을 라우터에서
> 합성(`reservation.slot_starts[0]` — 4.8 immutable 스냅샷, 취소/거절 후에도 보존, **추가 쿼리 0**)해
> 배너가 "{룸} {KST 날짜·시각} 예약이 거절됐어요." 처럼 **어느 예약인지 식별**하게 한다. reason별
> 카피 정밀화 + **L6/L7 손상-데이터 가드 의무 회수**(slot_start 파싱/렌더 경로를 status_change로
> 확장하므로 같은 함정 재발) + 테스트. **소형 백엔드(라우터 합성·마이그레이션 0·스키마 불변)+프론트.**
> **② 재탐색 CTA 미포함 — 정보 카피만.** 배너는 룸+시각+사유 정보만 전달한다. UJ-3 "거절 결말"의
> 재탐색은 기존 IA(예약현황·지도·목록)로 도달 — 전역 배너 안 CTA(내비게이션) 도입은 dead-button/
> 라우팅 리스크라 본 스토리 비포함(후속). [[comms-discuss-dont-collapse-decisions]]

**왜 정밀화가 실작업인가:** 5.3의 문자 그대로의 AC(거절/취소 → 1회 통지·확인 시 소멸·독립 트리거)는
5.1+5.2 인프라로 **이미 충족**된다. 5.1 forward note가 명시한 5.3의 잔여 책임은 **"상태변경 표시
정밀 카피"** 하나다. 현재 status_change 카피는 시각이 없어("예약이 거절됐어요") 예약이 여럿이면
어느 건인지 모호하다 — slot_start 표면화가 이 갭을 닫는 핵심 가치다.

## Acceptance Criteria

(epics.md Story 5.3 BDD[L924-940] + FR-18a + UJ-3 + KTH 본 스토리 확정 기반)

**AC1 — status_change 통지에 원래 슬롯 시각 표면화 (라우터 합성·추가 쿼리 0)**
**Given** `status_change` 통지 행(거절/취소·`list_pending` 반환)과 그 대상 예약(`reservation.slot_starts` 스냅샷 보유)
**When** `GET /api/v1/notifications`가 status_change 항목을 변환하면
**Then** `NotificationItem.slot_start`에 **가장 이른 원래 점유 슬롯**(`reservation.slot_starts[0]` — 생성 시 오름차순 ISO `...Z` 스냅샷, 4.8 immutable·취소/거절로 `reservation_slots` 행이 DELETE돼도 보존)을 담는다. 라우터가 이미 `session.get(Reservation)`로 들고 있는 `linked`에서 합성하므로 **추가 쿼리 0**(N+1 미증가). 손상/누락 `slot_starts`는 `None` 폴백(GET 500 금지 — AC5 L7 회수). **스키마 변경 0**(`NotificationItem.slot_start: datetime | None`은 5.2가 이미 추가 — status_change는 None→값으로 채우기만).

**AC2 — reason별 정밀 카피 (룸+KST 일시+사유·정보 전용)**
**Given** slot_start가 채워진 status_change 항목
**When** 전역 배너가 render-time 파생으로 카피를 만들면
**Then** reason별로 **룸·원래 KST 날짜·시각·사유**를 포함한다: `rejected` → "{룸} {KST 날짜} {시각} 예약이 거절됐어요.", `cancelled` → "{룸} {KST 날짜} {시각} 예약이 취소됐어요.". `slot_start` 없으면 generic 폴백("{룸} 예약이 거절됐어요."). 미지 reason은 "{룸} 예약 상태가 변경됐어요.". **KST 절대 표기**(slots 포맷터 `kstToday`+`formatDateKorean`+`formatSlotLabel` 재사용 — 상대일 "내일/오늘" 클라 재판정 금지·deferred L207). **재탐색 CTA 없음**(정보 카피만 — KTH 확정 ②). 룸 누락(`room_name=null`)도 막다른 화면 없이 카피.

**AC3 — '확인' 소멸 + 독립 트리거 (1회 통지)**
**Given** status_change 배너
**When** '확인'을 누르면
**Then** generic dismiss(`POST /api/v1/notifications/{notification_id}/dismiss` — 5.1, `dismissed_at` 설정·멱등)로 소멸하고 같은 사용자 재접속 시 다시 뜨지 않는다(1회 통지 — `UNIQUE(user_id,reservation_id,type)`가 예약당 status_change 1행 보장). 닫기 라벨은 **'확인'**(도래 리마인드의 '다시 보지 않기'와 분기). 도래 리마인드(FR-18)와 **독립 트리거**: status_change '확인'은 reservation_reminder 억제를 건드리지 않고 그 역도 같다(키·엔드포인트 분리 — 5.2). ≥44px·`aria-label`.

**AC4 — 손상-데이터 가드 의무 회수 (L6·L7)**
**Given** slot_start 파싱/렌더 경로가 status_change로 확장됨([[dev-workflow-policy-deferred-and-repeat-mistakes]] 트리거)
**When** 손상 `slot_starts`를 만나면
**Then** **백엔드(L7 회수):** 도출/합성 시 `datetime.fromisoformat` 실패(비-ISO)는 `ValueError` 전파(GET 500) 대신 안전 파싱 헬퍼가 `None` 반환 + `logger.warning`(reminder 윈도 판정은 skip·status_change slot_start는 None 폴백). reminder 경로(`reminders.py`)·status_change 경로(라우터) **양쪽** 동일 가드. **프론트(L6 회수):** 배너 카피가 `Number.isNaN(new Date(slot_start).getTime())`이면 generic 카피로 폴백(시각 없는 문장) — reminder·status_change **양쪽** 적용. `slot_start` 손상이 `InAppBannerSlot.map()`을 크래시시켜 정상 통지까지 사라지는 일 없음(막다른 화면 금지). [Source: deferred-work.md L6·L7]

**AC5 — 결정성·도메인 경계·게이트**
**Given** 도메인 경계·반복 함정·게이트
**When** 구현하면
**Then** ① `notifications.service`는 자기 테이블만(rooms/reservations import 0) — slot_start 합성은 **라우터**(`linked.slot_starts[0]`·4.8 패턴)가, 안전 파싱은 `reminders.py`/공유 헬퍼가 수행(순환 회피). ② **새 마이그레이션 0**(notifications·slot_starts 스키마 재사용)·**신규 ErrorCode 0**·**신규 의존성 0**. ③ **SDK 재생성 불요**(신규 엔드포인트·스키마 필드 변경 0 — slot_start는 5.2가 이미 노출) — 드리프트 게이트(Layer A·B) **0 확인**만. ④ 백엔드 `pytest`/`ruff`/`mypy` + 웹 `vitest`/`lint`(직접 fetch 0·set-state-in-effect 0)/`check-types`/`build` + `turbo` 전체 그린. ⑤ 라이브 Supabase head 불변(`a7c9e1b3d5f2`·`alembic upgrade head` no-op) 확인. ### 🎨 배너 카피 (UX-DR5 / AC2 — 정보 전용·CTA 없음)

- **status_change 정밀 카피(본 스토리):**
  - `rejected`+slot_start → "{룸} {KST 날짜} {시각} 예약이 거절됐어요." (예: "강남 스터디라운지 6월 20일 14:00 예약이 거절됐어요.")
