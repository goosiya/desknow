# Story 7.7: 범위 밖 질문 거절

## Story

As a 사용자,
I want 서비스 무관 질문에는 정중한 거절을 받길,
so that 챗봇이 근거 없는 답으로 오도하지 않는다 (FR-28, FR-26 분기 c, UJ-4, SM-C2).

## Acceptance Criteria

1. **Given** 일반 잡담·서비스 무관 질문 **When** 물으면 **Then** "그건 제가 도와드리기 어려운 주제예요. 스터디룸 찾기나 예약은 얼마든지 도와드릴게요!" 정중한 거절로 안내된다(분기 c). — *카피는 톤 목표이며, 판정은 강건한 부분 문자열로 한다(아래 Dev Notes §3 참조). 단, UX-DR10 마이크로카피로서 책임 전가 금지·막힘 뒤 다음 행동 제시("스터디룸 찾기나 예약은 도와드릴게요")를 반드시 유지한다.*
2. **And** 거절 시나리오가 골든셋(SM-7)으로 검증된다 — `golden_set.json`의 `out_of_scope` 분기가 실제로 단언되고, 잡담·서비스 무관 질문이 **툴을 호출하지 않고**(검색·예약 DB 모두) 거절 카피로 응답함을 확인한다.

## Dev Notes

### 이 스토리의 본질 — 프롬프트 + 테스트 전용 (범위 가드)

7.7은 **새 코드 경로를 만들지 않는다.** 거절은 react 그래프에서 모델이 **툴콜 없는 최종 AIMessage**를 내보내는 기존 "평범한 대화" 경로 그 자체다(`tools_condition` → END). 따라서:

- **변경 O:** `chatbot/prompts.py`(상수 1개 append) + 골든셋 fixture/test + 단위 테스트 1건.
- **변경 금지(불변 확인만):** `chatbot/graph.py`(노드/엣지/`_TOOLS`/`build_graph(model)` 시그니처/`MemorySaver` 싱글톤), `chatbot/tools.py`(새 툴·새 시그널 상수 금지), `chatbot/service.py`(`send_message`/`stream_message`/`get_transcript`/`_RECURSION_LIMIT`), `chatbot/router.py`(엔드포인트/스키마). **마이그레이션 없음**(라이브 `alembic upgrade head`는 no-op — [[dev-workflow-policy-live-db-migration]]), **OpenAPI/SDK 재생성 없음**(operationId 불변), **프론트 변경 없음**(거절은 평문 텍스트라 기존 `/stream`·`/messages` UI로 그대로 흐름 — 웹/모바일 패리티 영향 없음; 모바일 챗봇 UI는 "모바일 dev-build 푸시" 버킷 유지 — [[web-mobile-parity-on-changes]]).

> 이 스코핑은 7.5/7.6이 의도적으로 7.7에 넘긴 것이다: `prompts.py` 도크스트링 line 12("7.7: 범위 밖 거절 가드가 이 상수에 협력 추가된다 — 분산 금지"), `test_golden_set.py` 도크스트링/line 103-104("범위 밖(c) 버킷은 구조만 마련, 본 스토리 미단언 — 7.7 소유").

### 핵심 설계

1. **단일 출처 프롬프트 — append만, 분할 금지.** `ROOMMATE_SYSTEM_PROMPT`(`prompts.py` line 18-44)는 매 모델 호출 시 대화 맨 앞에 prepend되고 상태에 영속되지 않으므로(line 16-17), 상수를 바꿔도 과거 대화에 옛 프롬프트가 박제되지 않는다. §서비스 안내(7.5)·§예약 검색(7.6) 블록 뒤에 §범위 밖 거절(7.7) 블록을 이어 붙인다.

2. **react 툴콜 패턴 — 매 턴 무조건 검색 아님.** 모델이 질문 성격을 판단해 툴을 고른다([[chatbot-rag-tool-calling-react]]). 잡담이면 어떤 툴도 안 부르고 바로 답(거절). 두 툴 docstring은 이미 "잡담…에는 호출하지 않는다"를 명시(`tools.py`) — 7.7은 여기에 *답변 톤*(정중한 거절 카피)을 프롬프트로 못박는 것.

3. **거절 카피는 자구 완전 일치를 강제하지 않는다.** 분기 b("그건 확인이 안 돼요.")가 톤 목표였듯, 거절도 LLM 생성이라 강건한 부분 문자열(`REJECT_SIGNALS`)로 판정한다. 다만 **다음 행동 제시**("스터디룸 찾기나 예약은 도와드릴게요")는 UX-DR10 마이크로카피 원칙상 유지돼야 하므로, 시그널에 그 의도를 담은 부분 문자열을 포함시킨다.

4. **SM-7 최종 마감.** SM-7(FR-26~28 합산 정확성 지표)은 7.7에서 완성된다(7.5 노트). `out_of_scope` 분기 단언이 켜지면 골든셋이 a(근거 답변)/b(모름)/c(거절) 세 분기를 모두 검증한다.

5. **회귀 1순위 함정 — 거절 가드 과적용.** 프롬프트 가드가 *정상 범위 내* 질문(이용 규정·FAQ·룸 추천)을 거절로 오라우팅하면 7.5/7.6이 깨진다. 가드 문구에 "서비스 무관에 한정" 경계를 명시하고, Task 5에서 기존 골든셋 a/b 케이스가 여전히 green인지 확인한다.
