## Story

As a 시드 관리자,
I want 관리자 웹에서 `docs_corpus/` 디렉터리에 배치된 문서의 인제스트를 트리거하고 처리 결과(성공/스킵/실패/정리)를 확인하길,
so that 챗봇 "룸메이트"의 서비스 안내 지식을 최신으로 유지하고 어떤 문서가 실패했는지 식별한다 (FR-33).

## Acceptance Criteria

1. **(AC1 — 인제스트 트리거 → 7.2 파이프라인 적재, 동기·리포트 반환)** 시드 관리자가 인제스트를 트리거하면(`POST /api/v1/admin/ingest`, admin 가드), 서버가 **고정 디렉터리 `apps/api/docs_corpus/`**(UI 업로드 아님 — 디렉터리 배치 후 트리거)의 문서를 7.2 코어 파이프라인(`ingest_corpus`)으로 임베딩해 pgvector(`document_chunks`)에 적재한다. 처리는 **동기**로 수행되고(잡 큐·폴링 없음), 완료 시 **처리 리포트**(성공/스킵/실패/정리 + 총수)를 200 응답으로 즉시 반환한다. 라우터는 **`def`(동기) 핸들러**여야 한다 — `ingest_corpus`는 동기·블로킹(OpenAI 임베딩 + DB 쓰기)이라 `async def`로 만들면 이벤트 루프를 막는다(FastAPI가 동기 `def` 라우트를 스레드풀에서 실행 = 루프 비차단). 적재된 청크는 챗봇 문서검색(7.5 `search_service_docs`)에 즉시 반영된다.

2. **(AC2 — 재인제스트 멱등: 중복 벡터 0)** 동일 문서를 다시 트리거하면 **중복 벡터를 생성하지 않는다** — 내용 해시(sha256)가 이전 적재분과 같으면 재임베딩·재적재를 스킵(OpenAI 호출 0)하고 리포트 `skipped`에 식별된다. 내용이 바뀐 문서는 기존 청크 전량 DELETE 후 재적재(stale 벡터 0)되어 `succeeded`에 식별된다. 멱등은 7.2가 이미 구현(재구현 금지 — `ingest_corpus` 내부 해시 대조 + `(source_path, chunk_index)` UNIQUE 스키마 이중 방어).

3. **(AC3 — 부분 실패 식별 + stale 청크 정리)** 한 문서의 처리 실패가 **배치 전체를 중단시키지 않고**(문서 단위 try/except 격리·문서 단위 트랜잭션 — 7.2), 리포트 `failed: [(경로, 사유)]`로 **어떤 문서가 왜 실패했는지** 식별된다. **추가(deferred-work L344 회수): corpus에서 삭제/리네임된 문서의 stale 청크를 정리(reconcile)** 한다 — 인제스트 후 현 corpus 파일집합에 없는 `source_path`의 청크 행을 DELETE하고 리포트 `removed: [경로]`로 보고한다. orphan이 챗봇 문서검색(7.5)을 오염시키는 것을 막는다. **footgun 가드:** 현 corpus에 파일이 **하나도 없으면(present_paths 공집합) reconcile를 스킵**한다(디렉터리 부재/오설정으로 전체 청크를 wipe하는 사고 방지).

4. **(AC4 — 관리자 웹 인제스트 화면)** 관리자 웹 `/ingest` 화면(8.1이 만든 `ComingSoon` 골격을 교체)이 **인제스트 트리거 버튼**(파괴적 확인 1단계 — reconcile가 orphan을 삭제하므로)과 **결과 리포트 표시**(성공/스킵/실패/정리 개수 + 각 목록, 실패는 경로+사유)를 둔다. 트리거는 생성 SDK 함수 경유만(no-direct-fetch lint). 진행 중 `isPending` 표시, 실패 시 인라인 에러 카피("네트워크 연결이 끊겼을 수 있습니다"), 트리거 전 빈 패널은 안내 카피. 백엔드 호출은 `@/lib/api-client` 배럴 경유.

5. **(AC5 — RBAC 최종 강제 + ToolNode 에러정책 회수)** 엔드포인트는 `_require_admin` 가드: 비-admin(booker/provider) 403 `FORBIDDEN_ROLE`, 무토큰 401 `UNAUTHENTICATED`. 관리자 화면은 에러코드·숫자 상태를 노출하지 않고 고정 한국어 카피만 보인다. **추가(7.5 deferred L29 회수): 챗봇 그래프 `ToolNode`에 `handle_tool_errors` 명시 정책**을 둔다 — 툴(`search_service_docs`/`search_available_rooms`) 내부 예외(DB 단절·임베딩 실패 등)가 **에러 문자열로 모델에 재투입돼 "근거"로 오인·환각**되지 않도록, 내부 사유를 노출하지 않는 통제된 메시지로 치환하고 모델이 사용자에게 "일시적으로 검색할 수 없다"고 안내하도록 한다(동작 테스트로 검증).

> **이 스토리의 범위 결정(명시 — 매우 중요):**
> - **동기 + 리포트 반환(KTH 2026-06-18 선택):** 비동기 백그라운드+잡 폴링이 아니라, `POST /admin/ingest`가 **동기 `def` 라우터**(FastAPI 스레드풀)에서 `ingest_corpus`를 실행하고 `IngestReport`를 응답으로 즉시 반환한다. 아키텍처가 인제스트 관리 표면의 동기/비동기를 규정하지 않고(원문 부재 확인 — L202는 멱등·부분실패만), AC3 "완료되면 성공/실패 확인"이 동기 리포트에 정합하며, MVP corpus가 작아(현 README+샘플) 요청시간이 짧다. 잡 상태 영속 테이블·폴링 엔드포인트 **0**(스코프 최소). 대형 corpus의 요청 타임아웃·배치 토큰 한도는 별도 defer(아래).
> - **stale 청크 GC reconcile 포함(KTH 2026-06-18 선택):** deferred-work L344(corpus 삭제/리네임 문서 orphan 영구 잔존)를 회수한다 — 인제스트 후 현 corpus에 없는 `source_path` 청크를 DELETE하고 `IngestReport.removed`로 보고. 회고가 "8.4 관리 표면과 정면 정합"으로 지목·7.2 리뷰가 회수처를 8.4로 명시. reconcile는 `ingest_corpus`에 통합(CLI `scripts/ingest_docs.py`도 동일 동작).
> - **ToolNode 에러정책 포함(KTH 2026-06-18 선택):** 7.5 deferred L29(ToolNode `handle_tool_errors=True` 기본 → 에러 문자열이 모델에 "근거"로 재투입)를 회수한다 — 회고 "E7 하드닝 또는 8.4" 중 8.4를 택일. graph.py `ToolNode`에 명시 정책 추가. 8.4가 챗봇 운영 인접 마지막 스토리라 배포 전 정리.
> - **신규 공개 ErrorCode 0:** per-doc 실패는 리포트 내부 사유 문자열(예외 미발생)·corpus 디렉터리 부재는 빈 리포트로 흡수(404/503 신규 코드 불요). 단 **새 엔드포인트 1개**라 OpenAPI 변경 → **SDK 재생성 필수**(drift 게이트).
> - **스키마 마이그레이션 0건:** `document_chunks` 테이블·HNSW 인덱스·UNIQUE는 7.2가 생성·라이브 적용 완료. 새 테이블/컬럼/제약 없음 → 라이브 `alembic upgrade head` 신규 실행분 없음([[dev-workflow-policy-live-db-migration]]).
> - **웹/모바일 작업 0:** admin 전용 화면. `apps/web`·`apps/mobile` 무변경(admin=모바일 표면 없음 — [[web-mobile-parity-on-changes]] 비적용·E7≠모바일 버킷).
> - **defer 유지(회수 대상 아님):** 대용량 단일 문서 배치 임베딩 토큰 한도(deferred-work L343)는 "8.4/배포에 대형 문서가 들어오거나 견고성 하드닝 시" 트리거 — 현 MVP corpus엔 미발현이라 **defer 유지**(present 코퍼스가 작아 위험 0). PDF/docx 바이너리 파서·코퍼스 양방향 동기화 UI·문서 업로드·배포 시 corpus 교체는 범위 밖.

- **`ingest_corpus(store, corpus_dir, embeddings, *, chunk_size=1000, overlap=200) -> IngestReport`(`chatbot/ingest/service.py:51-120`):** 문서별 [로딩(BOM/CRLF 정규화) → sha256 해시 → 기존 해시 대조 → 동일=skip / 변경·신규=청크(공백 제거)→임베딩→DELETE 후 INSERT] + 문서 단위 try/except 격리 + 실패 전 `store.rollback()`(세션 abort 연쇄 차단 — 7.2 리뷰 patch c). **8.4는 이 함수에 reconcile 단계만 더해 호출** — 멱등·부분실패·원자성·격리를 공짜로 얻는다.
- **`IngestReport`(`:31-48`):** `succeeded: list[str]`·`skipped: list[str]`·`failed: list[tuple[str,str]]` + `total` property. 8.4는 `removed: list[str]` 추가.
- **`DocumentChunkStore` Protocol + `SqlDocumentChunkStore`(`store.py:24-92`):** `get_content_hash`·`replace_document_chunks`(빈 청크 no-op 가드·단일 트랜잭션 DELETE+INSERT·실패 rollback)·`rollback`. 8.4는 `delete_orphans(present) -> list[str]` 추가(distinct source_path 차집합 DELETE·자체 트랜잭션·실패 rollback). **빈 청크 no-op 가드(`:73-74`)는 7.2 리뷰가 "8.4 재사용 방어 목적"으로 선제 적용** — 이미 보호됨.
- **`build_embedder() -> OpenAIEmbeddings`(`embedding.py:36-51`):** 인자 0·`settings.EMBEDDING_MODEL`/`OPENAI_API_KEY` 명시 전달. import는 함수 내부(모듈 import만으로 OpenAI 미적재). 8.4 service가 그대로 호출.

### ★알려진 함정 (구현 전 반드시 확인 — 프리플라이트)

1. **`async def` 라우터 금지(★이벤트 루프 차단):** `ingest_corpus`·`build_embedder`(OpenAI HTTP)·`SqlDocumentChunkStore`(동기 psycopg3)는 전부 블로킹. FastAPI는 **동기 `def` 라우트를 스레드풀에서 실행**해 루프 비차단 — `async def`로 만들면 블로킹이 루프를 직접 막아 전 서버 정지. 반드시 `def`.
2. **reconcile 빈 present_paths wipe footgun(★):** reconcile가 "현 corpus에 없는 source_path 삭제"라, corpus 디렉터리가 비었거나 오설정이면 present_paths가 공집합 → **전체 청크가 orphan으로 삭제**된다. **`if reconcile and present_paths:` 가드로 빈 집합이면 reconcile 스킵.** (현 docs_corpus엔 README.md+샘플이 항상 있어 정상 경로엔 미발현이나, 방어 필수.)
3. **reconcile 트랜잭션·best-effort:** `delete_orphans`는 per-doc 루프 종료 **후** 자체 트랜잭션 DELETE+commit. per-doc 적재는 이미 commit됐으므로 reconcile 실패가 성공 적재를 롤백하면 안 됨 — `try/except: store.rollback()` best-effort(removed 빈 채로 진행·로깅). 전체 리포트를 죽이지 말 것.
4. **ingest_corpus 시그니처 변경 파급:** `reconcile` 파라미터 + `delete_orphans` Protocol 메서드 추가 → **FakeStore(테스트)·SqlStore·기존 7.2 테스트 동반 갱신** 필수(Protocol 미구현 시 mypy/런타임 실패).
5. **corpus 경로 settings 금지:** 신규 settings 필드는 `_assert_key_lists_match_model`(`config.py:189-207`)이 REQUIRED/OPTIONAL 동기화를 import-time 강제 → 누락 시 RuntimeError. **ingest 패키지 모듈 상수 `DEFAULT_CORPUS_DIR`로**(scripts와 공유·DRY).
6. **ToolNode 정책이 "근거" 오인 방지:** `handle_tool_errors`를 단순 에러 원문 반환으로 두면 여전히 모델이 근거로 오인. 통제된 메시지(내부 사유 미노출·"도구 오류"임을 명시)로 치환하고 **동작 테스트로 검증**.
7. **신규 ErrorCode 0이지만 SDK 재생성은 필수:** per-doc 실패는 본문 사유(예외 미발생)라 ErrorCode 추가 없음. 그러나 **새 엔드포인트 1개 = OpenAPI path 추가**라 Layer A/B drift 게이트가 잡음 → `export_openapi.py` + SDK `generate` 필수(스킵 시 게이트 적색).
8. **golden set teardown 파괴적:** `test_golden_set.py`는 document_chunks 전체 DELETE teardown — 전용 `TEST_DATABASE_URL` 게이트(미설정 시 skip)라 운영/dev DB 미겨냥([[chatbot-golden-set-destructive-teardown]]). 8.4 게이트 실행이 이를 트리거하지 않도록 키 미설정 유지.

### ★stale 청크 reconcile — 정확한 메커니즘 (deferred-work L344 회수)

- **기존 버그(7.2 defer L344):** 멱등 대조·삭제가 `source_path` 기준이라, 파일을 리네임하거나 corpus에서 삭제하면 **옛 경로의 청크 행이 DB에 영구 잔존**(orphan). 챗봇 문서검색(7.5 `search_service_docs`)이 그 orphan을 근거로 회수해 **삭제된 안내를 환각**할 수 있다.
- **본 스토리 해법:** `ingest_corpus`가 문서 루프에서 **존재한 corpus 파일의 rel_path를 `present_paths`에 수집**(성공/스킵/실패 무관 — 파일이 있으면 orphan 아님), 루프 후 `store.delete_orphans(present_paths)`로 **DB distinct source_path − present_paths = orphan**을 DELETE하고 `report.removed`로 보고. CLI(`scripts/ingest_docs.py`)도 동일 동작.
- **가드:** present_paths 공집합이면 스킵(함정 #2). reconcile는 best-effort(함정 #3).
- **재-defer 금지:** L344는 KTH가 8.4 회수로 선택했으므로 본 스토리에서 해소([[dev-workflow-policy-deferred-and-repeat-mistakes]]).

### ★ToolNode 에러정책 — 정확한 메커니즘 (7.5 deferred L29 회수)

- **기존 버그(7.5 defer L29):** `graph.py:98` `ToolNode(_TOOLS)`는 기본 `handle_tool_errors=True`라 툴(`search_service_docs`/`search_available_rooms`) 내부 예외(DB 단절·`get_engine` 설정 오류·임베딩 실패)를 **삼켜 에러 문자열을 ToolMessage로 모델에 재투입** → 모델이 에러 문자열을 그라운딩 근거로 오인해 엉뚱한 답변/환각.
