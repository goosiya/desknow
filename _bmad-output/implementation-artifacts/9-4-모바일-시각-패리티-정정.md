## Story

As a 예약자/제공자,
I want 모바일(Expo/RN) 전 화면이 진짜 웹앱과 시각·인터랙션까지 동등해지길,
so that 9.1~9.3이 가렸던 앱 셸·커스텀 컴포넌트·온보딩 드리프트가 제거되어 웹·앱 이질감이 사라진다 (FR-18·NFR-4 / audit 2026-06-19 정정).

> **이 스토리는 신규 기능이 아니라 "거짓 done" 정정이다.** 9.1~9.3은 기능을 RN으로 포팅했으나, 검증 당시 `:3000`·`:3001`이 **둘 다 Expo**였고 진짜 Next 웹앱은 어느 포트에도 떠 있지 않았다 → 과거 "패리티 통과"가 **모바일↔모바일 비교**라 셸/입력컨트롤/온보딩 드리프트가 적출되지 않았다. 본 세션에 웹을 `:3002`로 세워 전수 대조한 결과가 입력 audit이다. **백엔드/SDK/웹/admin은 무변경(diff 0) — 순수 모바일 클라이언트 작업**이다. 입력 = `_bmad-output/implementation-artifacts/parity-audit-web-vs-mobile-2026-06-19.md`(증거 `parity-audit-2026-06-19-shots/`), 정정 근거 = `_bmad-output/planning-artifacts/sprint-change-proposal-2026-06-19-visual-parity.md`.

## 범위 결정 (KTH 확정 2026-06-19 — correct-course)

1. **웹 = 정본. 드리프트는 모바일을 웹에 맞춰 정정한다.** done인 웹 코드(AppShell·AppNav·OnboardingOverlay 등)는 **무수정**. 9.1~9.3 스토리 본문·기능 AC도 무수정(재해석만). Epic 9는 `in-progress` 유지.
2. **[SYS-1·2 방향·KTH 확정] 인증화면(로그인·가입)에도 모바일 셸 추가.** 웹은 루트 `app/layout.tsx`가 **모든 페이지를 `AppShell`로 감싸** `/login`·`/signup`에도 글로벌 헤더 + booker 하단탭이 뜨는 **의도된 구조**(웹 버그 아님). → 모바일도 동일하게: 글로벌 헤더 추가 + 인증화면에 booker 하단탭 노출. (대안 "양쪽 셸 제거"는 기각 — done 웹 무수정.)
3. **[온보딩 닫기 정책·웹=정본이 모바일 확정③을 갱신] 모바일 온보딩의 "어떤 닫힘이든 영속(dismiss)"(확정③)을 웹 정책으로 교체한다.** 웹(2026-06-19): **"다시 보지 않기"만 영속 dismiss**, 시작하기/X/Esc/바깥 탭 = **비영속 close**(다음 방문 재노출). ONB-3가 S1 차단이고 KTH가 웹=정본을 확정했으므로 이 정책 변경은 본 스토리 범위다(과거 확정③ supersede).
4. **[audit 보정 — provider 탭 라벨] 웹 실제 라벨은 "내 스터디룸"이다.** audit ROOM-3("웹=스터디룸 vs 모바일=내 스터디룸")은 **오독**. 정본 코드 `apps/web/src/components/shell/AppNav.tsx:29` `PROVIDER_NAV`의 라벨이 "내 스터디룸"이고 모바일도 이미 "내 스터디룸" → **라벨은 이미 일치, 순서만 정정**(SYS-4). 라벨을 "스터디룸"으로 바꾸지 말 것.
5. **[실기기 재검증 = ⚠️사람-확인-필수·차단 AC 아님] 본 스토리 검증은 웹 vs Expo Web 한정.** iOS/Android 네이티브(시뮬레이터/실기기·EAS dev build)는 폰트·그림자·터치 하이라이트·세이프에어리어·키보드 회피에서 더 벌어질 수 있다 → AC6까지 Expo Web 패리티로 done, 네이티브 재검증은 배포 체크리스트 의무 라인(자동 빌드/푸시로 안 잡힘).
6. **단위 테스트 러너 미도입(9.1~9.3 패리티).** 검증 = typecheck(`pnpm --filter mobile check-types`) + lint(`expo lint`) + Playwright(MCP) Expo Web 실검증 + **화면별 웹↔모바일 스크린샷 diff 첨부**. 신규 테스트 인프라 도입 금지.

## Dev Notes

### 이 스토리의 본질

9.4는 **9.1~9.3의 "거짓 done"을 정정**한다. 기능은 이미 RN에 포팅됐고(콘텐츠=카드·폼·뱃지는 잘 옮겨짐), 드리프트는 세 갈래에만 집중한다 — ① **앱 셸**(글로벌 헤더 유무·인증화면 하단탭), ② **커스텀 컴포넌트**(시간 입력·룸형태 세그먼트·FAB 아이콘), ③ **미완 구현**(온보딩 4슬라이드 중 1장). 따라서 대부분 작업은 **시각/레이아웃 정렬 + 1건의 구현 완성(온보딩)** 이고, **데이터 훅·SDK·검증 로직·카피는 건드리지 않는다.** 가장 위험한 단일 작업은 **Task 1·2(앱 셸 구조 변경)** — 모바일 루트가 `headerShown:false` Stack + `(tabs)` NativeTabs 구조라, 글로벌 헤더와 인증화면 하단탭을 넣을 때 **기존 탭 네비·FAB/배너 z-순서·역할 스왑을 깨지 않게** 해야 한다. NativeTabs는 임의 상단 헤더/비-탭 라우트를 호스팅 못 하는 한계가 있으니 헤더/하단탭 표출 방식을 먼저 검증할 것.
