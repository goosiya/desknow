## 1. 새 영역 패턴 정리 (focus ①)

Epic 5는 1.x~4.x의 확립 패턴을 **새 도메인 3종에 재사용**하면서, 새 영역 고유 패턴을 추가로 확립했다.

### 1a. 기존 패턴의 검증된 재사용
- **user-scoped 도메인 = favorites 미러:** notifications가 favorites의 (멱등·소유권 404·FakeSession 단위 + TestClient 통합) 구조를 그대로 미러. 신규 도메인을 0에서 설계하지 않고 검증된 골격에 얹음.
- **TanStack 독립 키 + 옵티미스틱 + 정확 키 invalidate:** `useDismissNotification`/`useDismissReminder`가 `useToggleFavorite` 선례를 미러(광역 invalidate 금지 = 반복함정 #1 준수).
- **시각 규약:** UTC 저장(`now_utc`)·`...Z` 전송(`isoformat_utc`)·`fromisoformat` 파싱(3.12 aware) 일관 적용.

### 1b. Epic 5에서 새로 확립한 재사용 자산
| 패턴 | 구현 | 비고 |
|------|------|------|
| **외부 JS SDK 로더** (injectScript + load/error 양쪽 `script.remove()` + SRI `integrity`) | `kakao-share.ts` | maps 로더(3.2)의 누수를 5.4가 회수하며 share 로더는 처음부터 내장 — 같은 함정 재발 차단 |
| **순수 함수 시간 윈도 판정** (timedelta 정수 비교·부동소수 금지·naive 거부) | `is_within_reminder_window` | 4.7 `_CANCEL_WINDOW` 선례 계승·경계 테스트 6종 고정 |
| **손상 데이터 fail-safe 헬퍼** (포맷 실패→`None`/generic 폴백, throw 금지) | FE `formatKstDateTime` · BE `earliest_slot_start` | 손상 1건이 배너/조회 전체를 죽이지 않음 |
| **도메인 경계 격리** (notifications.service가 rooms/reservations import 0; 합성은 router/reminders) | notifications 도메인 | 결합도 억제 — 통지가 예약/룸 내부에 의존하지 않음 |

**관찰:** 새 영역이어도 "미러 → 고유 패턴 추가" 흐름이 잘 작동. Epic 1 강점("패턴 확립 후 재사용")이 도메인 경계를 넘어 유지됨.

## 2. deferred-work 트리아지 (focus ②)

### 2a. Epic 5 중 의무 회수 작동 (A3 정책 검증) ✅
Epic 1 회고의 A3("트리거 시 의무 회수, 재-defer 금지")가 실제로 동작:
- **L103 24h 경계 부동소수** → 5.2가 `timedelta` 정수 비교로 회수 (4.7 6h·5.2 24h 동시 표기).
- **L6/L7 손상 slot_start → 배너 크래시 / GET 500** → 5.3이 status_change 확장하며 의무 회수(공유 가드 + naive까지 동일 손상 클래스로 통일).
- **maps `injectScript` dead `<script>` 누수(3.2)** → 5.4가 회수.
- 이전 부분 회수(RoomSheet stale badge=3.3, 무제한 스캔→지역/반경 필터=3.4/3.5)도 정합 유지.

### 2b. 신규 deferred 트리아지 — 정상 이관 (유지 OK)
- **모바일 버킷:** 후기 UI(5.5)·카톡 공유(5.4) — 모바일 예약/상세가 Epic 4 placeholder라 진입점 없음. BE는 공용 엔드포인트라 재구현 불요. "모바일 dev-build 푸시" 버킷. [[web-mobile-parity-on-changes]]
- **배포 버킷:** 후기 공개조회 페이지네이션·레이트리밋 / 카카오 콘솔 프로드 도메인 등록. [[deployment-first-time-checklist]]
- **MVP 스코프 밖:** 후기 수정/삭제·작성자 표시명(users 표시명 컬럼 선행 필요)·익명 정책(KTH 결정).

### 2c. ★중대 — Epic 6에서 트리거되는 미회수 부채 (6.x create-story 시 의무 편입)
| 회수 항목 | 출처 | Epic 6 트리거 |
|-----------|------|---------------|
| **cancel↔reject 교차 race** (둘 다 stale `confirmed` 읽고 last-write-wins → 종료 상태/통지 의미 비결정) | 4.7·4.1 defer | **6.2 예약 거절** = 거절 엔드포인트 최초 도입 → 정면 트리거. `SELECT FOR UPDATE`/조건부 UPDATE(`WHERE status='confirmed'`)/버전 컬럼 중 택1 또는 last-write-wins 명시 수용 + 테스트 |
| **`confirmed_slot_starts` reader status 필터 부재** (DELETE-on-cancel 불변식에만 의존, 비-DELETE 종료 경로 추가 시 유령 점유 누출) | 4.9 defer | 6.2 거절이 비-DELETE 종료 경로면 방어심층 필요 |

### 2d. 누적 추세 패턴 (관찰)
- **옵티미스틱 동시 dismiss/toggle stale `previous` 클로버**가 이제 **3회 재발**(3.7 favorites · 5.1 · 5.2). 매번 "깜빡임 수준·`onSettled` 수렴"으로 defer. 진짜 결함은 아니나 **반복 신호** → 동일 키 뮤테이션 `scope` 직렬화 또는 함수형 롤백(설계 변경 규모)을 한 번에 정리할 후보. **트리거 임계 도달 시 전용 정리 스토리** 권장.

## 3. 코드리뷰 반복 지적 (focus ③)

### 3a. 프리플라이트 정착 ✅
Epic 1 A1("반복 함정 프리플라이트")이 **5-1~5-6 전 스토리 Dev Notes에 고정 섹션으로 존재**. 차단성 리뷰 지적 0 — 프로세스 개선이 형식이 아니라 실효.

### 3b. 적대적 리뷰의 반복 오탐 (noise 패턴)
- 5.2 Blind Hunter: `fromisoformat`이 "Z" 거부(**BLOCKER** 주장) → 오탐(3.12 aware 파싱 지원, 실행 확인). 옵티미스틱 `reservation_id` 형식 불일치(MED) → 오탐(동일 SDK 출처 + onSettled 수렴).
- **패턴:** 컨텍스트 0 Blind Hunter가 "이미 처리된 것"에 높은 심각도 오탐을 반복 생성. 검증·기각 비용은 낮으나, 리뷰 결과를 그대로 받지 말고 **런타임/실측으로 확인**하는 규율이 매번 필요(A1 ④ "docstring 아닌 소스/실측"과 동형).

### 3c. docstring → OpenAPI description 드리프트 함정 (신규 인지)
- 5.3: AC5③이 "SDK 재생성 불요"라 단언했으나, **클래스 docstring이 OpenAPI `description`으로 export**되어 Layer A 드리프트 발생 → 재생성 강제. 향후 docstring 수정도 동일 드리프트 유발 → **패턴으로 인지**(deferred-work에 AC 문구 보정 항목 기록됨).

### 3d. FE/BE 손상-관용 비대칭 (반복 테마)
- 손상 입력 가드가 한쪽(보통 BE)에만 있고 다른 쪽이 비대칭인 패턴 반복: `formatKstDateTime`(5.3 회수)·`formatReviewDate` 미가드(5.5 defer)·후기 손상 슬롯 409 루프(5.5 defer). 매 스토리 사후 대응 중 → **공유 포맷터/가드 승격**이 근본 정리 후보.
