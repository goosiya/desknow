## 1. 새 영역 패턴 정리 (focus ① — 이번 회고 최대 비중)

Epic 7은 desknow 최초의 **LLM/에이전트/RAG/스트리밍 도메인**이다. CRUD 미러가 통하지 않는 새 영역이라, "재발명 금지 + 페이크 주입 단위검증 + 단일 출처"라는 E1 강점을 새 패러다임에 이식하며 고유 자산을 다수 확립했다.

### 1b. E1~E6 강점의 새 영역 이식
- **재발명 금지**가 LLM 영역에서도 핵심: 7.1이 "프로바이더별 httpx 손코딩 = 1순위 disaster"를 명시 차단하고 LangChain v1 통합 인터페이스에 위임. 7.6 regions 역인덱스도 `legal_dong.json` 단일 출처 재사용(새 데이터 파일 0). [[region-code-legal-not-admin-dong]]
- **도메인 경계 격리**: 예약검색 툴(7.6)이 raw SQL이 아니라 `rooms.service`(내부적으로 4.9 예약차감) 경유 → 신선 반영·서비스 계층 단방향 유지(E5 notifications 경계와 동형).
- **손상 fail-safe·신호 폴백**: 7.5/7.6의 "모름 신호"가 E5 `formatKstDateTime`/`earliest_slot_start` fail-safe 정신의 LLM판.

**관찰:** 새 패러다임에서도 "검증된 골격(주입·단일출처·경계격리·재발명금지) → 영역 고유 패턴 추가" 흐름이 작동. **신규 도메인을 0에서 설계하지 않고 E1 원칙을 이식한 게 이번 에픽의 최대 강점.** 단 react 2-패스는 첫 토큰 지연이라는 새 트레이드오프를 안음(7.4 ≤2초 SLA 대비 인지·기준 OpenAI로 리스크 격리).

## 2. deferred 트리아지 + 의무회수 검증 (focus ②)

### 2b. ★LangGraph 실패 turn 입력 롤백 — 추적 체인 종료 ✅ (메모 항목 정리)
- **부채 추적([[langgraph-failed-turn-input-rollback]]):** 7.3에서 발견(노드 실패 전 입력 HumanMessage가 영속 → 재전송 시 user 메시지 **중복 누적**, AC4 무결성 위반). 7.3 patch가 `RemoveMessage`+`update_state` 롤백으로 해결. 메모가 "7.4~7.7 동일 그래프 재사용 시 재확인" 의무를 등재.
- **체인 검증:**
  - **7.4(스트리밍):** `stream_message`가 `aupdate_state(RemoveMessage)`로 7.3과 **동형 롤백** — 첫 토큰 전/스트림 중 실패 양쪽 회귀 테스트(`test_failed_turn_mid_stream_rolled_back`=`transcript==[]`, 부분 영속 없음).
  - **7.5/7.6:** react 추가가 롤백 범위를 **넓히지 않음**(7.3/7.4 동형 유지 명시).
  - **7.7(마지막 확인 지점):** 거절은 정상 turn(노드 실패 아님)이라 입력 롤백 로직과 무충돌 — `send_message` 1회·transcript 정상 적재 단위 확인. **→ 추적 체인 종료.**
- **잔존(별개 결정·계속 deferred):** 백엔드 롤백이 `native_exceptions`(LLM 업스트림)만 커버 → 비-LLM 예외·`asyncio.CancelledError`·checkpointer 오류 시 **고아 HumanMessage**. 이건 "코드 버그를 502로 위장 안 함" 의도적 미러(동기 `send_message`와 공유) → 롤백 트리거 전면 확장은 7.3+7.4 공유 별도 결정으로 유지. **신규 결함 아님.**

## 3. 코드리뷰 반복 지적 (focus ③)

### 3a. 프리플라이트 정착 ✅
Epic 1 A1("반복 함정 프리플라이트")이 7-1~7-7 Dev Notes에 고정 섹션으로 존재(Gemini `model_provider` 추론·Google env 이름 불일치·config 가드·env.py import 허브·그래프 싱글톤·react 빈 tool-call transcript 누출 등). 새 영역 footgun이 사전 문서화돼 **차단성 잔존 0**.

### 3b. ★decision-needed가 급증 — 새 영역의 자연 신호 (E6 대비 대조)
- E6는 decision-needed 0·patch 0(성숙 CRUD)이었으나 **E7은 거의 매 스토리 decision-needed**(7.1·7.5 각 2·7.6 3·7.7 2·7.2 1). **전부 KTH "지금 수정" 확정 → patch 즉시 해소·게이트 재그린.**
- **해석:** 결함 증가가 아니라 **새 영역이라 견고성 결정점이 더 많이 표면화**(팩토리 인자 정합·react recursion 가드·예약툴 N+1·지역 해석 견고성·linkify 경계·골든셋 분기 디스패치 else 가드). 적대적 리뷰가 새 패러다임의 미정 경계를 잘 끌어냄 → 리뷰 가치 높음. 단 **"defer가 아니라 즉시 하드닝"으로 처리한 비율이 높은 게 E7 특징**(미성숙 영역이라 미루면 7.6/7.7이 깨질 위험).

### 3c. 적대적 리뷰 오탐(noise) 패턴 지속
- 7.5 Blind Hunter "세션 종료 후 ORM 직렬화 = Critical" → 리뷰가 SQLAlchemy 2.0+SQLModel로 재현해 **현재 안전 확인·기각**. E5 3b·E6 4b("컨텍스트 0 고심각도 오탐")와 동형 — **리뷰 결과를 그대로 받지 말고 런타임/구조 재현으로 확인하는 규율이 매번 필요**(A1 ④). dismiss 다수(7.5 12·7.6 다수)도 같은 계열.

### 3d. Completion "게이트 그린" 자가보고와 리뷰 실측 불일치 (신규 인지)
- 7.5 리뷰가 ruff `F401`(미사용 `END` import)을 잡음 — Completion Notes의 "ruff 통과" 자가보고와 어긋남. 7.2도 mypy 2건·테스트 계산 오류를 리뷰/dev 중 발견. **자가보고 게이트 신뢰 대신 리뷰가 독립 실측하는 게 실효**(A2 테스트 충실도와 동형). 저심각이나 패턴으로 인지.

### 3e. 골든셋 시그널셋 협소 — 테스트 충실도 갭 (반복 테마)
- 거절/모름/그라운딩 판정이 **부분 문자열 시그널**에 의존해 거짓음성·과광폭·false-green이 7.5→7.7에 반복(2d 파생). 결정적 단위 테스트는 통과하나 **라이브 LLM 출력 변이를 못 덮음**. E5 3d("FE/BE 손상-관용 비대칭")처럼 "테스트가 실 변이를 못 잡는" 충실도 갭 — 전용 TEST DB로 라이브 게이트 켤 때 일괄 정밀화 후보.
