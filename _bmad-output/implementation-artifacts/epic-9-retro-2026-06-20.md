## 잘된 점 (Keep)

- **웹=정본 원칙으로 드리프트를 일방향 정정** — 모바일을 웹에 맞추고 done 웹 코드는 무수정(KTH 지시 예외 3파일만). 양쪽이 서로 끌어당기는 혼란 방지.
- **게이트 독립 재실행 + 코드 직접 대조** — 코드리뷰에서 스토리 주장(typecheck/lint·CORS 되돌림·E2E 게이팅)을 신뢰하지 않고 직접 재실행/grep 검증. 9.4에서 Hunter 발견 다수가 오탐/기존이슈로 판명됨.
- **Playwright 실검증으로 런타임 동작 확인** — 9.4 AuthBottomNav 역할 분기를 타입/미러 논증에 그치지 않고 provider 세션 주입으로 실제 운영탭 전환까지 확인.

## 잘못된 점 / 핵심 학습 (Root cause)

- **★ 거짓 done의 근본 원인 = 포트 정체 미확인.** 9.1~9.3 검증 당시 `:3000`·`:3001`이 **둘 다 Expo**였고 진짜 Next 웹앱은 어느 포트에도 안 떠 있었다 → "패리티 통과"가 실은 **모바일↔모바일 비교**라 셸/입력컨트롤/온보딩 드리프트가 적출되지 않았다. → 메모리 [[parity-harness-web-on-3002-port-identity]] 등재.
- **빈 화면만 비교했다.** 데이터 채워진 상태·인터랙션 결과·1회성/게이팅 화면을 쌍으로 안 봐서 누락 → 메모리 [[parity-audit-compare-populated-not-just-empty]].
- **플랫폼 분기 파일 누락.** `.tsx`(네이티브)만 고치고 `.web.tsx`(rn-web)를 놓쳐 Expo Web 검증에서 옛 동작이 남음(9.4 app-tabs 실발생) → 메모리 [[mobile-platform-split-web-variant-trap]].
