### ★ provider 웹 표면 전체 구축 (idea.md L30~39 회복)
**배경:** idea.md는 제공자도 웹 사용자(L56 "사용자(고객/스터디룸제공자)용 반응형 웹")로 명시했고
가입·스터디룸 설정(주소 포함)·예약자 현황(거부)·후기 답글을 요구했다. 그러나 구현은 "웹=booker
전용, provider=API-only"로 축소돼 provider가 룸을 등록할 화면 자체가 없었다(그래서 룸을 스크립트로
시드해야 했음). **백엔드 API는 거의 다 있었고 웹 UI만 없던 상태** → UI 배선 + 백엔드 구멍 1개(주소
컬럼)만 메웠다.

구현분(전부 타입체크·lint·web 356 테스트·SDK 드리프트 클린):
- **0. Room.address 백엔드** — `Room.address`(varchar 300, nullable) 컬럼 추가. 마이그레이션
  `f1ba2a78986a_add_rooms_address_column.py`(★autogenerate가 끼워넣은 HNSW 인덱스 drop 제거함 —
  메모리 [[alembic-env-import-hub-autogenerate-trap]]). **dev Supabase에 `alembic upgrade head` 적용
  완료**(head=f1ba2a78986a, HNSW 인덱스 보존 확인). create/update 저장·`RoomSummary`/`ProviderRoomDetail`
  노출. SDK 재생성.
- **바텀시트(RoomSheet)** — (1) **주소 표시**(가격/영업시간 아래, 핀 아이콘), (2) **콘텐츠 맞춤 높이**:
  snapPoints `[0.4,1]` 제거 → 콘텐츠에 맞춰 한 번에 열려 "상세 보기"까지 다 보임(넘치면 내부 스크롤,
  `max-h-[88dvh]`). 모바일에서 핀 클릭 시 정보 잘림 해소.
- **1. 가입 역할 선택** — `SignupView`에 예약자/제공자 토글(`AuthForm` topSlot). `useRegister(role)`.
  provider 가입 시 `/provider/room`으로 라우팅.
- **2. 스터디룸 등록/수정 폼** — `/provider/room`(`RoomForm`). 이름·**주소 지오코딩 검색**(roomsGeocodeAddress
  — 법정동 b_code 없는 도로명-only 결과는 선택 불가로 필터)·수용·금액·룸형태·부대시설·요일별 영업시간.
  `GET /rooms/mine`(신규, `ProviderRoomDetail`)로 prefill → 보유 룸 있으면 수정, 없으면 생성.
  **브라우저 E2E 검증 완료**(provider 가입→폼→주소검색→등록→실 룸 생성→재방문 수정모드).
- **3. 예약자 현황 + 거부** — `/provider/reservations`(`ProviderReservations`).
  reservationsListProviderReservations(예약자 익명 라벨) + 인라인 확인 거부(reservationsRejectReservation
  — 슬롯 재활성·통지는 백엔드). **빈 상태만 렌더 검증**(실 예약 거부는 미검증).
- **4. 후기 + 답글** — `/provider/reviews`(`ProviderReviews`). 내 룸 후기 목록 + 답글
  (reviewsCreateReply). **빈 상태만 렌더 검증**(실 답글 미검증).
- **5. 역할별 네비** — `AppNav`(client) 신설, `AppShell`이 사용. provider 로그인 시 헤더/하단 네비가
  **내 스터디룸·예약자 현황·후기**로 전환(예약자/미로그인은 기존 탐색 메뉴). `/provider` 랜딩도 추가.

신규 파일: `apps/web/src/features/provider/{useProviderRoom,RoomForm,useProviderReservations,
ProviderReservations,useProviderReviews,ProviderReviews}.tsx?`, `apps/web/src/app/provider/{page,
room/page,reservations/page,reviews/page}.tsx`, `apps/web/src/components/shell/AppNav.tsx`.
백엔드: `Room.address`, `ProviderRoomDetail`, `service.get_my_room`, `GET /rooms/mine`.
