- **PRD 커버리지:** **완전 승계.** FR-13 "비연속 선택은 허용되지 않는다(D1)", §6.2 Out of Scope "비연속 다중 슬롯 예약", §8 향후 고도화, PRD 애드덤 D1 표(채택=연속 구간만, 대안=비연속, 근거 명시). 브리프가 "PRD에서 결정"으로 남긴 ❓를 PRD가 명시적으로 **결정 완료**(불허)로 닫음.

### GAP-7. UX 수치 목표 → PRD 검증 기준 승계 — 승계됨 ✅
- **소스:** "첫 지도 표시 ≤2초 · 단독 예약 ≤3스텝 · 챗봇 첫 토큰 ≤2초 · 주요 상호작용 응답 ≤100ms. 경험 지표(첫 예약 ≤3분, 완주율 ≥80%, 제공자 등록 ≤5분)는 소규모 사용성 테스트(5명 내외)로 측정."
- **PRD 커버리지:** **완전 승계.** 네 수치 목표 모두 §10 Cross-Cutting NFRs + SM-6에 검증 기준화. 경험 지표 4종 모두 SM-1(≤3분)/SM-2(≤3스텝)/SM-3(≥80%)/SM-5(≤5분)로 SM화. "5명 내외 사용성 테스트" 측정 방법도 §7 도입부·§7 SM-3에 보존.
- **심각도:** 없음(모범 승계).

### GAP-8. 아키텍처 방향 — 부분 승계 ⚠️ (distortion / 일부 누락)
- **소스 (5개 항목):**
  1. FastAPI 모듈러 모놀리스 + 챗봇 같은 레포·별도 서비스 배포(스트리밍 부하 격리)
  2. Turborepo + pnpm 모노레포(사용자 웹/관리자 웹/RN)+FastAPI, OpenAPI → 타입 안전 TS SDK 자동생성
  3. 관리자 웹 별도 Next.js 앱 분리(공격 표면 축소). RBAC 최종 강제는 FastAPI.
  4. **인증: PyJWT + pwdlib(Argon2), refresh 무효화 저장소(Redis/DB), 웹=쿠키·앱=헤더 이원 추출**
- **PRD 커버리지:**
  - ①②③ → PRD 애드덤 "아키텍처 방향"에 모듈러 모놀리스 / Turborepo+pnpm / OpenAPI→TS SDK / 관리자 별도 Next.js 분리 모두 보존. ✅
  - ④ 인증 항목 → PRD 애드덤에서 "아키텍처 방향" 섹션이 아니라 **별도 "인증/세션" 섹션**으로 재배치되었으나, PyJWT+pwdlib(Argon2)·refresh 무효화 저장소(Redis/DB, §8 Q2)·웹=쿠키/앱=헤더 이원화·RBAC 백엔드 최종강제 **모두 보존**. ✅ (재배치는 distortion 아님 — 오히려 인증을 더 자세히 풀어냄.)
  - **⚠️ 누락 후보:** 소스 ①의 **"챗봇을 같은 레포·별도 서비스로 배포"**라는 배포 토폴로지 결정은 PRD 애드덤 아키텍처 항목엔 보존되나, §11 Constraints 본문은 "초기엔 챗봇을 API에 **통합**했다가 부하 시 분리"라는 **단계적 통합** 접근(브리프 기술결정 #2)을 말함. 즉 브리프 안에서 "별도 서비스 배포"(아키텍처 권고) vs "초기 통합 후 분리"(비용 기술결정)가 **긴장 관계**인데, PRD가 둘 다 옮기면서 이 긴장을 명시적으로 해소하지 않음. 아키텍처 단계에서 혼선 가능.
- **심각도:** **낮음.** 모든 항목이 어딘가에 보존돼 있어 정보 손실은 아님. 단 "초기 통합 vs 별도 배포" 긴장은 아키텍처 단계에서 결정 필요 — PRD가 명시적 Open Question으로 올리지 않은 점이 약점.

## 결론 및 권고

- **치명/높음 갭: 0건.** 브리프 애드덤이 "PRD로 반드시 넘기라"고 지목한 7대 결정(비연속 슬롯·동시성·카카오맵·NIST 충돌·LLM 파라미터 비대칭·UX 수치·아키텍처 방향)은 모두 PRD 또는 PRD 애드덤으로 정확히 승계됨. 다수는 모범적 승계(결정을 PRD에서 명시적으로 닫음).
- **유일한 약점(낮음, GAP-8/9):** "챗봇 = 초기 API 통합 후 부하 시 분리"(비용 관점)와 "챗봇 = 같은 레포 별도 서비스 배포"(아키텍처 부하격리 관점)의 **긴장**이 PRD에 둘 다 실렸으나 해소 책임이 명시되지 않음. **권고:** 아키텍처 단계 입력으로 "챗봇 배포 토폴로지(초기 통합 vs 별도 서비스) 확정"을 §8 Open Questions에 1줄 추가 검토.
- 그 외는 조치 불필요.
