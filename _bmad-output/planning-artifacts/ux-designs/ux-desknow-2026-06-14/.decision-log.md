## 소스에서 이미 확정된 사항 (UX 기반 사실)

### 폼팩터 / 플랫폼 (multi-surface)
- **사용자 웹** — 반응형 웹 (Next.js), 예약자·제공자 공용
- **모바일 앱** — React Native (Expo), 예약자·제공자 공용
- **관리자 웹** — 별도 반응형 웹 (Next.js), 시드 관리자 전용
- 단일 FastAPI 백엔드, OpenAPI 계약 → TS SDK 자동생성

### UI 시스템
- **shadcn 기반 디자인 토큰** (간격·색·타이포) — 웹/앱·화면 간 이질감 제로 목표
- → 두 스파인은 shadcn에서 상속, 토큰은 shadcn 기본값을 참조/확장

### 미감의 닻 (Design Anchor)
- **토스** — 단순함 + 단계별 진행감
- **에어비앤비** — 지도 + 카드 예약 경험
- **카카오맵** — 핀 + 바텀시트 패턴
- → 셋의 장점만 취함

### 사용자 역할 / 페르소나
- 예약자(Booker): 총무형 "지우" / 개인형 "민재"
- 제공자(Provider): 사장 "현우"
- 관리자(Admin): 시드 관리자 (UJ 외)

### 핵심 사용자 여정 (PRD 정의)
- UJ-1 총무 지우 — 모임 장소 탐색 & 카카오톡 공유 (모바일 앱)
- UJ-2 개인 민재 — 즉시 근처 빈 방 확보, 상세 진입부터 3스텝 이내 (지도)
- UJ-3 제공자 현우 — 공간 등록(5분 이내) & 예약 관리
- UJ-4 누구든 — 플로팅 챗봇으로 실시간 질문 (첫 토큰 2초 이내)

### 정보구조 핵심 화면
- 1급 진입 3개: 스터디룸 찾기(지도/목록 전환), 예약현황, 즐겨찾기
- 전역: 플로팅 챗봇(모든 주요 화면), 로그인/계정
- 흐름: 탐색 → 상세(같은 페이지 내 예약 전개) → 확정 → 공유
- 제공자: 공간 등록/수정, 예약현황·거절, 후기 답글
- 관리자(별도 앱): 계정 관리, 예약 임의취소, 챗봇 문서 인제스트

### UX 성능/품질 바 (정량)
- 첫 지도 표시 ≤2초 / 챗봇 첫 토큰 ≤2초 / 로컬 UI 피드백 ≤100ms
- 단독 예약 상세 진입부터 ≤3스텝 / 첫 예약 ≤3분(중앙값, 안내 없이)
- 7개 마찰 지점 전부 통과 기준 충족

### 접근성 바
- 키보드 완전 지원(웹) + 스크린리더 ARIA 라벨
- **색상 의존 금지** — 핀 색 구분은 색 + 보조 표식(아이콘) 병행
- 상태 설계 매트릭스: 주요 화면 × {빈 상태 / 에러 / 오프라인 / 로딩} 전부 설계 존재

### 핵심 제약 (UX 영향)
- 결제·푸시·OAuth·이미지업로드 의도적 비포함 (MVP)
- 연속 슬롯만 선택(비연속 불허, D1) / 1시간 단위 고정 슬롯
- 제공자당 스터디룸 1개 / 자정 넘김 영업시간 없음
- 알림 = 접속 시 인앱 배너만 (푸시 아님): 도래 리마인드(24h) + 상태변경 통지
- 타임존: 모든 시간 판정은 룸 소재지 기준 (MVP = Asia/Seoul)
- 동시성: `UNIQUE(room_id, slot_start)` → UI는 동시 예약 충돌 실패를 우아하게 안내해야 함
- 지도 제공자: 카카오맵 (좌표 + 행정동 코드 동시 저장)

## Discovery 결정 기록 (진행 중)

### D-01 브랜드 감성 (2026-06-14)
- **활기·친근·경쾌** — 당근/카카오에 가까운 따뜻하고 말 걸기 쉬운 톤.
- 토스의 차분한 블루보다 따뜻한 결로 의도적 차별화. (닻 중 토스는 "단순함·단계별 진행감"만 취하고, 색·온도는 따뜻하게)

### D-02 다크 모드 (2026-06-14)
- **MVP = 라이트 전용.** 단, 토큰은 다크 확장 가능하도록 시맨틱 구조로 설계. 지금은 다크 화면을 그리지 않음.

### D-03 정보 밀도 (2026-06-14)
- **균형형.** 지도/목록 = 정보 밀도, 예약/상세 = 여백. 화면 성격에 따라 밀도 조절.

### D-04 포인트 컬러 방향 (2026-06-14)
- **웜(코랄/오렌지) 계열** 출발점. 구체 색은 시각 옵션(HTML 팔레트)으로 픽 예정.
- ⚠️ 검토 필요: 따뜻한 코랄/오렌지 프라이머리와 **핀 가용성 색(초록=예약가능 / 회색=마감)** 의 공존. 프라이머리가 핀 상태색과 충돌하지 않도록 가용성 색 계열을 명확히 분리할 것.

### D-05 색상 팔레트 확정 — "에너제틱 만다린" (2026-06-14)
- 시각 옵션(`.working/color-themes-1.html`, 테마 5)에서 **5번 에너제틱 만다린** 선택. 경쾌·발랄, 젊고 캐주얼한 결.
- 라이트 모드 시맨틱 토큰 (light only):

| 역할 | HEX |
|------|-----|
| primary | `#FF8A1E` |
| primary-foreground | `#3A2400` |
| secondary | `#FFF0D6` |
| accent | `#FFC24D` |
| background | `#FFFCF4` |
| surface/card | `#FFFFFF` |
| muted | `#F4EDDF` |
| border | `#F4E2C2` |
| foreground | `#28200F` |
| destructive | `#E03A2E` |
| **pin-available / success (초록)** | `#19A65A` |
| **pin-full (회색)** | `#B2AA9C` |

- 가용성 초록(`#19A65A`)이 만다린 프라이머리(`#FF8A1E`)와 색상환에서 충분히 멀어 충돌 우려 해소. (D-04 우려 항목 종결)
- 배경이 순백이 아닌 따뜻한 크림(`#FFFCF4`) → 친근함 강화.

### D-06 타이포그래피 (2026-06-14)
- **Pretendard** — 한국어 우선 제품의 사실상 표준. 깔끔·친근·고가독. shadcn/웹/RN 모두 적용 용이.
- 스케일은 shadcn 기본 타입 스케일 기반으로 Finalize에서 구체화. 한글 가독성 위해 line-height 넉넉히.

### D-07 모서리/형태 (2026-06-14)
- **shadcn 기본 라운드**(~8px, `radius: 0.5rem` 계열). 친근하되 절제된 균형.
- 활기 감성은 색·모션·마이크로카피로 살리고, 형태는 과하게 둥글지 않게.

### D-08 깊이감/엘리베이션 (2026-06-14)
- **혼합(고도별).** 기본 면은 플랫(경계선/면색으로 구분), **떠야 하는 요소만 그림자**: 바텀시트, 모달/다이얼로그, 토스트, 플로팅 챗봇 버튼.
- 바텀시트가 지도 위로 부드럽게 올라오는 카카오맵식 패턴과 정합.

### D-09 보이스 & 톤 (마이크로카피) (2026-06-14)
- **친근한 해요체.** 토스/당근 결. 따뜻하고 말 걸기 쉬움.
- 예: 성공 "예약이 완료됐어요!" / 동시성 실패 "앗, 방금 다른 분이 먼저 잡았어요. 가까운 빈 시간을 다시 보여드릴게요." / 빈 상태 "아직 등록된 스터디룸이 없어요."
- 브랜드 보이스 자체는 DESIGN.md.Brand & Style, 화면별 마이크로카피는 EXPERIENCE.md.Voice and Tone에 귀속.

### D-10 챗봇 성격·호칭 (2026-06-14)
- **이름 있는 친근한 도우미.** 말 걸기 편한 조력자. 범위 밖 질문은 정중히(해요체) 거절.
### D-11 챗봇 이름 확정 (2026-06-14)
- **"룸메이트" (Roommate).** 스터디룸 공간 컨셉 + "함께 찾아주는 친구" 결. 친근한 해요체로 안내.

### D-12 "더 있나요?" 프로브 결과 (2026-06-14)
- 추가 시각 자료(imports) 없음 / 특별 우려 화면 없음 / i18n: **MVP 한국어 전용** 확정.
- **모션**: 사용자 미지정 → `[ASSUMPTION]` 활기·경쾌 감성에 맞춰 **부드럽되 약간 생기 있는 마이크로모션**(짧은 스프링감, 100ms 이내 로컬 피드백, 과하지 않게)으로 채택. Finalize 검토 대상.

### D-13 접근성 검증 기반 색상 보정 (2026-06-14)
검증 리뷰(`review-accessibility.md`)에서 4개 색이 WCAG 2.2 AA 미달로 확인되어 보정. **만다린 브랜드 인상은 유지**(프라이머리 채움색 `#FF8A1E` 불변, 버튼 텍스트 6.22:1 합격), 의미색·링만 톤다운:

| 토큰 | 기존 | 보정 | 사유 |
|------|------|------|------|
| success / pin-available | `#19A65A` | **`#157F45`** | 배지 흰 글씨 대비 3.16→≥4.5:1 |
| destructive | `#E03A2E` | **`#CC3328`** | 흰 글씨 대비 4.36→≥4.5:1 |
| pin-full | `#B2AA9C` | **`#7E7466`** | 그래픽 지표 대비 2.25→≥3:1 |
| ring(포커스) | `#FF8A1E` | **`#D86E0A`** | 크림 배경 대비 2.30→≥3:1. 채움 primary와 분리된 ring 토큰. |

추가 접근성 강화(스파인 반영): prefers-reduced-motion 존중 / 터치 타겟 ≥44px(iOS)·48dp(Android) / aria-live 범위를 챗봇 외 인앱 배너까지 / 아이콘 단독 컨트롤(하트·FAB·지도목록 토글) 접근 이름 / 슬롯 선택 스크린리더 피드백 / RN 모달 포커스 격리 / **색+아이콘 병행 규칙을 핀뿐 아니라 배지·슬롯·배너 등 모든 상태색으로 확장**.
