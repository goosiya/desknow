# docs_corpus — 챗봇 문서 인제스트 대상 디렉터리

데스크나우 챗봇 "룸메이트"의 **서비스 안내 문서 RAG**(7.5)가 근거로 삼는 텍스트 문서를 이
디렉터리에 배치한다(architecture L341).

- 인제스트 파이프라인(Story 7.2)이 이 디렉터리의 `.md`/`.txt` 문서를 청크로 쪼개
  OpenAI `text-embedding-3-small`로 임베딩해 pgvector(`document_chunks`)에 **멱등 적재**한다.
- 수동/개발 실행: `uv run python scripts/ingest_docs.py`
- 관리 화면 실행(Story 8.4): 관리자 웹 `/ingest`에서 트리거하거나 `POST /api/v1/admin/ingest`(admin
  가드, 동기·리포트 반환)를 호출한다 — 이 디렉터리에 문서를 배치한 뒤 트리거한다(UI 업로드 아님).
- 재인제스트해도 내용이 같은 문서는 스킵된다(중복 벡터 미생성). 내용이 바뀐 문서만 재적재된다.
- corpus에서 삭제/리네임된 문서의 기존 청크는 인제스트 시 정리(reconcile)된다(orphan 잔존 방지 —
  8.4). 단, 디렉터리에 파일이 하나도 없으면 정리를 스킵해 전체 wipe를 막는다.

실 운영 문서 적재·관리 화면은 8.4(FR-33)가 가용화했다(트리거 + 리포트). 본 파일은 디렉터리 보존
및 파이프라인 수동 점검용 샘플이다.
