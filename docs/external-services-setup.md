## ⚠️ 시작 전 반드시 읽기

- **이 작업은 사람만 수행할 수 있습니다.** 가입 · 결제 수단 등록 · 콘솔 플랫폼 등록은 dev 에이전트가 대신할 수 없습니다.
- **키는 절대 프론트 코드나 저장소에 커밋하지 마세요.** 프론트에 노출되는 유일한 키는 **카카오 JavaScript 키(도메인 키)** 하나뿐이며, 이마저도 도메인 화이트리스트로 보호됩니다. 나머지 키는 모두 **백엔드 환경변수 전용**입니다 (NFR-6).
- 발급한 값은 `apps/api/.env` 파일에만 기입합니다. 이 파일은 `.gitignore`로 추적에서 제외됩니다. 형식·변수 이름은 `apps/api/.env.example`을 참고하세요.

## 1. 카카오 디벨로퍼스 (필수)

지도 · 주소 검색(지오코딩) · 카카오 공유에 사용합니다.

### 가입 및 앱 생성
1. 가입 URL: https://developers.kakao.com → 카카오 계정으로 로그인.
2. **내 애플리케이션 → 애플리케이션 추가하기** → 앱 이름/사업자명 입력 후 생성.
3. 생성된 앱의 **앱 키** 메뉴에서 아래 키를 확인합니다.

### 키 받는 위치 → `.env` 변수 매핑

| 콘솔에서 받는 위치 | `.env` 변수 | 용도 | 노출 범위 |
|---|---|---|---|
| 앱 키 → **JavaScript 키** | `KAKAO_JS_KEY` | 웹 지도 JS SDK | **프론트 노출(유일)** — 도메인 화이트리스트 필수 |
| 앱 키 → **REST API 키** | `KAKAO_REST_API_KEY` | 지오코딩 / 주소 변환 | 백엔드 전용 |
| 앱 키 → **네이티브 앱 키** | `KAKAO_NATIVE_APP_KEY` | RN 앱 카카오 공유 | 백엔드/앱 설정 |

### 플랫폼 등록
- **앱 설정 → 플랫폼**에서 사용할 플랫폼을 등록합니다.
  - **Web**: 서비스 웹 도메인 등록 (아래 "도메인 키 화이트리스트" 참고).
  - **iOS**: 번들 ID 등록 + URL 스킴 설정(`LSApplicationQueriesSchemes`에 `kakaokompassauth`, `kakaolink` 등 추가).
  - **Android**: 패키지명 + 키 해시 등록.

### 🔐 도메인 키 화이트리스트 등록 (AC 7 — 중요)
- 카카오 콘솔 **앱 설정 → 플랫폼 → Web → 사이트 도메인** 에 서비스가 실제로 배포될 도메인(예: `https://desknow.example.com`, 개발 시 `http://localhost:3000`)을 **명시적으로 등록**합니다.
- 이렇게 하면 프론트에 노출되는 `KAKAO_JS_KEY`도 **등록된 도메인에서만** 동작하여, 타 도메인에서의 무단 사용이 차단됩니다(NFR-6).

### ✅ 카카오맵 약관 서비스 유형 적합성 1차 확인 (AC 6 — PRD §8 Q1)

스터디룸 예약 서비스가 카카오맵 오픈 API 이용약관의 허용 서비스 유형에 부합하는지 1차 확인하고 결과를 아래에 기록합니다.

- **확인한 약관 위치/조항:**
  - 카카오 디벨로퍼스 운영정책(상업적 이용 / 쿼터·제한 조항) 및 지도/로컬 FAQ "상업적 사용" 안내.
  - 카카오 운영자 공식 답변(데브톡):
    - "지도 API의 상업적인 서비스에의 사용을 별도로 제한하지 않습니다." (R&D·회사·학교·개인 프로젝트 모두 가능, 트래픽 한도 내 무료) — devtalk.kakao.com/t/api/127999
    - "고객이 주소로 위치를 등록 → DB에 저장된 좌표를 카카오맵에 마커로 표시 → 관련 항목 조회" 형태에 대해 "말씀해주신 정도의 사용은 가능한 것으로 보입니다." — devtalk.kakao.com/t/topic/138475
- **확인 일자:** 2026-06-14
- **판정:** ☑ **적합** ☐ 부적합 ☐ 보류
- **판단 근거:** DeskNow는 카카오맵을 **도구로 활용**하는 일반 서비스(주소→좌표 지오코딩, 등록된 공간 좌표를 지도에 마커 표시, 슬롯 가용성 조회)로, 카카오 운영자가 허용 사례로 명시한 패턴과 사실상 동일하다. API 재판매·지도 기능 자체에 대한 별도 과금 없음(MVP 결제 없음).
- **운영 시 준수해야 할 제약(중요 — 위반 시 API 차단 사유):**
  1. 지도 표시·출력물은 **카카오맵 가이드라인**(로고/저작권 표기 등) 준수.
  2. **쿼터(일/월/MAU 한도) 모니터링** — 초과 시 쿼터 상향 신청. 한도 초과 사용 금지.
  3. **Local(로컬) API 검색 결과를 대량으로 로컬 저장/DB화 금지.** DeskNow는 제공자가 직접 등록한 공간의 좌표만 저장(허용 범위). 주변 장소 검색 결과를 긁어 축적하지 말 것.
  4. 문서화된 방법(JS SDK, 공식 REST 엔드포인트)으로만 호출. 비공식 리소스 직접 접근 금지.
- **부적합/보류 시 대안 메모 (대체 지도 제공자 검토 — 후속 스토리 E3 진행 판단에 반영):**
  - 현재 판정 "적합"이므로 즉시 대안 불필요. **단, 카카오 정책 변경 또는 위 제약 위반으로 API 차단 리스크가 현실화될 경우**: 지오코딩/지도 제공자 대체(예: 네이버 지도, VWorld, OSM/Mapbox 등) 재검토. `KAKAO_*` 키 격리 구조는 그대로 두고 지도 클라이언트만 교체 가능하도록 E3에서 추상화 권장.

> **부적합으로 판명되면** 그 사실과 대안 필요성을 위에 반드시 남겨 후속 스토리(Epic 3 탐색)의 진행 가부 판단에 반영합니다. (현재 1차 판정: **적합** — E3 진행 가능)

### 검증 방법
- `.env`에 `KAKAO_JS_KEY`, `KAKAO_REST_API_KEY`(+ 선택 `KAKAO_NATIVE_APP_KEY`)를 채운 뒤, 본 가이드 마지막의 [키 로드 검증](#키-로드-검증-환경-점검) 절차를 실행합니다.

## 2. OpenAI (필수)

기준 LLM(GPT, ≤2초 SLA)과 임베딩(`text-embedding-3-small`)에 사용합니다.

1. 가입 URL: https://platform.openai.com/signup
2. **⚠️ 결제 수단 등록(API 과금 발생):** Billing → Payment methods에서 카드 등록. API 사용 시 사용량 기반 과금이 발생합니다. 사용량 한도(usage limit) 설정을 권장합니다.
3. **API 키 발급:** https://platform.openai.com/api-keys → Create new secret key.

| `.env` 변수 | 값 | 비고 |
|---|---|---|
| `OPENAI_API_KEY` | 발급받은 secret key(`sk-...`) | 백엔드 전용 |
| `EMBEDDING_MODEL` | `text-embedding-3-small` | **단일 고정** — 교체 시 전체 재임베딩 필요. 기본값이 이 값이므로 변경 불필요 |

## 키 로드 검증 (환경 점검)

소유자가 `apps/api/.env`를 채운 뒤, 백엔드가 필수 키를 정상 로드하는지 확인합니다.

```bash
cd apps/api
uv sync                       # 최소 의존성(pydantic-settings) 설치
uv run python -m app.core.config --check
```

- **필수 키가 누락된 경우:** 어떤 키가 빠졌는지 명시하는 오류와 함께 **기동이 실패**합니다(fail-fast). 누락된 변수를 `.env`에 채운 뒤 다시 실행하세요.
- **모든 필수 키가 존재하는 경우:** 정상 종료되며, 각 키의 **존재 여부와 마스킹된 값**(예: `sk-…1234`)이 출력됩니다. 비밀 값 전체는 절대 출력되지 않습니다.

> 필수 키: `KAKAO_REST_API_KEY`, `OPENAI_API_KEY`, `KAKAO_JS_KEY` (그리고 기본값이 설정된 `EMBEDDING_MODEL`).
> 선택 키: `ANTHROPIC_API_KEY`, `GOOGLE_AI_API_KEY`, `KAKAO_NATIVE_APP_KEY`, `DATABASE_URL`(1.4에서 필수화).
