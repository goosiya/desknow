## 스터디룸 예약 서비스 - 데스크나우(DeskNow)

사용자가 스터디룸 제공자가 등록한 스터디룸을 지도 또는 목록에서 찾아 예약하는 서비스(MVP)

1. 기능 범위
    - 고객(예약자):
        - 계정 가입(이메일)
          - 이메일 유효성 체크, 비밀번호 강화 정책 적용(최소 8자리, 대문자/특수문자/숫자 각 1개 이상 포함)
          - 같은 메일 중복 가입 배제(MVP 계정 비활성 상황 고려 안함)
          - MVP는 OAuth 사용 안함
        - 로그인
        - 스터디룸 찾기
            - 기본
                - 현재 위치한 곳이 기술적으로 가능한 경우 기준으로 반경 몇 km 내외 또는 동/읍/면 기준
                - 지도 상의 핀 위치를 표현할 때 위치 정보 데이터와 주소 기반 데이터의 변환 등에 대한 기술적 가능 및 제약에 대해 확인
            - 지도에서 찾기 - 등록된 스터디룸 핀 표현 및 예약 가능 표시 또는 핀 색상 구분, 핀 클릭 시 바텀시트로 해당 스터디룸의 기본 정보 표현(업체 상세 페이지 이동), 스터디룸 즐겨찾기
            - 목록으로 찾기 - 시/군/구, 동/읍/면 기준의 콤보를 이용하여 조회하고 목록으로 스터디룸 나열, 선택 시 스터디룸 정보 화면, 스터디룸 즐겨찾기
        - 스터디룸의 상세 정보를 확인하고 예약하기(같은 페이지 내에서 UI 전개)
          - 시간 선택(달력으로 일자 선택 → 1시간 단위로 예약이 비어 있는 시간 선택)
            - 금일, 현재시간 이후만 달력이나 시간 선택 활성(예약된 시간 비활성 포함)
          - 중복 예약 방지(예약을 잘못한 경우 취소하고 다시 예약)
        - MVP 에서는 예약 시 결제를 붙이지 않음(바로 예약)
        - 예약 취소 기능(예약일 예약시간 기준 6기간 전까지)
        - 예약현황 페이지 제공(이전 예약 히스토리), 즐겨찾기 스터디룸 페이지
        - 웹/앱 로그인 시 예약 도래 알림(1일 기준으로, 접속 시 마다 알림 - 다시 보지 않기 처리 구현)
        - 예약 사항 sns 공유(MVP는 카카오톡만)
        - 사용 후기 등록(스터디룸 업체명)
          - 평점 등록(별 5개 만점) 및 후기 작성(500자 제한)
          - 고객 자신이 등록한 후기 모아보기 없음
    - 스터디룸제공자
        - 계정 가입(이메일)
          - 이메일 유효성 체크, 비밀번호 강화 정책 적용(최소 8자리, 대문자/특수문자/숫자 각 1개 이상 포함)
          - 같은 메일 중복 가입 배제(MVP 계정 비활성 상황 고려 안함)
          - MVP는 OAuth 사용 안함
        - 로그인
        - 스터디룸 설정/수정 - 스터디룸 이름(업체명), 인수 기준, 시간 당 금액, 영업시간, 주소(지도 검색을 위한 주소 찾기 기반), 부대시설(주차/빔프로젝터(TV)/커피머신 등등), 화이트보드, 오픈/룸 환경 구분, wifi, 기타
          - MVP 에서는 가능 최대 인원 기준으로 한개만 등록
        - 예약자 현황 페이지 제공(예약거부 가능, 거부 시 해당 예약 시간 활성)
        - 후기 보기 및 답글 달기
    - 관리자
        - 로그인
            - MVP는 시드 관리자만 사용
            - 챗봇(RAG) 문서 관리
                - MVP 에서는 서비스 이용방법, FAQ 등 원본 문서를 지정 디렉토리에 올려 배포(업로드 X)
                - 문서 인제스트 처리 관리만 함
        - 고객 관리, 스터디룸 제공자 관리(계정 비활성)
        - 요청 관리(예약 임의 취소)
    - 챗봇
        - RAG를 이용한 챗봇 제공
        - 챗봇은 어느 페이지에서든 아이콘 노출(플로팅)
        - 로그아웃 전까지 챗봇 대화 내용 유지
        - 문서검색 및 DB검색을 이용한 사용자의 질문에 대한 답변(서비스 범위 내에서만 대답, 일반적인 대화는 거부 안내)
        - LLM API(openAI, google,  anthropic 스위치 가능하게)
        - 예약 가능한 스터디룸을 결과로 표현할 경우 상위 3개와 하단 더보기 표현. 링크로 해당 상세 페이지 이동
2. 기술 스택
    - 사용자(고객/스터디룸제공자)용 반응형 웹 : Next.js
    - 관리자용 반응형 웹 : Next.js
    - 사용자(고객/스터디룸제공자)용 모바일 앱 : React Native
    - 인증/세션 : JWT 기반
    - UI/UX : shadcn
    - 통합 API(FastAPI)
    - DB (PostgreSQL) - RAG를 위한 vector DB 확장 포함
    - 배포(Railway) : 프론트엔드, 백엔드, PostgreSQL, 모바일 빌드(EAS)
    - 챗봇 : RAG, langGraph, langChain, LLM(API)
3. UI/UX
    - 웹/앱에서 사용자가 주요 페이지에 접근이 가능한 네비게이션 구성
      - 스터디룸 찾기, 예약현황, 즐겨찾기 등
4. 지도 및 주소 찾기
    - 네이버 지도, 네이버 주소 찾기 사용 가능한지 확인
5. DB
    - 개발용으로 PostgreSQL(Supabase)
    - 운영 배포용 PostgreSQL(Railway)
    - 운영 배포 시 이관(dump 파일)
6. 향후 고도화 대상
    - 개인정보 처리와 관련된 데이터 분리보관/삭제
    - OAuth 적용
    - 실제 결제(PG)
    - 가입 시 서비스 이용 약관 동의 처리(약관 버전 관리)
    - 환불 정책
        - 예약 후 노쇼에 대한 정책 - 부분환불/미환불
        - 예약 시간 기준 취소 시간에 따른 환불금 차등 정책
    - 예약 리마인드 푸시, 메일 알림
    - 업로드 기능(이미지, 파일) 및 스토리지 사용 정책
    - 예약 내용 sns 공유 사이트 확장
    - 계정 비활성과 중복 가입에 대한 정책
    - OAuth 적용(기존 가입 메일 적용)
    - 예약 내용 sns 확장
    - 관리자 계정 생성 기능
    - 스터디룸 등록 고도화
      - 가능 인원별 또는 개인석/단체석 확장
      - 고도화 기점으로 기존 단일 등록 진행했던 사항은 벡데이터로 남길지 확장에 포함시킬지 결정
    - 챗봇 고도화
